# 任务单：P5.7-R9（模型真实能力验收门）

优先级：P0（能力扩展前的硬门禁）

## 目标（冻结）

1. 用真实场景验证模型的基础智能体能力是否可用。
2. 将“工具调用成功”和“用户可展示结果”分离验收，避免伪成功。
3. 固化 8 项能力基线，作为后续能力扩展的前置门禁。

## 范围（8 项验收）

1. 文件查看：用户询问“可以查看我的文件吗”，模型必须触发真实工具并返回真实结果。
2. 自拍编排：模型能为“生成自拍”任务做计划并执行，最终产出图片文件。
3. 定时提醒：模型先补齐信息（事项/时间/时区），再落地定时任务并在触发时回复。
4. 记忆能力：短期上下文记忆 + 长期记忆（明确“请记住”后可检索召回）。
5. 任务管理：任务可被编排、文件化存储、再读取。
6. 系统提示词索引：提示词中有命令/技能索引，模型按索引使用能力。
7. 工具与命令使用：模型正确调用系统提供的工具和 CLI 命令。
8. 重点能力：记忆存储与召回、任务编排、定时触发。

## 非范围

1. 不新增新业务能力（仅验证现有能力基线）。
2. 不修改业务语义（仅修复导致验收失败的缺陷）。
3. 不放宽错误码和可观测字段口径。

## 验收原则（冻结）

1. R1（工具命中）：需要工具的请求必须出现真实工具调用证据。
2. R2（可展示结果）：最终回复必须是可展示自然语言/可访问产物路径，不是协议碎片。
3. E2E（端到端）：从用户输入到可用产物/回复全链路成功。

## 证据要求（每项必须留痕）

1. 输入（用户原始请求）
2. 工具调用证据（tool name / step / 错误码）
3. 输出（最终用户可见回答或文件路径）
4. 产物（如图片、任务文件、记忆文件）
5. 结论（PASS/FAIL + 原因）

## 执行方式

1. 使用 `scripts/r9-real-smoke.ts` 生成验收模板。
2. 在真实环境逐项执行 8 个场景，并回填证据。
3. 回填结果文件默认输出到 `AIDOCS/reports/r9-real-smoke-template.md`。

## 硬验收

1. `npx tsc --noEmit`
2. `npm test`（0 fail）
3. `npm run docs:check`
4. 8 项能力全部有证据；重点三项（记忆召回/任务编排/定时触发）必须 PASS。
