# notes 目录规范

## 目录结构

```text
docs/notes/
├── README.md
├── research-template.md
└── research-YYMMDD-<topic>.md
```

## 架构决策

1. `docs/notes/` 用于研究、实验与验证留痕，不替代 issue/plan。
2. 研究文档聚焦证据，不承载最终决策。

## 开发规范

1. 文件名固定：`research-YYMMDD-<topic>.md`。
2. 必须包含：受影响模块 / 数据流 / 风险点 / 验证方式。
