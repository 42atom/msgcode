# PL0397：Nexu Host And Thin Service Selective Adoption Baseline

## 结论

不照抄 `nexu`，只吸收两刀：

- 正式宿主 runtime
- 薄 service

明确不吸收：

- 重量级 runtime plugin 树
- 初始化 wait/poll 兜底
- controller 平台化

## 对照结论

### 可以直接吸收

1. 宿主必须是真宿主  
   对应 msgcode：
   - `src/electron/main.ts`
   - `src/electron/preload.ts`
   - `src/electron/renderer.ts`

2. 外部能力前面要有薄 service  
   对应 msgcode：
   - 后续本机音频
   - `homepod-bridge`
   - 其他 sidecar 能力

### 不直接吸收

1. 重 plugin runtime 树  
2. 模块全局 runtime + wait/poll 初始化兜底  
3. 大 controller 中心化

## 落地顺序

1. 继续主线：`tk0377 / tk0381`  
   先把正式 UI in `src` 收稳。

2. 继续音频支线：`tk0395 / tk0396`  
   先把 `input_channel / output_channel / playback_policy` 定稳。

3. 后续若接 HomePod  
   不做 openclaw 兼容层，只做 `homepod-bridge` pack：
   - 薄 HTTP bridge
   - health
   - start/stop
   - Shortcut 模板

## Guardrails

- service 只翻译协议，不做第二个大脑
- pack 可以带边车代码，但不能带第二套 runtime
- renderer 不直接 import runtime
- 不为“更完整”提前做 plugin host
