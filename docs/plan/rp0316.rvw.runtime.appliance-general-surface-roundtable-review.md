# RP0316：appliance general 圆桌复核

Decision:
- Recommendation: go
- Confidence: high
- Scope: 允许这条切片进入 rvw，但只限当前三类真相源

Why:
- `SystemsArchitect`：这次没有新建 `settings.json`，结构上仍是单一主链。
- `SkepticalOperator`：`WORKSPACE_ROOT` 缺失时返回 warning 空态，比偷偷猜默认目录更诚实。
- `UserAdvocate`：对小白来说，日志路径和启动方式是“通用”里最像设置的两项，其余原型字段现在没有真相源，先不做更不误导。
- `ExecutionLead`：这刀小、可验证、可回滚，不会把 UI 原型直接变成配置平台。

Risks:
1. UI 可能继续拿这条 CLI 去承接原型里尚未实现的字段。
2. `WORKSPACE_ROOT` 仍来自环境变量，若宿主没配，会长期停留 warning。
3. `doctor` 与 `general` 都会展示日志相关信息，后续要避免文案重复。

Guardrails:
- 前置条件:
  - 只允许读取 `WORKSPACE_ROOT`、log 路径、launchd 状态
  - 不允许顺手把语言、账号连接、默认场景工作区做掉
- 监控指标:
  - `appliance general` 的 warnings 是否只剩真实缺项
  - UI 是否把 warning 空态渲染成“待配置”，而不是假装已有默认值
- 回滚条件:
  - 若这条切片开始要求新增 `general.json` 或本地 settings 写回，立即回滚

Next Actions:
1. 维持 `appliance general` 为只读面
2. 后续若真要做多语言或账号连接，单开新切片
3. UI 只消费当前已有字段，不替缺失字段编默认值

Evidence:
- Docs: `tk0314`, `rp0315`
- Code: `src/runtime/workspace-general.ts`, `src/cli/appliance.ts`
- Tests: `bun test test/p7-tk0314-appliance-general-contract.test.ts`
