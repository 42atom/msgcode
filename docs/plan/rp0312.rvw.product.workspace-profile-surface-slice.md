# RP0312：工作区“我的资料”读面实现切片

## 结论

这次没有新建 `profile.json`，也没有把 Settings 扩成总接口。

只做了一条最薄读面：

- `msgcode appliance profile --workspace <...> --json`

它只复用三份现有真相源：

1. `.msgcode/config.json`
2. `.msgcode/SOUL.md`
3. `.msgcode/ORG.md`

唯一新增字段是：

- `profile.name`

## 为什么这么收

因为设置页“我的资料”本质上只缺一个东西：

- 我的称呼

其余两段已经有文件：

- SOUL -> `SOUL.md`
- 组织信息 -> `ORG.md`

如果这时新起一个 `profile.json` 或 `settings.json`，就是重复中心。

## 这次的文件读面

### 1. 我的称呼

来源：

- `<workspace>/.msgcode/config.json`
- key: `profile.name`

理由：

- 只是一个标量
- 放配置最薄
- 不值得为它单独起新文件

### 2. 智能体全局行为准则

来源：

- `<workspace>/.msgcode/SOUL.md`

这次直接返回全文，不再发明第二份摘要。

### 3. 组织信息

来源：

- `<workspace>/.msgcode/ORG.md`

解析规则：

1. `名称`
2. `位置城市`
3. 若没有 `位置城市`，兼容旧字段 `交税地`

## 这次刻意不做

1. 不做 settings 写入器
2. 不做“通用/诊断/能力配置”统一 JSON
3. 不做 profile 中心
4. 不复制 ORG/SOUL 到别的文件
5. 不把 `交税地 -> 位置城市` 做成仓库级大迁移

## 当前已知边界

1. `profile.name` 目前是新字段，真实工作区还未必都已填
2. `ORG.md` 仍是 appliance 老字段口径，Settings 页这边只是兼容读取
3. 这条切片只负责只读，不承诺 UI 写回
