# PL0335：Web composer RuntimeRouter one-shot consumer

## 最小路径

1. 读取单条 inbox 请求
2. 还原最小 `InboundMessage`
3. 构造 `HandlerContext`
4. 调用 `RuntimeRouterHandler.handle()`
5. 结果由现有 session/thread 落盘承接

## 为什么不走 listener

- listener 是通道适配层
- 它自带 Feishu 路由、当前 transport、sendClient 假设
- 为 Web 去碰它，会污染现有通道主链

## Guardrails

- 不起 daemon
- 不扩 transport
- 不要求 UI 维护消息状态
- 第一版只承接自然语言输入
