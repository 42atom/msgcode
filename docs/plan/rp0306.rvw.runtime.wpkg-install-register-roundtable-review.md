# RP0306：wpkg 安装注册圆桌复核

Decision:
- Recommendation: go
- Confidence: medium
- Scope: 保留最小 install/register 主链，拒绝把它长成包管理器

Why:
- 当前缺口确实只剩安装注册动作，继续空谈会让 `0290` 长期停在协议层。
- 现有实现已经坚持工作区文件真相源，没有引入数据库、后台守护或市场控制面。
- 黑盒测试已经覆盖“能装”和“重复装报错”两条主链。

Risks:
1) 静默覆盖已有包或站点，会把工作区注册表写脏。
2) 注册死路径，会让 UI 能看见但打不开。
3) 假装 pack skill 已可用，会制造“装了但 runtime 找不到”的假能力。

Guardrails:
- 前置条件:
  - 只接受带 `manifest.json` 的合法 `.wpkg`
  - `pack id` / `site id` 冲突直接失败
- 监控指标:
  - `install-pack` 成功后 `packs.json` 和 `sites.json` 是否同时落盘
  - `appliance hall/sites` 是否立即可见
- 回滚条件:
  - 发现会静默改写坏注册表
  - 发现能注册不存在的站点或 skill 路径

Next Actions:
1) 保留当前 `install-pack` 薄入口，不新增升级/卸载。
2) 文档统一到 `.msgcode/packs/user/<pack-id>/...` 真实路径。
3) 把 skill 自动发现继续留给后续独立切片，不在这轮偷做。

Evidence:
- Docs: `WPKG.md`, `tk0305`, `rp0305`
- Code: `src/runtime/workspace-wpkg-install.ts`, `src/cli/appliance.ts`
- Tests: `bun test test/p7-tk0295-appliance-packs-surface.test.ts test/p7-tk0263-appliance-sites-contract.test.ts test/p7-tk0305-wpkg-install-register.test.ts`
- Logs: 无需新增日志；当前以黑盒测试和工作区文件落盘为准

## 角色摘要

SystemsArchitect：
- 保留单一安装入口是对的。
- `channel/market/license` 都不该进这轮。
- `sites.json` 必须继续作为唯一 UI 站点入口真相源。

SkepticalOperator：
- 重名包和重名站点必须直接失败，不能自动覆盖。
- 写路径时不能静默吞掉坏 `packs.json / sites.json`。
- sidecar 入口如果指向不存在文件，就不该允许安装成功。

RiskGuardian：
- `skill` 自动发现还没打通，必须显式 warning。
- `external` 入口如果存在，也要和本地 sidecar 分开处理。
- 当前切片不要接触支付、签名和远程下载。

ExecutionLead：
- 这轮目标已经足够，继续扩大会拖慢主线。
- 下一步该回到 `0290` 父线，决定是否继续拆“卸载/升级”还是先接 UI 可见面。

## 吸收结果

这轮实际吸收了四条：

1. 安装时拒绝静默覆盖
2. 安装时拒绝注册死路径
3. `external` 入口按 URL 处理，不和 sidecar 混写
4. CLI 对 skill 自动发现缺口显式 warning

没吸收的：

1. 不做升级/卸载
2. 不做签名/支付
3. 不做包中心或下载源
