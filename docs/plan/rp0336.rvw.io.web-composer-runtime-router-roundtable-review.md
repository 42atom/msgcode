# RP0336：Web composer one-shot consumer 圆桌复核

Decision:
- Recommendation: go
- Confidence: medium
- Scope: 保留 `inbox file -> one-shot consume -> RuntimeRouterHandler -> session/thread truth` 这一条短主链

Why:
- `listener` 仍留在通道适配层，没有被 Web 入口反向污染。
- `consume-web` 只处理一条 request，没有 daemon，没有 transport，没有 optimistic bubble。
- 回归锁已经证明 session/thread 真相继续复用现有主链，而不是 UI 自己造气泡状态。

Risks:
1. 目前只支持自然语言文本，附件和 slash 仍未定义。
2. `consume-web` 还是显式命令，尚未接到 UI 的自动消费触发点。
3. inbox 坏文件现在会报错并保留 `new`，后续可能需要单独的人类修复入口。

Guardrails:
- 前置条件: Web 输入必须先落 `.msgcode/inbox/*.md`
- 监控指标: triaged 推进、session window 落盘、thread 落盘
- 回滚条件: 一旦需要新 transport、daemon 或 UI optimistic state，就暂停扩张，回到文件主链

Next Actions:
1. 不继续扩 transport，只考虑 UI 什么时候调用 `consume-web`
2. 如要支持附件，继续沿 inbox 文件真相扩，不加并行状态中心
3. UI 只读 `.msgcode/sessions/*.jsonl` 与 `.msgcode/threads/*.md`

Evidence:
- Docs: `pl0335`, `tk0335`
- Code: `src/cli/inbox.ts`, `src/runtime/inbox-store.ts`, `src/handlers.ts`
- Tests: `bun test test/p7-tk0335-web-composer-runtime-router-consumer.test.ts`
