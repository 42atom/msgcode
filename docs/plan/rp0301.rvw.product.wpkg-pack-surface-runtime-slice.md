# RP0301：wpkg packs surface 实现切片

## 结论

`0295` 第一版只做到了：

- 读 `packs.json`
- 透给 `appliance hall`
- 让主界面看见“默认内置 / 用户安装”

没有做：

- 安装器
- 卸载/升级
- skill 发现
- 授权/市场

## 本轮实现

新增：

- `src/runtime/workspace-packs.ts`
- `test/p7-tk0295-appliance-packs-surface.test.ts`

扩展：

- `src/cli/appliance.ts`
  - `appliance hall` 现在会输出 `packs`

## 读面主链

当前只读：

- `<workspace>/.msgcode/packs.json`

若文件缺失：

- 返回空分组

若 JSON 非法或顶层 schema 漂移：

- 返回 warning
- 同时仍降级为空分组

## 关键收刀

这轮最重要的不是“能读 pack”，而是“不要把 hall 变成市场面”。

所以 `hall.packs` 当前只暴露：

- `builtin[]`
- `user[]`

每个包项只保留：

- `id`
- `name`
- `version`
- `enabled`

刻意不暴露：

- `sourcePath`
- `author`
- `commercial`
- `licenseType`

这些字段如果未来要用，也该留在注册表或别的 surface，不该先塞进主界面。

## 圆桌吸收

本轮圆桌最有效的意见有三条，已经吸收：

1. `hall.packs` 不能偷长市场味  
   所以删掉了 `commercial / licenseType`

2. `hall.packs` 不能带调试路径  
   所以删掉了 `sourcePath`

3. 顶层 schema 漂移不能静默吞掉  
   所以新增 `WORKSPACE_PACKS_INVALID_SCHEMA`

## 结果

`0295` 现在已经有一条可站住的最小主链：

- `packs.json` 是工作区真相源
- `hall.packs` 是主界面的最薄显示面
- UI 不需要扫目录

这符合当前原则：

- 克制
- Unix 哲学
- Everything is a file
