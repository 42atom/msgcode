# post-review runtime repair plan

## 为什么现有文件不能直接承载

- 现有 review 文档只记录结论，不承载执行状态
- 这三条修复线的验收口径不同，不能塞进同一份 slice 里混写

## 实施顺序

### 1. 0358 appliance CLI 模块化

- 先止住 `appliance.ts` 继续膨胀
- 目标：
  - `appliance.ts` 只保留命令注册/wire-up
  - profile / capabilities / people / archive / thread 拆到子模块

### 2. 0359 wpkg 解包路径清理

- 再收 `workspace-wpkg-install.ts`
- 目标：
  - `zstd` 改懒解析
  - 解包去掉 `/bin/sh -c`
  - 保持现有 `.tar.zst` 主链不变

### 3. 0360 front matter parser 去重

- 最后收共用 util
- 目标：
  - 不再在各 runtime 文件各写一份 `parseSimpleFrontMatter`
  - 只抽最薄 util，不引入完整 markdown 框架

## 非目标

- 不做大规模目录翻修
- 不上 UI 框架
- 不重写无关 CLI
