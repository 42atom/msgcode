# RP0298：status.log 实现级圆桌复核

## 结论

这刀可以保留，但要再砍一层。

保留：

- 单一 writer
- 单行格式
- append-only
- tail 最近 10 条

删掉：

- 写入侧去重
- 线程过滤
- 会让调用方做主的额外旋钮

## 圆桌意见

### 实现简洁性

`status.log` 当前方向是对的：

- 一个文件
- 一个 writer
- 一个 tail reader

但去重一加，writer 就开始替日志做判断，已经偏离“只追加”的主职责。

### 合同硬度

需要补的不是更多层，而是更硬的输入校验：

- `thread` 不能为空
- `summary` 不能为空
- `refPath` 必须位于工作区内
- `refLine` 必须是合法正整数

### 未来演进

如果以后要处理噪音，优先在消费侧做，不在 writer 侧吞日志。

第一版先接受：

- 可能有重复
- 但每条都是真实追加

## 本轮吸收

1. writer 不再做最近窗口去重
2. `readWorkspaceStatusTail()` 不再承担线程过滤
3. 新增坏输入测试
4. 文档同步改成：v1 只做 append + tail，不做去重

## 结果

`0294` 当前实现已经符合这条边界：

- runtime 只提供最薄 append/read 合同
- UI 不扫描全文件
- 模型不直接写原始行

下一步可以提交，不需要再为 `status.log` 长更多机制。
