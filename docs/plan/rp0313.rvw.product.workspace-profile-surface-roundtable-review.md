# RP0313：工作区“我的资料”读面圆桌审议

## 参会视角

- UserAdvocate
- SystemsArchitect
- SkepticalOperator
- ExecutionLead

## 共识

0312 现在的收法可以接受。

保留：

1. 只读
2. 只补 `profile.name`
3. 继续复用 `SOUL.md`
4. 继续复用 `ORG.md`
5. `位置城市` 兼容旧字段 `交税地`

否决：

1. 新建 `profile.json`
2. 新建 `settings.json`
3. 把 Settings 一次性做成总出口
4. 现在就做 ORG 字段全仓迁移

## 主要意见

### User 视角

“我的资料”第一页必须有真相源，否则原型只是摆设。

但用户不会接受：

- 为了一个“我的称呼”再多一份中心文件

所以 `profile.name` 放进 config 是合理的。

### 架构视角

SOUL 和组织信息已经有现成文件。

若这次再复制一份到 profile 中心，就是重复真相源。

因此这条线必须坚持：

- 一标量进 config
- 大文本继续留原文件

### 实施视角

Settings 页下一步很可能还会继续想加：

- 通用
- 诊断
- 能力配置

所以这次必须明确：

- 0312 不是 settings 总线
- 它只是“我的资料”读面

### 真实工作区核对

对真实 `family` 工作区跑 `msgcode appliance profile --workspace /Users/admin/msgcode-workspaces/family --json` 后，当前结果是：

- `profile.name` 为空
- `SOUL.md` 缺失
- `ORG.md` 缺失

这说明 0312 的当前实现口径是诚实的：

- 缺什么就 warning 什么
- 不偷偷补第二真相源
- 不把原型假数据伪装成真实配置

## 审议后结论

可以继续按当前实现提交。

但要保留一个提醒：

- 若以后“位置城市”和 appliance 的 `ORG.md` 长期冲突，再单开迁移切片，不在 0312 里顺手大改。
