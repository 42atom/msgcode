# PL0322：settings mutation 最小基线

## 问题

设置页现在已经有读面。

若继续往前走，最危险的不是“写接口没做”，而是：

- UI 直接写 `.md/.json`
- 一部分 tab 假装可保存
- 不同 tab 各自长一套写协议

## 单一主链

唯一允许的写主链：

`UI -> msgcode CLI mutation -> 真相文件`

禁止：

- `UI -> 直接改 ORG.md/SOUL.md/config.json`
- `UI -> 内置正则替换 markdown`
- `UI -> 维护第二份 settings 缓存文件`

## 可写分组

当前只开放：

### 我的资料

- 我的称呼 -> `.msgcode/config.json`
- SOUL -> `.msgcode/SOUL.md`
- 组织名称 / 位置城市 -> `.msgcode/ORG.md`

## 继续只读的分组

### 通用

当前主要是系统事实：

- `WORKSPACE_ROOT`
- log 路径
- launchd 状态

其中大部分不是用户在设置页里稳定改写的本地真相。

### 智能体能力配置

当前只有只读硬事实：

- 当前 lane/provider
- 本地模型路径
- ASR/TTS 可见性

还没有统一 provider 写主链。

### 诊断

永远是只读面。

## 推荐动作

1. 先实现 profile mutation，不扩 general/capabilities。
2. UI 只在 profile 页显示保存动作。
3. 其他 tab 显示当前状态，但不承诺保存。
4. 只读 tab 不给输入框和保存按钮假象。
