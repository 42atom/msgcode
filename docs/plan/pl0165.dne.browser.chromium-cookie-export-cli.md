# chromium-cookie-export-cli

## Problem

需要一个不进 Tool Bus 的本机脚本，从 macOS 系统 Chromium profiles 提取 cookies，供多角色浏览器自动化显式复用登录态。

约束：

- 只做脚本，不做 browser/tool 厚集成
- 默认不在 stdout 打印 cookie value
- 支持多 profile 扫描与导出
- 导出格式直接对齐 Playwright cookies JSON

## Decision

采用单文件 CLI：

- `scripts/chromium-cookies-export.mjs`

实现边界：

- macOS only
- 读取 Chromium `Cookies` SQLite
- 通过 Keychain `Safe Storage` 密码解密 `encrypted_value`
- `--list` 只列 profile
- `--domain` 只输出摘要
- `--out` / `--out-dir` 才写文件
- `--redact true` 可生成脱敏文件用于验证流程

不做：

- 不进 Tool Bus
- 不自动注入 browser 工具
- 不在 stdout 暴露 cookie values

## CLI Contract

支持：

- `--list --browser <chrome|edge|brave|arc|chromium>`
- `--browser <b> --domain <d>`
- `--browser <b> --domain <d> --profile <p> --out <file>`
- `--browser <b> --domain <d> --out-dir <dir>`
- `--redact true`

配套说明书：

- `src/skills/runtime/patchright-browser/SKILL.md`

防误提交：

- `.gitignore` 忽略 `*.cookies.json`

## Verification

- `node scripts/chromium-cookies-export.mjs --list --browser chrome`
- `node scripts/chromium-cookies-export.mjs --browser chrome --domain google.com`
- `node scripts/chromium-cookies-export.mjs --browser chrome --domain google.com --profile Default --out /tmp/msgcode-cookie-one.cookies.json --redact true`
- `node scripts/chromium-cookies-export.mjs --browser chrome --domain google.com --out-dir /tmp/msgcode-cookie-many --redact true`

结果：

- 能列出系统 Chrome profiles 与 display name
- 域名扫描只输出 `cookieCount / uniqueNames / sampleNames`
- `--out` 能写单 profile Playwright cookies JSON
- `--out-dir` 能按 profile 批量写文件
- `--redact true` 下文件内容不含真实 values
