# RP0293：family 工作区上的 Main UI 数据面对照

## 结论

`family` 真实工作区已经足够支撑 Main UI 的三块主面：

- 左栏线程
- 当前线程聊天
- 日历 / 定时任务

当前还缺两块真相源：

- `status.log`
- 邻居 mailbox/health

所以 `0293` 的正确落地不是补新控制层，而是：

- 有真相源就直接读
- 没真相源就明确降级

## 已存在的真相源

### 工作区当前运行上下文

文件：

- `/Users/admin/msgcode-workspaces/family/.msgcode/config.json`

当前可读到：

- `runtime.current_transport`
- `runtime.current_chat_id`
- `runtime.current_chat_guid`

这足够支撑：

- 当前工作区运行 transport
- 当前默认线程来源属性

### 线程列表

目录：

- `/Users/admin/msgcode-workspaces/family/.msgcode/threads`

当前文件形态是：

- 一线程一份 `.md`
- front matter 包含 `threadId/chatId/workspace/workspacePath/createdAt`
- 正文按 `## Turn / ### User / ### Assistant` 组织

这足够支撑：

- 左栏线程列表
- `Recent` 排序
- 当前线程正文

不用先发明：

- `thread-index.json`

### 日历 / 定时任务

目录：

- `/Users/admin/msgcode-workspaces/family/.msgcode/schedules`

当前已有：

- `pick-up-kids.json`
- `pickup-kid.json`
- `send-kids-school.json`

这足够支撑：

- 右栏日历 / 定时任务

第一版只需要读取：

- 名称
- 时间
- 启用状态

## 当前缺失的真相源

### 工作状况

当前不存在：

- `/Users/admin/msgcode-workspaces/family/.msgcode/status.log`

所以右栏“工作状况”第一版必须降级为：

- 仅展示当前线程最近一条可确认状态
- 或直接空态

不能由 UI 自己拼一段“聪明摘要”。

### 人物入口

当前已经有稳定人物真相源：

- `.msgcode/character-identity/*.csv`
- `.msgcode/people-pending.json`

并且已有只读出口：

- `msgcode appliance people --workspace <...> --json`

所以主界面第一版应直接读：

- `counts.people`

并明确：

- 不把 `pending` 算进人数
- 底层文件不存在时人数就是 `0`
- 不让 UI 自己扫 CSV

### 邻居未读

当前不存在稳定的：

- mailbox
- health

文件合同。

所以主界面第一版：

- 不显示邻居未读 badge

## Main UI 映射结论

### 左栏

直接读：

- `threads/*.md`

展示：

- 场景工作区名
- 最近线程
- 当前线程

### 中间正文

直接读：

- 当前选中的 `threads/<name>.md`

展示：

- 历史 turn
- 当前输入框

输入框仍然只是投递，不先自行上屏。

### 右栏

优先级：

1. `status.log` 存在则读最近几条
2. 没有则空态或退回当前线程可确认状态
3. 日历直接读 `schedules/*.json`

## 结论口径

`0293` 已经有足够证据进入评审。

真正要补的不是 UI 树，而是：

- `0294` 的 `status.log` 合同
- Neighbor mailbox/health 真相源
