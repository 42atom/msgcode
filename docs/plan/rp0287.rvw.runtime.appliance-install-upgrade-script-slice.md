# RP0287：Appliance Install And Upgrade Script Slice

## 结论

`0265` 现在已经不只是“入口脱离 `tsx` + first-run 骨架”。

这次又补齐了 appliance bootstrap 的三段脚本骨架，并让 `install/upgrade` 开始读取随包 manifest：

1. install
2. first-run-init
3. upgrade

它们仍然是薄骨架，但已经能在本地临时 bundle 上真实跑通，不再只是纸面接口。

## 已落地实现

## 1. install-appliance

新增：

- [install-appliance.sh](/Users/admin/GitProjects/msgcode/bootstrap/install-appliance.sh)

当前合同：

1. 接收：
   - `--bundle-root <path>`
   - `--install-root <path>`
2. 只认已经组装好的 `bundle-root/runtime`
3. 把 `runtime/` 复制到安装目录
4. 写入 `install-root/bin/msgcode` launcher
5. 如存在 `bundle-root/appliance.manifest`，优先按 manifest 读取路径

launcher 当前只做三件事：

1. 解析 `runtime/`
2. 如存在 `runtime/node/bin`，把它前置进 `PATH`
3. 导出 `MSGCODE_RUNTIME_ROOT` 并执行 `runtime/bin/msgcode`

## 2. first-run-init

新增：

- [first-run-init.sh](/Users/admin/GitProjects/msgcode/bootstrap/first-run-init.sh)

当前合同：

1. 接收：
   - `--install-root <path>`
   - `--workspace <labelOrPath>`（可选）
2. 不自己造初始化逻辑
3. 只负责调用安装后的 launcher：
   - `msgcode init`
   - 或 `msgcode init --workspace ...`

结论：

- shell 只是桥
- 真正的 first-run 逻辑仍在 `msgcode init`

## 3. upgrade-appliance

新增：

- [upgrade-appliance.sh](/Users/admin/GitProjects/msgcode/bootstrap/upgrade-appliance.sh)

当前合同：

1. 接收：
   - `--bundle-root <path>`
   - `--install-root <path>`
2. 只替换安装目录下的 `runtime/`
3. 重写 launcher
4. 不触碰 workspace 真相源
5. 如存在 `bundle-root/appliance.manifest`，优先按 manifest 读取路径

结论：

- 升级当前只覆盖运行物
- 不覆盖 `ORG.md`、`SOUL.md`、`memory/`、`issues/`、`AIDOCS/`

## 验证证据

我跑过：

1. `PATH="$HOME/.bun/bin:$PATH" bun test test/p7-tk0265-appliance-bootstrap-scripts.test.ts`
2. `./node_modules/.bin/tsc --noEmit`
3. `npm run docs:check`

其中新测试 [p7-tk0265-appliance-bootstrap-scripts.test.ts](/Users/admin/GitProjects/msgcode/test/p7-tk0265-appliance-bootstrap-scripts.test.ts) 已锁住三件事：

1. `install-appliance.sh` 能复制 runtime 并写 launcher
2. `first-run-init.sh` 会经由 launcher 调 `msgcode init --workspace`
3. `upgrade-appliance.sh` 会替换 runtime，升级后 launcher 实际执行的是新 runtime
4. 自定义 `runtime/launcher` 路径能通过 `appliance.manifest` 生效

## 现在还没做的

这次仍然没有做：

1. 真正的 bundle assembly
2. DMG/PKG/install builder
3. 校验签名/版本元数据
4. doctor/rollback
5. pack 安装器

所以当前判断仍然是：

- appliance bootstrap 已有最小可运行骨架
- 但还不是完整交付器

2. 在线更新
