# RP0314：appliance doctor 读面实现切片

## 结论

这次没有新建 doctor 模块。

只做了一件事：

- 给 Settings 页补一个 `appliance doctor` 只读出口

并且继续复用现有：

- `runAllProbes()`

## 这次怎么收

### 1. runtime 事实只保留一份

`hall` 原本已经能读到：

- `appVersion`
- `configPath`
- `logPath`
- `summary`
- `categories`

这次只是把这份 runtime 事实抽成一个薄 surface，让 Settings 页单独消费。

### 2. doctor 只读，不控流程

`appliance doctor` 不负责：

- 触发额外修复
- 生成新 probe
- 改写 probe 状态

它只负责把当前 probe 事实端出来。

### 3. 不让 hall 和 settings 分叉

为避免两个出口各自拼 runtime 字段，这次把 runtime surface 统一到一个 helper：

- `readRuntimeSurface()`

因此：

- `hall.runtime`
- `doctor.runtime`

来自同一份组装逻辑。

## 不做

1. 不新增 `doctor.json`
2. 不新增 probe cache
3. 不把设置页动作也塞进 doctor 出口
4. 不在这里顺手做日志 tail / evidence 浏览

## 当前边界

这条切片只覆盖：

- summary
- categories
- appVersion / configPath / logPath

设置页里更细的“打开日志 / 打开 evidence / 运行 doctor”动作，以后如果真要做，再单开。
