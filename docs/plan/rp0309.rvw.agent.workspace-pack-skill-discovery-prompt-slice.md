# RP0309：workspace pack skill discovery prompt slice

## 结论

这轮不引入工作区 skill 索引。

只做一条更薄的主链：

- 安装器把 pack skill 路径写进 `packs.json`
- 主脑在构建 system prompt 时，按需读取当前工作区 `packs.json`
- 若发现合法且真实存在的 `SKILL.md`，就把路径提示给模型

一句话：

- pack skill 只进入 prompt 提示面
- 不进入全局技能中心

## 新增主链

### 真相源

- `<workspace>/.msgcode/packs.json`

### 读取器

- `src/runtime/workspace-pack-skills.ts`

职责只有：

- 读取当前工作区 `packs.json`
- 提取合法 pack 项里的 `skills`
- 只保留位于当前工作区内、且真实存在的 `SKILL.md`

### 注入点

- `src/agent-backend/tool-loop.ts`

只有在当前真实暴露了：

- `read_file`
- `bash`

时，才会把这段提示拼进 `[技能系统]`。

## 刻意不做

1. 不新增 `<workspace>/.msgcode/skills/index.json`
2. 不把 pack skill 同步到 `~/.config/msgcode/skills/index.json`
3. 不让主脑默认扫描 pack 目录
4. 不让 UI 额外维护一份 pack skill 状态

## Prompt 口径

主脑看到的是：

- 当前工作区已安装能力包
- 对应 `SKILL.md` 路径
- 明确提示：这些 skill 只属于当前工作区 pack，不同步到全局索引；只有任务确实涉及该包能力时才读取

## 验证

- `test/p7-tk0309-workspace-pack-skill-discovery.test.ts`

锁住两件事：

1. 已安装 pack skill 会进 system prompt
2. 缺少 `packs.json` 或缺少 skill 文件时，不会注入这段提示
