# RP0311：neighbor baseline roundtable review

Decision:
- Recommendation: go
- Confidence: medium
- Scope: 0291 可以收口为通过基线，不再保留为活跃设计入口

Why:
- `0302` 已经把邻居页第一阶段真相源钉成三份文件：`config.json`、`neighbors.json`、`mailbox.jsonl`
- 原型已经对齐到“节点头 + 节点列表 + 邮箱记录流”，没有再把它做成社交平台
- `0291` 的核心边界仍成立：可选模块、默认不开、第一阶段只做发现/联系人三态/异步邮箱/轻量可达探测

Risks:
1. 若后续把 mailbox 继续做肥，容易重新滑回“第二聊天系统”
2. 若把 `health` 解释成业务活跃度，会污染 UI 和用户心智
3. 若把节点内多角色关系抬进邻居页，会把当前单一主链重新做碎

Guardrails:
- 前置条件: 继续只读三份文件，不新增网络发现细节和写入编排
- 监控指标: UI 只读 surface 是否仍能用现有三份文件解释清楚
- 回滚条件: 一旦邻居页开始要求线程 owner、多人角色或远程执行权，就退回 0291 当前边界

Next Actions:
1. 0291 推到 `pss`
2. 邻居后续只围绕 `0302` 的已定义 surface 做实现
3. 网络发现、写入器、社交编排继续后移，不进入当前主线
