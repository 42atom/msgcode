# RP0333：people pending 键合同圆桌审阅

Decision:
- Recommendation: go
- Confidence: high
- Scope: 直接把 pending 读面收成 `channel + chatId + senderId`，旧键条目只 warning，不做兼容

Why:
- CSV 已经把唯一键钉死在 `channel + chatId + senderId`
- 继续接受旧键，只会把后续过滤和认领逻辑拖回猜测式匹配
- 当前仓库里还没有正式 pending writer，趁早收口成本最低

Risks:
1. 旧测试数据或手工 pending 文件会立刻掉到 warning
2. 如果导入侧后续仍吐旧键，会让右侧 pending 变空

Guardrails:
- UI 不得因为 pending 为空就自作聪明补猜测结果
- 导入侧如果还未升级，允许 warning，但不允许 runtime 宽容兼容
- 后续过滤实现前，继续保持“只读、不隐藏”

Next Actions:
1. 升级 runtime 读面
2. 升级最小测试样本
3. 过滤逻辑另起下一刀，不混进本轮
