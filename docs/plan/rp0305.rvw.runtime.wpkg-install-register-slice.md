# RP0305：wpkg 安装注册切片

## 结论

这轮只补了一条最小主链：

- `wpkg`
- 解压
- 校验 `manifest.json`
- 落到 `<workspace>/.msgcode/packs/user/<pack-id>/`
- 写 `packs.json`
- 写 `sites.json`

没有做：

- 升级
- 卸载
- 签名
- 市场
- skill 自动发现

## 本轮实现

新增：

- `src/runtime/workspace-wpkg-install.ts`
- `test/p7-tk0305-wpkg-install-register.test.ts`

扩展：

- `src/cli/appliance.ts`
  - `msgcode appliance install-pack --workspace <workspace> --file <pack.wpkg> --json`

## 安装主链

安装时只做这些动作：

1. 用系统 `ditto -x -k` 解压 `.wpkg`
2. 读取并校验 `manifest.json`
3. 拒绝坏包、重名包、重名站点
4. 拷贝到 `.msgcode/packs/user/<pack-id>/`
5. 原子写回 `packs.json`
6. 原子写回 `sites.json`

## 关键收刀

这轮刻意收了四刀：

1. 不做包管理器  
   只有安装注册，没有升级/卸载/市场。

2. 不静默覆盖  
   已存在同名 `pack` 或 `site` 时直接报错。

3. 不注册死路径  
   `sidecar` 站点和 `skills` 都必须指向包内真实存在的文件。

4. 不假装 skill 已可用  
   只把 `skills` 路径写进注册表留位；CLI 明确返回 warning，说明当前 runtime 还没接自动发现。

## 落盘结果

最小落盘结果现在是：

- `<workspace>/.msgcode/packs/user/<pack-id>/`
- `<workspace>/.msgcode/packs.json`
- `<workspace>/.msgcode/sites.json`

其中：

- `packs.json`
  - 记录 `id / name / version / enabled`
  - 并保留 `author / sourcePath / skills / requires / commercial / licenseType` 留位

- `sites.json`
  - 记录主界面真正需要的站点入口
  - sidecar 入口统一写成工作区相对路径
  - 例如：`.msgcode/packs/user/company-finance/web/index.html`

## 验证

本轮验证通过：

3. `npm run docs:check`

## 留后

这轮故意没做：

1. `wpkg` 升级
2. `wpkg` 卸载
3. pack skill 自动发现
4. 授权 / 支付 / 市场

这些仍然应该在后续独立切片里处理，不借这刀偷长更多层。
