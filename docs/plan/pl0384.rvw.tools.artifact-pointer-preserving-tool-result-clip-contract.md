# PL0384：Artifact Pointer Preserving Tool Result Clip Contract

## 结论

这条线先做一件事：

让 `tool-loop` 在裁剪工具结果时，永远保留继续主链所需的 artifact 指针。

## 为什么现有文件不能直接承载

- `pl0128` 收的是 preview footer，不是最后一跳裁剪
- `pl0131` 收的是 browser text artifact 生成，不是 artifact 指针保留
- 这次问题发生在 `tool-loop` 最后一跳，所以要单独计划

## 实现顺序

### Step 1

盘点三类当前最真实的 artifact 指针：

- `bash.fullOutputPath`
- `browser.textPath`
- `ghost.artifacts[]`

### Step 2

给 `tool-loop` 定“硬保留字段”规则。

目标：

- 先保路径和状态
- 再裁正文

### Step 3

第一刀只覆盖 `bash/browser/ghost`。

目标：

- 不扩到 `read_file`
- 不扩到全工具平台

### Step 4

补最小验证。

目标：

- 大输出 bash 仍能看到 `fullOutputPath`
- browser 大文本仍能看到 `textPath`
- ghost 大结果仍能看到 artifact 路径

## 不做什么

- 不重写所有 preview builder
- 不做新的 preview controller
- 不扩到 `read_file`
- 不扩到全工具家族

## 一句话

继续入口比正文摘要更重要；先保指针，再裁内容。
