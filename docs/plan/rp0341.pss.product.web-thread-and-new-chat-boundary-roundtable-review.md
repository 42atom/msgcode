# RP0341：web thread 与新建聊天边界圆桌审议

## 共识

圆桌一致认为：

1. 现在感觉 thread 碎，先不是 thread 哲学错了
2. 先是 archive 缺位
3. 其次是 web 端没有稳定 `chatId`

## 采纳

### 1. 不推翻 thread

继续坚持：

- 文件为真相
- UI 只读
- runtime 写 thread

### 2. 主界面统一只讲 thread

用户不再看到：

- session
- channel 会话

只看到：

- 工作区
- 线程

### 3. web 必须有稳定 `chatId`

否则 web thread 一定会碎。

### 4. 新建聊天不能直接造空 thread

它只能切到一个新的 web `chatId` 上下文。

真正建 thread，仍要等第一条消息落盘。

### 5. 标题不能由 UI 发明

标题必须由 runtime 写进 thread front matter。

## 不采纳

### 1. 不把整个浏览器只绑一个全局 `chatId`

这会让多个 web thread 失去意义。

### 2. 不把 archive 问题伪装成 thread 问题

工作区噪音优先靠 archive 清掉。

### 3. 不让 UI 拥有 thread 创建权

否则很快就会长出第二状态中心。

## 结论

`0341` 方向通过。

后续只需补三刀：

- web `chatId`
- thread front matter
- 新建聊天边界
