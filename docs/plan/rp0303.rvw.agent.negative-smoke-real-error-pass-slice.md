# RP0303：负向 Smoke 真实错误通过口径切片

结论：

- `0266` 先不做新控制层
- 先用两条负向样本把口径钉死
- 顺手把 `read_file` 的物理错误码保真

这刀只收三件事：

1. `read_file` 缺文件时，`ENOENT` 不再被抹成 `TOOL_EXEC_FAILED`
2. `bash` 缺命令时，模型上下文里能看到 `exitCode=127` 与 `command not found`
3. live smoke 文档默认通过条件改成“真实错误已进入证据面与模型上下文”

验收样本：

- 缺文件：`.msgcode/missing.txt` -> `ENOENT`
- 缺命令：`nonexistent-msgcode-command` -> `127 + command not found`

不做：

- 不做新的入口守卫
- 不做系统催报
- 不做新的 negative smoke 编排层

边界补充：

- 这里的“物理边界”只收口到“工具启动前物理不可达”
- 工具已经真正启动后的底层失败，默认仍回真实错误，不按 fail-closed 处理
