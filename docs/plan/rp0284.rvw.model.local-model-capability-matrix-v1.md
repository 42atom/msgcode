# RP0284：Local Model Capability Matrix v1

## 结论

当前仓库的“本地模型”其实是两种东西：

1. 本地 HTTP 后端
   - `LM Studio`
   - `OMLX`

2. 本地命令行媒体模型
   - `mlx-whisper`

这两者不能再混着讲。

## 当前矩阵

| 能力 | 当前主链 | 证据 | 平台锁 | v1 口径 |
| --- | --- | --- | --- | --- |
| 文本对话 | 云/API 或 `local-openai` | [config.ts](/Users/admin/GitProjects/msgcode/src/agent-backend/config.ts), [registry.ts](/Users/admin/GitProjects/msgcode/src/local-backend/registry.ts) | 无硬锁 | core 可直接用云；本地模型仅作可选 pack |
| 视觉理解 | 本地 HTTP `vision` | [vision.ts](/Users/admin/GitProjects/msgcode/src/runners/vision.ts), [registry.ts](/Users/admin/GitProjects/msgcode/src/local-backend/registry.ts) | 无明显 mac-only 硬锁，但依赖后端自己暴露视觉 API | 归入 Local Model Pack |
| Embedding | `/v1/embeddings` | [embedding.ts](/Users/admin/GitProjects/msgcode/src/memory/embedding.ts) | 无明显 mac-only 硬锁，但依赖 OpenAI 兼容接口 | 归入 Local Model Pack |
| ASR | `mlx-whisper` CLI | [pipeline.ts](/Users/admin/GitProjects/msgcode/src/media/pipeline.ts), [run.ts](/Users/admin/GitProjects/msgcode/src/cli/run.ts), [model-paths.ts](/Users/admin/GitProjects/msgcode/src/media/model-paths.ts) | 明显 Apple Silicon / macOS 倾向 | 单列为 Apple Media Pack，不进 cross-platform 承诺 |

## 关键观察

## 1. `local-openai` 已经是正确边界

[config.ts](/Users/admin/GitProjects/msgcode/src/agent-backend/config.ts) 已经把：

- `lmstudio`
- `omlx`
- `agent-backend`
- `local-openai`

统一到了 `local-openai` 这层语义。

这说明当前真正的模型边界不是：

- “msgcode 是否支持 LM Studio”

而是：

- “msgcode 是否支持一个本地 OpenAI 兼容后端”

## 2. `OMLX` 与 `LM Studio` 不是同一种强度

[registry.ts](/Users/admin/GitProjects/msgcode/src/local-backend/registry.ts) 当前已经写明：

- `lmstudio`
  - `nativeApiEnabled: true`
  - `supportsModelLifecycle: true`
  - `modelsListPath: /api/v1/models`

- `omlx`
  - `nativeApiEnabled: false`
  - `supportsModelLifecycle: false`
  - `modelsListPath: /v1/models`
  - `modelsStatusPath: /v1/models/status`

所以 v1 不应该把它们宣传成完全等价。

## 3. 真正锁平台的是 ASR，不是整个本地模型

[pipeline.ts](/Users/admin/GitProjects/msgcode/src/media/pipeline.ts) 里，音频预处理直接检查 `mlx-whisper`。

[model-paths.ts](/Users/admin/GitProjects/msgcode/src/media/model-paths.ts) 默认模型目录也是：

- `~/Models/whisper-large-v3-mlx`

这才是当前最明确的 Apple 锁定点。

## v1 建议

### A. Core Appliance

不承诺：

1. 默认本地模型
2. 默认本地视觉
3. 默认本地 embedding
4. 默认本地 ASR

只承诺：

1. appliance 能启动
2. agent 能跑
3. 可以接云/API 模型

### B. Local Model Pack

承接：

1. `LM Studio`
2. `OMLX`
3. 其他本地 OpenAI 兼容 HTTP 后端

适用能力：

1. 文本
2. 视觉
3. embedding

### C. Apple Media Pack

承接：

1. `mlx-whisper`
2. 未来同类 Apple Silicon 媒体模型

特点：

1. mac-only
2. optional
3. 不在 cross-platform 基线里做承诺

## 不该现在承诺的东西

1. “本地模型完整跨平台”
2. “Windows/Linux 也能无缝用 MLX”
3. “appliance 默认就自带本地模型”
4. “`LM Studio` 与 `OMLX` 一键自动切换”
5. “本地 ASR 已有跨平台替代实现”

2. 再让 `0263` 的 App/UI 方案服从这条边界
3. 最后让 `0265` 再决定 pack 是否随包交付
