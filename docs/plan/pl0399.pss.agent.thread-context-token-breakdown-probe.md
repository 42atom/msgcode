# PL0399：Thread Context Token Breakdown Probe

## 结论

这条线只做观察，不做控制。

第一刀输出固定五段：

- `system_prompt`
- `tool_definitions`
- `project_context`
- `history_dialogue`
- `current_input`

每段至少给：

- `tokens`
- `ratio`
- `status: ok | unavailable`

## 数据来源

- `system_prompt`
  - `src/agent-backend/prompt.ts`
- `project_context`
  - `src/runtime/context-policy.ts`
- `history_dialogue`
  - 当前对话窗口/历史消息输入面
- `current_input`
  - 当前用户新问题
- `tokens`
  - `src/budget.ts` 的现有 token estimator

第一刀对 `tool_definitions` 的要求是：

- 能取到就算
- 当前若无单一稳定落点，则标 `unavailable`
- 不为了凑齐这段重造工具拼接器

## 输出格式

### text

```text
[system_prompt 10234 tokens | 18%]
[tool_definitions unavailable]
[project_context 42110 tokens | 72%]
[history_dialogue 5210 tokens | 9%]
[current_input 138 tokens | <1%]
```

### json

```json
{
  "totalTokens": 57692,
  "segments": [
    { "key": "system_prompt", "tokens": 10234, "ratio": 0.177, "status": "ok" },
    { "key": "tool_definitions", "status": "unavailable" },
    { "key": "project_context", "tokens": 42110, "ratio": 0.730, "status": "ok" },
    { "key": "history_dialogue", "tokens": 5210, "ratio": 0.090, "status": "ok" },
    { "key": "current_input", "tokens": 138, "ratio": 0.002, "status": "ok" }
  ]
}
```

## 接线顺序

1. 在 `src/probe/` 下新增最薄 thread/context probe
2. 先接 `system_prompt / project_context / history_dialogue / current_input`
3. `tool_definitions` 若当前无稳定单一来源，先显式 `unavailable`
4. 最后接 text/json formatter

## Guardrails

- 不重造 prompt 拼接器
- 不引入自动 compact 决策
- 不做 token 预算控制器
- 不给“建议值”，先只给事实
- 缺数据直接 `unavailable`，不做模糊估算
