# RP0283：Appliance Bootstrap Route v1

## 结论

当前仓库已经有两条 bootstrap 事实：

1. 开发者 bootstrap
2. appliance bootstrap（还不存在，只在产品文档里）

`0265` 的第一件事不是写安装器，而是承认：

- 现有 `bootstrap/*` 不能再承担 appliance 正式主链

## 当前链路

### 开发者 bootstrap

入口：

1. [bootstrap/bootstrap.sh](/Users/admin/GitProjects/msgcode/bootstrap/bootstrap.sh)
2. [bootstrap/bootstrap-agent-pack.sh](/Users/admin/GitProjects/msgcode/bootstrap/bootstrap-agent-pack.sh)
3. [bin/msgcode](/Users/admin/GitProjects/msgcode/bin/msgcode)

特点：

1. 依赖 Homebrew
2. 依赖 `npm install`
3. 依赖 `tsx`
4. 依赖源码目录

结论：

- 适合开发
- 不适合交付

## 目标链路

### Appliance bootstrap

入口应改成三段：

1. 安装器入口
2. 首次启动初始化入口
3. 升级入口

特点：

1. 不跑 `brew`
2. 不跑 `npm install`
3. 不依赖源码目录
4. 不依赖 `tsx`

## 逐项替换

### 1. `bootstrap/bootstrap.sh`

现状：

- 安装托管 Bash
- 直接要求 Homebrew

处理：

- 保留
- 降级为开发者 bootstrap

不再承担：

- appliance 安装主链

### 2. `bootstrap/bootstrap-agent-pack.sh`

现状：

- 安装 `tmux/uv/bun/rg/fd/...`

处理：

- 保留
- 降级为开发者增强工具脚本

不再承担：

- appliance 默认能力安装

### 3. `bin/msgcode`

现状：

- 本地 `tsx` 优先
- 回退 `npx tsx`

处理：

- 必须被 appliance 入口替代

这条是最硬的替换点。

## 建议的 appliance 主链

### A. install

职责：

1. 拷贝 runtime
2. 拷贝 Node
3. 拷贝模板
4. 写启动入口

### B. first-run-init

职责：

1. 创建 `~/.config/msgcode/*`
2. 创建 `<workspace>/.msgcode/*`
3. 写 `ORG.md`
4. 写默认 `SOUL.md`

### C. upgrade

职责：

1. 替换 runtime 目录
2. 保留用户真相源
3. 不覆盖 workspace 身份和记忆

## 哪些内容不进入 bootstrap v1

1. Feishu pack 自动安装
2. Browser pack 自动安装
3. Desktop pack 自动安装
4. Tmux coding pack 自动安装

原因：

- 它们都不属于第一版 Core Appliance 必需物

## v1 验收

只要满足这 4 条，就算 bootstrap route v1 成立：

1. 已明确开发者 bootstrap 与 appliance bootstrap 是两条线
2. 已明确 `bin/msgcode` 现状不能继续作为 appliance 正式入口
3. 已明确 install / first-run-init / upgrade 三段结构
4. 已明确哪些 pack 不进入 v1 默认安装
