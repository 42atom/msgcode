# Plan: Tasks Directory Archive Physical Closure

Issue: 0195

## Problem

`docs/tasks/` 已经在协议上退役，但历史任务单仍物理留在原目录，造成三层误导：

1. 新同学会以为这里还是现役任务目录
2. 老命名文件继续挂在现役路径下，视觉上像“迁移没做完”
3. 目录职责和 README 口径不一致

## Decision

这次不再改名，不再重写正文，只做目录物理收口：

1. 历史任务单整体迁入 `docs/archive/tasks/`
2. `docs/tasks/` 只保留 `README.md`
3. 必要入口文档同步说明“现役真相源在 issues + docs/plan”

## Workstreams

### 1. 建立归档承接目录

- 新增 `docs/archive/tasks/`
- 作为历史 `p5-* / p6-*` 任务单的统一落点

### 2. 批量迁移

- 对 `docs/tasks/*.md`（除 `README.md`）执行批量 `git mv`
- 不改单文件命名，避免把低价值历史整理扩大成二次迁移

### 3. 回链与入口

- 更新：
  - `docs/tasks/README.md`
  - `docs/README.md`
  - 必要的 archive 入口 README
- 保证读者一眼能知道：
  - 现役任务去 `issues/`
  - 历史派单包去 `docs/archive/tasks/`

### 4. 验收

- `docs/tasks/` 只剩 `README.md`
- `docs/archive/tasks/` 成为历史任务单唯一目录
- `npm run docs:check` 通过

## Risks

1. 历史正文里的相对路径可能失效
   - 处理：只修关键入口与高价值回链，不追求 archive 全量正文重写

2. 批量移动后目录引用继续误导
   - 处理：同步改 `README` 与目录说明

## Acceptance

- 目录职责与物理路径一致
- 新 agent 不会再把 `docs/tasks/` 误认成现役目录
- `docs:check` 保持通过
