# PL0377：Desktop Main Window Readonly Thread Surface Mainline

## 结论

执行顺序只分两步：

1. 左栏接 `workspace-tree`
2. 中间正文和右栏接 `thread`

UI 只保留 3 个瞬时状态：

- `selectedWorkspace`
- `selectedThreadId`
- `loading-error`

正式 UI 的落点在 `src`，`ui-protype` 只保留原型参考，不再继续堆现役实现。

## 最小顺序

### 1. 左栏

只消费：

- `msgcode appliance workspace-tree --json`

只负责：

- 展示工作区树
- 记录当前选中的 `selectedWorkspace`
- 记录当前选中的 `selectedThreadId`

不在左栏偷做：

- archive 入口
- new chat 入口
- settings 入口

### 2. 正文和右栏

只消费：

- `msgcode appliance thread --workspace <...> --thread-id <...> --json`

只负责：

- 当前线程正文
- 当前线程附带的右栏信息
- `loading-error` 呈现

不在这里偷做：

- composer
- send
- optimistic message
- 本地派生缓存

## 边界

`main.html` 只可作为布局参考，不再继承它的多余入口。

`src` 里要承接正式 UI 的第一刀，`ui-protype` 不再作为现役实现面。

`rf0001` 只作为阶段性约束参考，不替代这张执行计划。

## 验收

1. 实现时只能接 `workspace-tree` 和 `thread` 两个正式读面。
2. 主窗口只能保留 3 个瞬时状态。
3. 发消息、新建聊天、archive、settings、美学重做、Electron 大壳都不进入本刀。

## 一句话

这刀把主窗口收成一个只读壳：

- 左边选
- 中间看
- 右边读
