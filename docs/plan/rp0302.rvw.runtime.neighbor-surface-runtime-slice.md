# RP0302：Neighbor 只读主链运行时切片

## 结论

这条主链已经成立：

- `config.json`
- `neighbors.json`
- `mailbox.jsonl`
- `appliance neighbor`

邻居页第一阶段现在可以只靠文件真相源起数据，不需要先做发现协议或聊天系统。

## 当前读取主链

新增：

- `/Users/admin/GitProjects/msgcode/src/runtime/workspace-neighbor.ts`

新增壳出口：

- `msgcode appliance neighbor --workspace <...> --json`

位置：

- `/Users/admin/GitProjects/msgcode/src/cli/appliance.ts`

## 当前字段

### config.json

只认：

- `enabled`
- `nodeId`
- `publicIdentity`

### neighbors.json

只认：

- `nodeId`
- `displayName`
- `state`
- `lastMessageAt`
- `lastProbeAt`
- `lastProbeOk`
- `latencyMs`

### mailbox.jsonl

每行只认：

- `at`
- `nodeId`
- `direction`
- `type`
- `summary`
- `unread`

## 当前输出 surface

顶层保留：

- `enabled`
- `self`
- `summary`
- `neighbors`
- `mailbox`

其中 `summary` 只保留：

- `unreadCount`
- `lastMessageAt`
- `lastProbeAt`
- `reachableCount`

## 本轮圆桌吸收

吸收了三条：

1. `neighbors.json` 顶层 schema 漂移必须显式 warning，不能静默空态
2. 文件合同只认一套 camelCase 字段，不留隐藏 alias
3. surface 再瘦一刀，不给 UI 额外喂冗余计数和调试路径

## 刻意不做

不做：

- 发现协议
- mailbox 写入器
- 邻居线程系统
- artifact 富对象
- 审计 owner
- `counts`
- mailbox 每条记录的 `sourcePath`
- mailbox 每条记录的 `displayName`
