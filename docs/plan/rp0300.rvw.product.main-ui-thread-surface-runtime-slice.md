# RP0300：Main UI 线程读面实现切片

## 结论

`0293` 第一版已经落成一条薄读面：

- 线程列表直接读 `threads/*.md`
- 当前线程仍直接读当前那一份 markdown
- 工作状况直接读 `status.log`
- 日历 / 定时任务直接读 `schedules/*.json`

通过：

- `msgcode appliance threads --workspace <...> --json`

把主界面需要的最小数据一次性只读暴露出来。

## 本轮实现

新增：

- `src/runtime/workspace-thread-surface.ts`
- `test/p7-tk0293-appliance-threads-contract.test.ts`

扩展：

- `src/cli/appliance.ts`
  - 新增 `appliance threads`

## 读取主链

### 线程列表

直接读：

- `<workspace>/.msgcode/threads/*.md`

但只取最薄字段：

- `threadId`
- `title`
- `source`
- `lastTurnAt`

不再把：

- `chatId`
- `filePath`
- `createdAt`
- `turnCount`
- `isCurrent`

这些调试或 UI 派生字段抬进主界面合同。

### 当前线程

当前线程选择规则已经收硬：

1. 先认 `runtime.current_chat_guid`
2. 再认 `runtime.current_chat_id`
3. 若配置里明确给了 current chat，但匹配不到任何线程，返回 `currentThread: null`
4. 只有在配置里根本没给 current chat 时，才退到最近线程

这避免了把“最近线程”误装成“当前线程”。

### 工作状况

直接读：

- `<workspace>/.msgcode/status.log`

当前输出两路：

- `currentThreadEntries`
- `recentEntries`

其中当前线程过滤仍保持薄：

- 只按当前线程标题或来源标签匹配
- 不为此新增 thread id 写入规则

### 日历 / 定时任务

直接读：

- `<workspace>/.msgcode/schedules/*.json`

并继续保持最小降级：

- 非法 JSON -> warning 跳过
- 缺 `cron` / `message` -> warning 跳过

## 圆桌吸收

本轮圆桌最有价值的意见有三条，已经吸收进实现：

1. 不要为线程列表先把所有正文都读出来  
现在先读线程摘要；只有当前线程才解析完整消息。

2. 不要把 UI 派生状态写进合同  
`isCurrent` 已删，只留 `currentThreadId` 一处。

3. 当前线程选择要更硬  
stale config 不再偷偷退到最近线程。

## 明确不做

这轮继续不做：

1. `thread-index.json`
2. UI 专用缓存层
3. 自动跨线程合并原文
4. 为当前线程状态新增独立写入协议
5. 邻居 mailbox / people / wpkg 的联动读取

## 结果

`0293` 现在已经有一条可站住的最小主链：

- 现有工作区文件就是主界面真相源
- CLI 只做 envelope 包装
- UI 后面只读这份 surface，不再发明第二状态层

这符合当前原则：

- 克制
- Unix 哲学
- Everything is a file
