# RP0286：Appliance Bootstrap Entry And First-Run Slice

## 结论

`0265` 目前已经落地两件关键事实：

1. 正式入口不再被源码态 `tsx` 绑死
2. `msgcode init --workspace` 已能生成 appliance 需要的最小 first-run 骨架

这还不是完整安装器，但已经把 bootstrap 主链最危险的两处断点收掉了。

## 已落地实现

## 1. runtime 入口解析

新增：

- [runtime-entry.ts](/Users/admin/GitProjects/msgcode/src/runtime/runtime-entry.ts)

当前合同：

1. 显式提供 `MSGCODE_RUNTIME_ROOT / MSGCODE_CLI_ENTRY / MSGCODE_DAEMON_ENTRY` 时
   - 优先走 compiled / bundled JS

2. 开发仓库默认
   - 回退到源码 + `tsx`

这条合同已经接进：

- [bin/msgcode](/Users/admin/GitProjects/msgcode/bin/msgcode)
- [cli.ts](/Users/admin/GitProjects/msgcode/src/cli.ts)
- [launchd.ts](/Users/admin/GitProjects/msgcode/src/runtime/launchd.ts)

结论：

- appliance 可以显式指向 bundled runtime
- 开发仓库不会被旧 `dist/` 污染

## 2. first-run init 骨架

新增：

- [first-run-init.ts](/Users/admin/GitProjects/msgcode/src/runtime/first-run-init.ts)

并接进：

- [cli.ts](/Users/admin/GitProjects/msgcode/src/cli.ts)

现在：

- `msgcode init`
  - 继续负责全局 `.env + skills`

- `msgcode init --workspace <path>`
  - 额外生成 workspace 骨架

当前生成内容：

1. `~/.config/msgcode/.env`
2. `~/.config/msgcode/souls/default/SOUL.md`
3. `~/.config/msgcode/souls/active.json`
4. `<workspace>/.msgcode/SOUL.md`
5. `<workspace>/.msgcode/ORG.md`
6. `<workspace>/.msgcode/config.json`
7. `<workspace>/memory/`
8. `<workspace>/AIDOCS/reports/`

其中 `ORG.md` 第一版只保留：

1. 名称
2. 交税地
3. 统一社会信用代码

## 验证证据

我跑过：

1. `bun test test/p7-tk0265-first-run-init.test.ts test/p7-tk0265-runtime-entry-resolution.test.ts test/runtime.launchd.test.ts`
2. `./node_modules/.bin/tsc --noEmit`
3. `npm run docs:check`
4. 黑盒：
   - `HOME=<tmp> WORKSPACE_ROOT=<tmp>/workspaces node bin/msgcode init --workspace acme/ops`

黑盒结果已确认：

1. CLI 参数真实生效
2. 全局配置骨架被生成
3. workspace 骨架被生成
4. `ORG.md`、`SOUL.md`、`config.json`、`memory/`、`AIDOCS/reports/` 全都存在

## 剩余缺口

`0265` 还没做这些：

1. 独立 install 入口
2. 独立 upgrade 入口
3. 真正的 appliance 安装目录布局
4. 随包 Node/runtime 组装
5. DMG/PKG/installer

所以当前判断应是：

- bootstrap 主链已从“纯文档”进入“最小可运行骨架”
- 但还没到“完整交付器”

## 推荐下一步

如果继续推进 `0265`，下一刀应该只做一件事：

1. 明确 install / upgrade 的正式脚本入口

不要现在就去碰：

1. Electron
2. DMG
3. pack 下载器
4. 复杂授权控制面
