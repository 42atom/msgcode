# PL0264：Local Model Backend And MLX Cross Platform Strategy

## 结论

`0264` 先定三层，不先碰实现：

1. Core Appliance
   - 默认不要求本地模型
   - 云/API 模型可以作为正式基线

2. Local Model Pack
   - 承接 `LM Studio / OMLX` 这类本地 OpenAI 兼容 HTTP 后端
   - 可选，不进 core

3. Apple Media Pack
   - 承接 `mlx-whisper` 这类 Apple Silicon / macOS 专属本地媒体模型
   - 明确是 mac-only，不说跨平台

一句话：

- 先把“本地模型”从 core appliance 里剥出来
- 再把“OpenAI 兼容本地后端”和“Apple MLX 媒体能力”分开

## 当前事实

当前仓库已经有一层很薄的本地后端注册表：

- [registry.ts](/Users/admin/GitProjects/msgcode/src/local-backend/registry.ts)
  - 只支持 `lmstudio` / `omlx`
  - 统一收口 `baseUrl / model / visionModel / embeddingModel`
  - 不做自动切换，不做恢复编排

Agent 文本主链已经把本地模型统一抽成：

- [config.ts](/Users/admin/GitProjects/msgcode/src/agent-backend/config.ts)
  - 外部 provider 统一表现为 `local-openai`
  - 下面再细分 `lmstudio` 或 `omlx`

但媒体能力还是分裂的：

- [embedding.ts](/Users/admin/GitProjects/msgcode/src/memory/embedding.ts)
  - 走 `/v1/embeddings`
  - 本质还是 OpenAI 兼容 HTTP

- [vision.ts](/Users/admin/GitProjects/msgcode/src/runners/vision.ts)
  - 也走本地 HTTP
  - `omlx` 还额外依赖 `/v1/models/status`

- [pipeline.ts](/Users/admin/GitProjects/msgcode/src/media/pipeline.ts)
  - 音频转写仍依赖 `mlx-whisper`

所以当前真正的硬锁不是“所有本地模型都锁死在 macOS”，而是：

- 文本/视觉/embedding 可以抽成 local HTTP 后端
- ASR 这条线目前明确锁在 Apple Silicon / macOS

## 目标

### Goal

1. 明确本地模型能力矩阵
2. 区分：
   - core baseline
   - local optional pack
   - mac-only pack
3. 给 `0262 / 0263 / 0265` 一个稳定前提

### Non-Goals

1. 不实现模型控制面
2. 不写本地模型进程守护
3. 不做自动 failover
4. 不把 `LM Studio` 或 `OMLX` 打包进 v1 core appliance

## 策略

## 1. Core Appliance

第一版 appliance 基线只承诺：

1. `msgcode` 可以正常跑
2. 可以接云/API 后端
3. 不因为本地模型缺失而无法交付

也就是说：

- “本地模型可用”是加分，不是 v1 核心交付门槛

## 2. Local Model Pack

这层承接：

1. `LM Studio`
2. `OMLX`
3. 其他未来仍然暴露 OpenAI 兼容 HTTP 的本地后端

这层只承诺：

1. `chat`
2. `vision`
3. `embedding`

前提是后端自己暴露稳定 HTTP API。

`msgcode` 在这层只负责：

1. 读配置
2. 打请求
3. 读错误

不负责：

1. 安装模型
2. 拉起模型进程
3. 自动重启模型服务
4. 跨不同本地后端做智能切换

## 3. Apple Media Pack

这层单独承接：

1. `mlx-whisper`
2. 未来同类 Apple Silicon 本地媒体模型

这层明确：

1. 是 mac-only
2. 是 Apple Silicon 优先
3. 不进入 cross-platform 承诺

原因很直接：

- [pipeline.ts](/Users/admin/GitProjects/msgcode/src/media/pipeline.ts) 和 [run.ts](/Users/admin/GitProjects/msgcode/src/cli/run.ts) 当前已经把 `mlx-whisper` 当成真实依赖
- [model-paths.ts](/Users/admin/GitProjects/msgcode/src/media/model-paths.ts) 也默认指向 `~/Models/whisper-large-v3-mlx`

## 推荐顺序

1. 先冻结能力矩阵
2. 再在 `0262` 里把 core appliance 口径对齐
3. 再到 `0263` 定 UI / shell
4. 最后才决定 `0265` 是否要支持随包带本地模型 pack

## Deferred

以下都明确延后：

1. 本地模型控制面
2. 模型生命周期管理器
3. 自动切云/切本地
4. 跨平台统一本地 ASR 方案
5. 把 `LM Studio / OMLX / MLX` 写成同一种东西

## 验收

1. 明确 core appliance 不默认承诺本地模型
2. 明确 `local-openai` 类本地 HTTP 后端和 `mlx-whisper` 不是同一层
3. 明确哪个是 cross-platform optional，哪个是 mac-only optional
