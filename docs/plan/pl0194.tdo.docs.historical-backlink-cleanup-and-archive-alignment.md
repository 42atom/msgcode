# Plan: Historical Backlink Cleanup And Archive Alignment

Issue: 0194

## Problem

正式协议已经收口，但历史 markdown 仍保留两类旧真相源：

1. `issues/000x-*.md`
2. `docs/plan/plan-YYMMDD-*.md`

它们主要分布在 `docs/tasks/`、`docs/archive/`、历史 `issues/`、`AIDOCS/`。这层噪声不阻断开发，但会让搜索结果和阅读路径偏离当前协议。

## Decision

按“现役优先、入口优先、历史次之”处理，不追求全量重写：

1. 活跃 issue / 主线任务包必须回到当前 `tk/pl` 真相源。
2. `docs/tasks/` 与 `docs/archive/` 的关键入口文档应消除明显误导链接。
3. `AIDOCS/` 只清理会误导当前入口的链接，不做大规模历史润色。

## Workstreams

### 1. 盘点与分层

- 用 `rg 'issues/000[0-9]|docs/plan/plan-' issues docs AIDOCS` 盘点残留。
- 分成三层：
  - P1: 活跃主线相关
  - P2: 高频归档索引
  - P3: 辅助材料与低频历史文本

### 2. 活跃主线回链修正

- 重点看：
  - 活跃 `issues/*.md`
  - 与 `0009/0027/0034/0035` 等主线关联的 `docs/tasks/*`
- 要求：
  - issue 回 issue：改到 `tkNNNN...`
  - issue 回 plan：改到 `plNNNN...`

### 3. 归档索引收口

- 清理 `docs/tasks/` 中仍保留旧 issue / plan 路径的任务包
- 清理 `docs/archive/` 中仍作为入口文档存在的旧路径
- 允许正文保留“历史描述”，但不允许入口回链继续误导

### 4. AIDOCS 降噪

- 只处理会影响当前检索和交接的文档
- 外部参考、历史笔记可保留内容，但链接口径尽量指向现协议

## Deliverables

1. 一轮旧回链命中清单
2. 一批已修正的高价值历史索引
3. `docs:check` 通过结果

## Acceptance

- 搜索 `issues/000[0-9]` 与 `docs/plan/plan-` 时，剩余结果主要是纯历史叙述，不再是关键入口或真相回链。
- 活跃主线相关文档不再引用旧命名真相源。
- `npm run docs:check` 通过。
