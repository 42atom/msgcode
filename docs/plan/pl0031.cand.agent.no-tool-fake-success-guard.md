1. 执行型请求没调工具却宣称已完成
2. 伪 `[TOOL_CALL]` / `<invoke>` 协议文本穿透到用户面

## 为什么取消

`pl0031` 原方案的三刀现在都不该再落：

1. 保留 `allowNoTool`

2. 新增执行意图检测

3. 对伪工具文本触发补跑
   - 当前更薄做法是直接清洗协议垃圾，不做 parser/补跑层
