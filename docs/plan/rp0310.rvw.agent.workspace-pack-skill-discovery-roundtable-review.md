# RP0310：workspace pack skill discovery roundtable review

## 结论

0309 这条路可以保留。

它足够薄：

- 真相源还是 `packs.json`
- 全局 skill 单源不变
- 主脑只多看到一段工作区 pack skill 提示

## 吸收的建议

### 1. 只提示真实存在的 `SKILL.md`

保留。

原因：

- 避免主脑看到脏路径
- 避免把坏包状态抬成 prompt 噪音

### 2. 不默认读取所有 pack skill

保留。

原因：

- 减少上下文污染
- 只在任务确实涉及对应 pack 能力时再读

### 3. 不同步到全局 skills 索引

保留。

原因：

- 继续守住 `~/.config/msgcode/skills/index.json` 单源
- 不为工作区 pack 再造第二技能中心

## 明确拒绝的建议

### 1. 自动把 pack skill 合并进全局 index

拒绝。

这会把工作区局部能力抬成全局能力，污染主链。

### 2. pack 安装后立刻预解析全部 SKILL.md

拒绝。

这会把安装器做成内容索引器，不符合当前切片边界。

### 3. 为 pack skill 单独做 UI 状态面

拒绝。

当前主界面只需要 pack 可见面，不需要再看 skill 内部状态。

## 收口

这轮维持一条短主链：

- `wpkg install`
- 写 `packs.json`
- prompt 按需提示 pack skill 路径
- 主脑任务相关时再读对应 `SKILL.md`
