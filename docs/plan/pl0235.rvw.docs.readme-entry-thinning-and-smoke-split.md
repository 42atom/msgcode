# Problem

根 `README.md` 同时承担了四种角色：

1. 产品入口
2. 上手手册
3. 维护者约束
4. 渠道 smoke 基座

结果是首页信息流倒挂：

- 读者还没跑起来，就先被长段哲学和维护规则拦住
- 真机 smoke 基座混在首页，和普通使用者无关
- `README.md`、`docs/README.md`、`test/README.md` 三处重复同一套 smoke 细节

## Occam Check

- 不加新层，能不能解决？
  - 能。只需要重排入口顺序，并把重复内容拆成一份 testing 文档。
- 这次改动是不是在替 runtime 做决定？
  - 不是。只是在收口文档表达。
- 拿掉重复 smoke 段后会不会更接近单一主链？
  - 会。渠道 smoke 只保留一个正式入口。

## Decision

采用最小收口方案：

1. 根 `README.md`
   - 保留一句话定义
   - 前置最小主管道图
   - 快速开始提到最前
   - 重哲学压缩成短原则与链接
2. 新增 `docs/testing/feishu-live-smoke.md`
   - 承接真实飞书 smoke 默认基座
   - 承接 `test-real`、默认 workspace、默认凭据来源、执行步骤、约束
3. `docs/README.md` 与 `test/README.md`
   - 删除重复 smoke 细节
   - 只保留索引和入口

## Plan

1. 新建任务真相源与计划文档
2. 重排根 `README.md`
3. 新增 `docs/testing/feishu-live-smoke.md`
4. 更新 `docs/README.md` 与 `test/README.md`
5. 更新 `docs/CHANGELOG.md`
6. 验证：`npm run docs:check`

## Risks

- 风险：README 收得太狠，丢掉必要上下文
  - 回滚：把关键边界留在“当前主链/设计原则”而不是重新塞回首页顶部
- 风险：新增 testing 文档变成第二套测试协议
  - 回滚：明确它只承接既有 `pl0098` 口径，不发明新流程

## Result

- 首页已切回用户入口优先
- 主管道图已补上
- 渠道 smoke 已独立成文
- `docs/README.md` 与 `test/README.md` 已改为单点指向
