# PL0382：Conflict Mode Prompt Injection

## 结论

这条线只做一件事：

把 `full | assisted` 收成可热切换的 prompt mode 注入。

不做 runtime 冲突治理平台。

## 为什么现有文件不能直接承载

- `AGENTS.md` 是仓库级铁律，不是具体任务
- `rf0000` 是 Electron 长期原则，不是 agent mode 合同
- `pl0211` 是主线导航，不承载具体方案

所以需要一张很薄的 plan。

## 实现顺序

### Step 1

找出现有主脑 prompt/skill 注入落点。

目标：

- 不新增层
- 只复用现有 prompt 主链

### Step 2

新增 mode prompt 片段：

- `full`
- `assisted`

要求：

- 只描述 AI 遇到本机冲突时的姿态
- 不扩写成更大的权限/治理模式

### Step 3

补热切换入口。

要求：

- 用户切换后下一轮注入立即生效
- 不要求重启 app
- 不要求重建整套 runtime

### Step 4

加最小验证：

- 当前 mode 能被读出
- prompt 注入内容随 mode 改变
- 不影响既有 runner 合同

## 不做什么

- 不做 conflict-policy helper
- 不做 browser/session 自动修复层
- 不做更大的“电脑管理模式”系统
- 不做 UI 大改

## 一句话

先让 AI 知道该怎么处理冲突，再决定是否需要系统替它多做事。
