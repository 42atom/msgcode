# RP0332：workspace people pending 键实现切片

## 结论

`people-pending.json` 的读面现在正式要求：

- `channel`
- `chatId`
- `senderId`

缺任一字段：

- 给 warning
- 不进入输出

旧 pending 键不再接受。

## 本轮实现

改动：

- `src/runtime/workspace-people.ts`
  - `WorkspacePendingPerson` 改成 `chatId + senderId`
  - `readPendingPeople(...)` 只认新键

- `test/p7-tk0296-appliance-people-contract.test.ts`
  - 正常样本改成新键
  - 新增旧键样本 warning 锁

## 为什么这样收

People 已知人物 CSV 的唯一键已经是：

- `channel + chatId + senderId`

pending 如果还保留旧键：

- 后续“已认领自动隐藏”只能靠猜
- 同一个 sender 在不同 chat 里会混淆

所以这轮先把前提钉死，不做兼容层。

## 明确不做

1. 不实现 pending 差集过滤
2. 不实现 importer
3. 不做旧格式自动迁移
