# RP0331：People 最薄写口实现切片

## 结论

People 第一版写入口已经落成：

- `msgcode appliance people-save`

它只做一件事：

- 按 `channel + chatId + senderId` upsert 一条 CSV 记录

## 本轮实现

新增：

- `src/runtime/workspace-people-csv.ts`
- `src/runtime/workspace-people-save.ts`
- `test/p7-tk0331-appliance-people-save.test.ts`

扩展：

- `src/runtime/workspace-people.ts`
- `src/cli/appliance.ts`

## 结果

当前写口支持：

- 新建 CSV 并写表头
- 追加新人物
- 更新已有人物
- 保留 `first_seen_at`
- 保留已有 `role`

当前写口不支持：

- `tags`
- `role` 自由编辑
- `pending` 过滤
- 批量导入

## 为什么这样收

因为原型里真正有真相源的位置只有：

- `统一称谓 -> alias`
- `备注 -> notes`

`tags` 目前没有稳定文件承接，硬上只会把 People 做成半套 CRM。
