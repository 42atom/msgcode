# Plan: Doc Filename Tidy (Outliers Only)

## Goal

- 用最小改动整理文档文件名“离群点”，不升级成全仓迁移工程。

## Non-Goals

- 不改变仓库已冻结的文档协议与 `docs:check` 规则。
- 不为旧文件名保留 stub/墓碑文件。
- 不引入新的索引系统或中间层。

## Decision

1. 本轮采用 **outliers-only**：只动明显离群、影响小、可回滚的点。
2. 命名协议以仓库现有检查为准：
   - `issues/NNNN-<slug>.md`
   - `docs/plan/plNNNN.<state>.<board>.<slug>.md`
   - `docs/plan/rsYYMMDD-<topic>.md`

## Phases

- [ ] Phase 1（盘点确认）：列出离群文件与引用点清单（只针对改名目标）
- [ ] Phase 2（执行改名）：`git mv` + `rg` 更新引用
- [ ] Phase 3（验证）：跑 `npm run docs:check`；抽查关键文档跳转
- [ ] Phase 4（收口）：回写 Issue 0183 的 Notes/结果

## Task List（建议顺序）

### P0：修 `docs/tasks/` 离群命名

- 改名：
  - from: `docs/tasks/benchmark-task-phase3-soul-context-fix-260310.md`
  - to: `docs/tasks/p6-agent-core-benchmark-task01-phase3-soul-context-fix-260310.md`
- 引用更新：
  - 先跑：`rg -n \"benchmark-task-phase3-soul-context-fix-260310\" -S .`
  - 对命中的路径逐个更新
- 验证：
  - `npm run docs:check`

### P1：定 `docs/archive/` 噪声口径（不急）

- 输出一条明确规则：`.build/checkouts/**` 这类 vendor 文档是否允许长期留在 `docs/archive/`。
- 若不允许：另起 issue，走单独迁移批次（避免混入 P0 改名批次）。

### P2（可选）：绝对路径链接收敛

- 先生成清单（按文件分组），再决定是否做批量替换。
- 原则：只做“能证明不破链接”的替换。
