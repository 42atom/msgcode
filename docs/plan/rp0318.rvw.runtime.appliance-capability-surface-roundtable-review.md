# RP0318：appliance capabilities 圆桌复核

Decision:
- Recommendation: go
- Confidence: medium
- Scope: 允许这条切片进入 rvw，但必须接受“多数能力位仍然空着”这一现实

Why:
- `SystemsArchitect`：这次没有新建 provider registry，只是把现有 runtime/config/env 事实做了只读组合。
- `SkepticalOperator`：tmux 工作区被明确收成 warning 空态，避免把执行臂误装成配置中心。
- `UserAdvocate`：对用户最重要的是诚实；空态比假配置更好。
- `ExecutionLead`：这刀可验证、可回滚，且不会阻塞后续 UI 只读接线。

Risks:
1. UI 很容易嫌“空态太多”，然后倒逼 runtime 去补假数据。
2. TTS/ASR 目前带 env/本地路径色彩，后续若改交付方式，需要同步调整读面。
3. local lane 的视觉默认值来自本地后端，不同宿主可能表现不完全一致。

Guardrails:
- 前置条件:
  - 只允许复用现有 runtime/config/env/path 事实
  - 不允许顺手做 provider 编辑
- 监控指标:
  - 空态能力位是否继续保持空态
  - tmux 工作区是否仍然返回 warning
- 回滚条件:
  - 若这条切片开始要求新增 provider 控制面或 secrets 管理，立即回滚

Next Actions:
1. UI 只消费当前空/实状态
2. 后续若真要做 provider 编辑，单开新切片
3. image/video/music/search 继续等统一真相源，不抢现在的主线
