# Plan: AIDOCS Structure Closure And Archive Cleanup

Issue: 0196

## Problem

`AIDOCS/` 目前是辅助材料总仓，但目录边界仍不清：

1. `refs/` 与 `references/` 重复
2. `design/`、`notes/` 仍像“半现役真相源”
3. `imessage-compare/`、`p5-7-r3c-data/` 等历史资料没有统一降权口径
4. `AIDOCS/README.md` 与真实目录不完全一致

结果是：

- 新人看不懂哪些该读、哪些只是历史输入
- 搜索噪声高
- 目录职责漂

## Decision

按“辅助现役 / 历史参考 / 一次性数据”三层收口：

1. `refs/`：统一承接参考资料
2. `notes/`、`design/`：明确为草稿 / 笔记 / 临时研究，不是正式真相源
3. 历史数据与对照资料迁入更明确的 archive/legacy 位

## Workstreams

### 1. 重复目录合并

- `AIDOCS/references/` → `AIDOCS/refs/`
- 修正旧引用与 `README`

### 2. 历史资料降权

- `AIDOCS/imessage-compare/` → `AIDOCS/refs/imsg/` 或 archive 子目录
- `AIDOCS/p5-7-r3c-data/` → 历史数据位（保留可追溯性）

### 3. 草稿/笔记口径

- `AIDOCS/design/`：历史设计草稿
- `AIDOCS/notes/`：工作笔记 / 临时研究
- 目录说明里明确“不等于正式 plan”

### 4. 根 README 收口

- 保证 `AIDOCS/README.md` 只描述真实存在且仍有效的目录

## Acceptance

- `AIDOCS` 目录职责一眼可懂
- 新同学不会把 `AIDOCS` 误读成第二套正式文档体系
- `npm run docs:check` 保持通过
