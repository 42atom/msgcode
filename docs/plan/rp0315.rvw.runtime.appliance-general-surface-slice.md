# RP0315：appliance general 读面实现切片

## 结论

这次没有去实现“通用设置中心”。

只做了一条最薄读面：

- `msgcode appliance general --workspace <...> --json`

它只复用三类现有事实：

1. `WORKSPACE_ROOT`
2. msgcode 日志路径
3. launchd 启动事实

## 为什么这么收

因为设置页“通用”原型里，只有这几项现在有硬来源：

- 场景工作区根目录
- 日志路径
- 启动方式

而这些项都已经能从现有系统拿到。

其他项现在没有稳定真相源：

- 默认场景工作区
- 多语言
- 账号连接

如果这时强做，就是在 UI 原型后面补一套假的控制面。

## 这次的读面来源

### 1. 工作区根目录

来源：

- `getVersionInfo().workspaceRoot`
- 底层来自 `WORKSPACE_ROOT`

处理原则：

- 有就展示
- 没有就 warning
- 不自动猜默认值

### 2. 日志路径

来源：

- `resolveMsgcodeLogDir()`
- `resolveLaunchAgentLogPaths()`

输出：

- `msgcode.log`
- `daemon.stdout.log`
- `daemon.stderr.log`

### 3. 启动方式

来源：

- `readLaunchAgentRuntime()`

输出：

- 当前模式：`launchd` / `manual`
- 是否支持 launchd
- 是否已安装
- 当前状态
- plist 路径

## 这次刻意不做

1. 不做 `general.json`
2. 不把原型里的全部字段都硬接进 CLI
3. 不做账号连接
4. 不做多语言
5. 不做“默认场景工作区”偏好写回

## 当前边界

这条切片只说明：

- 通用页里有哪些东西已经有真相源
- 哪些还没有

它不是 settings 总线，也不承诺现在就把原型字段全部落地。
