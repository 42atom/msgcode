# PL0408：Desktop Thread Surface Read Model Aggregation

## 顺序

1. 先盘点 UI 真正需要的字段
2. 再把聚合下沉到 runtime
3. bridge 继续只做转运
4. 最后把 renderer 改成纯消费

## 裁决

- 对：runtime 产出更完整的 surface read model
- 不对：main process 组装完整 UI JSON 树
- 不对：为普通桌面对话再引入 task 平台

## 口径

- runtime 负责真相聚合
- bridge 负责通道
- renderer 负责渲染

一句话：

这张 plan 只负责把“谁来组装 UI 真相”钉死，不扩新层。
