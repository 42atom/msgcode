# PL0276：WORKSTATE 纯文本试跑

## 结论

先不改 runtime。

先只让 agent 在两个边界按协议手工/半自动写 `WORKSTATE`：

1. compact 前
2. `doi -> rvw` 前

这样最薄，也最容易看清 `WORKSTATE` 是否真有价值。

## 目标

- 把 `WORKSTATE` 从“设计存在”推进到“可试跑”
- 保持 file-first
- 不增加 manager / controller / observer

## 边界

### 写入时机

只允许两类：

1. context compaction 即将发生时
2. 当前任务准备从 `doi` 推进到 `rvw` 时

### 写入对象

只针对：

- 当前活跃主任务
- 非轻量任务
- 已经形成明确 hypothesis / evidence / touched files 的任务

### 不写

- 闲聊
- 一步完成的小任务
- 没有形成工作态骨架的信息不足场景

## Prompt 合同

prompt 需要明确这四点：

1. 模板来源：`docs/protocol/WORKSTATE.md`
2. 写入路径：`<workspace>/.msgcode/workstates/<taskId>.md`
3. 写法：覆盖当前有效骨架，不做 append-only 日志
4. 冲突边界：`issues/`、dispatch、evidence 仍是真相源

## 验收

1. agent 在关键边界知道要不要写 `WORKSTATE`
2. agent 写出来的内容不脱离协议
3. 没有新增 runtime 自动层
