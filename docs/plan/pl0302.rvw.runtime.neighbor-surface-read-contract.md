# PL0302：Neighbor 只读文件合同

## 结论

邻居页第一版只承认三份真相源：

1. `<workspace>/.msgcode/neighbor/config.json`
2. `<workspace>/.msgcode/neighbor/neighbors.json`
3. `<workspace>/.msgcode/neighbor/mailbox.jsonl`

UI 不直接拼协议，不直接扫网络，也不自己记状态。

## 页面块与文件

### 模块开关 / 当前节点

读 `config.json`。

最小字段：

- `enabled`
- `nodeId`
- `publicIdentity`

不先加：

- 心跳
- 广播状态
- 在线人数

### 已发现节点列表

读 `neighbors.json`。

每项最小字段：

- `nodeId`
- `displayName`
- `state` (`discovered | known | contact`)
- `lastMessageAt`
- `lastProbeAt`
- `lastProbeOk`
- `latencyMs`

mailbox 只补：

- `unreadCount`

### 右侧记录流

读 `mailbox.jsonl` 最近几条。

每条最小字段：

- `at`
- `nodeId`
- `direction` (`in | out | system`)
- `type` (`hello | handshake | message | delivery`)
- `summary`
- `unread`

不先加：

- artifact 富对象
- owner
- patch / review / handoff

字段名只认一套：

- `nodeId`
- `displayName`
- `lastMessageAt`
- `lastProbeAt`
- `lastProbeOk`
- `latencyMs`

第一版不保留隐式 snake_case / alias 兼容。

## 读面原则

1. 缺文件返回空态
2. 非法 JSON / JSONL 给 warning
3. UI 只读最近几条 mailbox，不把 mailbox 当聊天全文
4. mailbox 未读数可按节点聚合
5. 不从 UI 反写任何邻居状态
6. 邻居页头部只读：
   - `summary.unreadCount`
   - `summary.lastMessageAt`
   - `summary.lastProbeAt`
   - `summary.reachableCount`

## 壳出口

新增：

- `msgcode appliance neighbor --workspace <...> --json`

它只做两件事：

1. 忠实读文件
2. 返回最薄 surface

不负责：

- 自动发现
- 自动修复
- 自动补全联系人

当前 surface 刻意不带：

- `counts`
- mailbox 每条记录的 `sourcePath`
- mailbox 每条记录的 `displayName`
