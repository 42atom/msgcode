# Review

Decision:
- Recommendation: go
- Confidence: high
- Scope: Web composer 继续推进，但共享核心改为 `RuntimeRouterHandler.handle()`，不是 `listener.handleMessage()`

Why:
- `listener` 是通道适配层，里面明确带 Feishu 路由、`saveCurrentSessionContext(transport=feishu)`、`sendClient` 假设
- `RuntimeRouterHandler` 才是真正的工作主链，并且已经会把 user/assistant 双向写入 `.msgcode/sessions/*.jsonl`
- 同一路径也已经会把 turn 写进 `.msgcode/threads/*.md`

Risks:
1. 若继续碰 listener，会把 Web 需求反向污染 Feishu 主链
2. 若改走 daemon，会长出第二消息主链
3. 若让 UI 先画气泡，再回写 runtime，会立刻形成第二真相源

Guardrails:
- 前置条件: 只做 one-shot consumer，不做常驻消费器
- 监控指标: 输出必须继续落在现有 session/thread 真相源
- 回滚条件: 一旦需要新 transport、daemon、或 optimistic bubble，就停

Next Actions:
1. 关闭 0321 基线
2. 开 0335，只做 RuntimeRouter one-shot consumer
3. 第一版只承接自然语言，不抢 slash/route 控制面

Evidence:
- Code: `src/listener.ts`, `src/handlers.ts`, `src/session-window.ts`, `src/runtime/thread-store.ts`, `src/runtime/inbox-store.ts`
- Tests: `test/p7-tk0319-inbox-add-cli.test.ts`, `test/p7-tk0293-appliance-threads-contract.test.ts`
