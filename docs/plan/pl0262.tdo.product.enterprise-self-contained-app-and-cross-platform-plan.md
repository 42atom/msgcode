# PL0262：企业版自包含、App/UI 与跨平台计划

## Context

当前主线已经能跑，但正式交付口径仍偏开发者环境：

- 安装依赖 `brew` / `npm install`
- 运行依赖源码态 `tsx`
- shell 合同绑定 Homebrew Bash
- 浏览器、desktop、模型后端都还带明显的机器前提

用户现在提出的是另一条线：

1. 企业内网自包含，不现场下载
2. 可能要进入 App/UI 形态
3. 将来要考虑跨平台
4. 但当前本地模型又依赖 macOS + MLX

这已经不是单点修 bug，必须先拆成几条边界清楚的任务。

配套调研报告见：

- [rp0262](/Users/admin/GitProjects/msgcode/docs/plan/rp0262.rvw.product.enterprise-app-capability-layering-and-ip-boundary.md)

## 结论

这条线先拆成四刀：

1. 先定义企业版“自包含、不下载”的基线与验收
2. 再定 App shell / UI 形态，不先默认 Electron 就是答案
3. 再定本地模型后端与 MLX/mac 的边界
4. 最后收安装链，让企业环境不再依赖 `brew` 与 `npm install`

## Goal / Non-Goals

### Goal

- 为企业版形成一条清晰推进路线
- 把“交付形态”“App/UI”“模型后端”“安装链”拆开
- 避免后续把产品、运行时、模型和打包问题揉成一团

### Non-Goals

- 不在本单里直接选定 Electron
- 不在本单里直接实现打包系统
- 不在本单里直接重写模型后端

- 不先定义“企业版到底要求什么”，后面所有技术选择都会漂

2. 后续可以按这四张单逐张推进，而不是开一张“大而全企业版”
3. plan 明确推荐顺序与非目标
