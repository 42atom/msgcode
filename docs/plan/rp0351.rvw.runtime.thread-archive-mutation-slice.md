# RP0351：thread archive / restore 切片

## 结果

已增加两条正式 mutation：

- `msgcode appliance archive-thread --workspace <labelOrPath> --thread-id <threadId> --json`
- `msgcode appliance restore-thread --workspace <labelOrPath> --thread-id <threadId> --json`

## 主链

- archive：`threads/` -> `archived-threads/`
- restore：`archived-threads/` -> `threads/`

只移动 thread markdown 文件。

## 为什么这样够

- 归档页和主界面只需要 thread 文件层的活跃/非活跃切换
- session jsonl 仍可通过 `chatId` 回看，不必跟着搬
- `/clear` 继续只负责开新线程，不碰 archive

## 失败口径

- workspace 不存在：报错
- `threadId` 找不到：报错
- 同名目标文件若未来出现：底层 rename 失败，保持 error，不做隐式重命名
