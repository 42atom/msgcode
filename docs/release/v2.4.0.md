# v2.4.0 发布说明

发布日期：2026-03-13

## 版本概述

v2.4.0 是 msgcode 当前主链的一次收口版本。重点不是加新控制层，而是继续按 Unix 哲学把系统做薄：删代决、删解释层、删 legacy 桥，统一到 `ghost_*` 现役桌面能力面，并把历史桌面桥归档出主树。

## 主要变更

### Runtime 主链继续做薄

- `tool-loop` 删除 recovery nudge 与 synthetic recovery prompt
- 工具失败默认回到“模型 -> 工具 -> 结果 -> 模型”主链
- quota continuation 继续保留结构化事实，不回退成系统代答

### Tool Bus 收口为薄网关

- 删除 `read_file` 的教学式 guidance 与多余解释层
- 删除 success preview 里的重复确认句
- `desktop`、`feishu_*`、`file tools` 的厚执行逻辑外移到独立 runner
- Tool Bus 更接近“参数校验 -> 执行真实能力 -> 返回结构化结果”

### Desktop 主链切换到 ghost

- `ghost-os` 已挂入正式工具主链
- Agent 直接使用 `ghost_*` 原生工具
- legacy `desktop` 已退出现役 Tool Bus、slash 命令与默认工具暴露
- 高风险动作约束继续留在 skill 与 prompt 的语义契约，不新增 confirm gate

### CLI 与历史债务继续清退

- `file/system/web/media` 等二手 Unix 包装层退役
- `jobs/skills/gen-image/gen-audio` 等隐藏或复数兼容入口继续退出主链
- CLI 默认回到更直接的 canonical 命令面

### 仓库结构继续瘦身

- legacy Desktop Bridge 已整体迁入 `docs/archive/retired-desktop-bridge/`
- 根目录历史文件与 retired `imsg` 脚本继续归档
- 现役树继续强调当前主链与当前能力面

## 当前主线能力面

- 桌面能力：`ghost_*`
- 本地文件/系统壳：优先原生 Unix/macOS 工具
- Agent 执行核：默认忠实回传真实工具结果，不再替模型做恢复决策

## 迁移提示

1. 旧 `desktop` 主链已退出：默认桌面能力请改用 `ghost_*`
2. 旧 CLI 包装层已大幅退役：文件与系统操作请优先直接用原生 shell / 工具
3. 若本机缺少 Ghost OS，请先安装并初始化：

```bash
brew install ghostwright/ghost-os/ghost-os
ghost setup
ghost doctor
ghost status
```
