# msgcode Tool Loop SLO v1

状态：生效（建议作为默认验收口径）  
更新时间：2026-02-20

## 1. 目标

为 `agent tool loop` 建立可量化、可回归、可降级的稳定性标准，避免“看起来能用、线上不稳”。

## 2. 八条运行原则（冻结）

1. R1 与 R2 分门禁  
  R1 命中 `tool_calls` 不算通过，必须同时满足 R2 产出可展示自然语言。
2. 模型分级，不混用  
  A 级模型跑 tool loop；B 级模型只跑纯对话。
3. 参数基线固定  
  tool loop 全链固定 `temperature=0`；其他采样参数走固定 profile。
4. 工具协议最小化  
  schema 小而硬（required 明确，禁止额外字段）；返回统一 envelope。
5. 单步单工具优先  
  每轮最多 1 个工具调用；复杂任务走多轮，不追一轮多工具。
6. 失败可诊断、可重试  
  区分协议失败/工具失败/清洗空答复，并有分类重试策略。
7. skill 文件真相源  
  系统只注入 skill 索引与调用方法；详细内容按需 `read_file`。
8. 持续回归  
  每日固定 20-case 作为 smoke gate；SLO 由线上连续流量统计。

## 3. 指标定义（SLI/SLO）

统计口径分层：

1. `SLO 口径`：基于线上连续流量的 7 日滚动统计（用于发布与降级决策）。
2. `Smoke 口径`：每日 20-case 固定集（用于快速回归与阻断明显回归）。

| 级别 | 指标 | 定义 | SLO | 告警阈值 | 处置 |
|---|---|---|---|---|---|
| P0 | R1 命中率 | 需要工具请求中，R1 返回结构化 `tool_calls` 比例 | >= 98% | < 95% | 降级到安全模型/纯文本 |
| P0 | R2 可展示率 | 工具执行后，R2 返回非空自然语言比例 | >= 97% | < 94% | 触发兜底回复并标记失败 |
| P0 | E2E 成功率 | 从请求到最终可展示答复的端到端成功率 | >= 95% | < 92% | 启动降级路由 |
| P1 | 工具执行成功率 | 工具调用中 `ok=true` 比例 | >= 97% | < 93% | 按错误码分类重试 |
| P1 | 二轮漂移率 | `tool_calls=[]` 且文本出现 `<tool_call...>` 比例 | <= 1% | > 2% | 启用漂移兜底并告警 |
| P1 | 清洗后空答复率 | 输出清洗后为空比例 | <= 1% | > 2% | 记录样本并降级 |
| P2 | P95 时延 | E2E 95 分位耗时 | <= 12s | > 20s | 降 token/减重试 |

### 3.1 分母定义（防口径漂移）

1. `需要工具请求（offline）`  
  使用固定标注基准集（L1-L4）中标为 `tool-required=true` 的样本。
2. `需要工具请求（online）`  
  以运行时路由判定为准：`route in {"tool","complex-tool"}`。
3. `R1 命中率`  
  `R1_toolcall_hit_rate = R1_结构化tool_calls成功数 / 需要工具请求总数`。
4. `R2 可展示率`  
  `R2_displayable_rate = R2_非空可展示自然语言数 / 已执行工具请求总数`。

## 4. 统一返回合同（工具侧）

工具返回 envelope 统一为：

```json
{
  "ok": true,
  "data": {},
  "errorCode": "",
  "error": "",
  "exitCode": 0
}
```

失败时必须满足：

1. `ok=false`
2. `errorCode` 为固定枚举
3. `exitCode != 0`（CLI 进程返回码同步非 0）

## 5. 失败分类与重试策略

1. MODEL_PROTOCOL_FAILED  
  表现：无结构化 `tool_calls` 或响应结构异常。  
  策略：切安全模型重试 1 次；失败则降级纯文本。
2. TOOL_EXEC_FAILED  
  表现：工具返回 `ok=false`。  
  策略：按 `errorCode` 决定是否重试（参数类不重试，暂时性错误可重试 1 次）。
3. EMPTY_AFTER_SANITIZE  
  表现：R2 有响应但清洗后为空。  
  策略：走兜底文案 + 记录样本 + 触发告警。

## 6. 每日 20-case 回归（Smoke Gate）

1. L1（5 条）：单工具成功（`read_file` / `write_file` / `bash`）
2. L2（5 条）：工具失败可诊断（路径/权限/参数）
3. L3（5 条）：二轮收口稳定（无漂移/无空答复）
4. L4（5 条）：复杂任务链路（计划->执行->收口）

Smoke 通过阈值（每日）：

1. 总通过数 `>= 19/20`
2. L1+L2 不得出现连续失败
3. 若触发失败，阻断发布并进入回归修复

## 7. 默认运行配置（当前建议）

1. tool profile：`temperature=0`
2. no-tool profile：`temperature=0.2`
3. complex-tool：串行，不并发

## 8. 上线/降级规则

1. 通过线：P0 三项全部满足线上 7 日滚动阈值（不使用 20-case 直接替代）。
2. 触发降级：任一 P0 低于告警阈值，立即切安全路由。
3. 恢复条件：连续 3 天回到 SLO 目标以上。
