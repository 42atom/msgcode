## Context

- 真实 `~/.config/msgcode/routes.json` 里存在：
  - `chatGuid = test-chat-guid`
  - `status = active`
  - `workspacePath` 指向临时目录
- 该路径已经不存在，但旧的 RouteStore 仍把它当 active route 留在真相源里。
- 这会污染：
  - `getActiveRoutes()`
  - schedule projection rebuild
  - probe / info 等依赖 active routes 的主链

## Structure Review

- route 真相源既然是 `routes.json`，就不能把“已失效的 active 绑定”长期保留。
- 这不是编排问题，也不需要新层。
- 最薄收口就是：读 `routes.json` 时发现 active route 的 workspacePath 已不存在，就直接删掉并落盘。

## Goal / Non-Goals

### Goal

- 自动删除 active 且 workspacePath 已不存在的 route 条目。
- 保持 archived/paused route 与正常 active route 不受影响。
- 让 active routes 重新回到“当前仍可用”的真相。

### Non-Goals

- 不自动恢复缺失 workspace。
- 不替用户重建目录。
- 不扩展 route schema。
