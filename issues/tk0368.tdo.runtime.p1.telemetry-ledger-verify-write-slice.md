# 任务：verify telemetry writer

## Parent Task

- `tk0363`

## Context

现有 telemetry ledger 合同已经固定：

- 单一日账本：`<workspace>/.msgcode/telemetry/telemetry-YYYY-MM-DD.ndjson`
- append-only
- 最小字段
- 默认保留 7 天

`tk0367` 已把 tool bus 的稳定单一写点接进 ledger。

这一刀只收 verify 主链的最薄写侧：

- `verify run`
- `verify pack`

不把 `probe / runner / provider / inbound` 一起拉进来，不做 reader。

## Goal / Non-Goals

### Goal

- 让 verify 主链在单一稳定写点 append verify 事件到同一份日账本
- 保持现有 `status.log`、工具 ring buffer、failure snapshot 主链不变

### Non-Goals

- 不做 ledger reader
- 不改 `status.log`
- 不做 dashboard
- 不扩到 `probe / runner / provider / inbound`
- 不碰 `tool` writer slice

## Acceptance Criteria

1. `verify run` 和 `verify pack` 结束后，会把 verify 事件追加到 `telemetry-YYYY-MM-DD.ndjson`。
2. `TelemetryLedgerKind` 仍然是同一份合同，不新增第二套账本。
3. 现有 `tool` ring buffer 与 `/toolstats` 读法不变。
4. 不新增 server、数据库、控制面。

## Plan

- [x] 确认现有 `tk0363` / `tk0367` 不能直接承载 verify writer slice
- [x] 新建最薄 `tk0368` 实现单
