# 任务：总工作区归档 root 与恢复合同

## Context

工作区归档的目标不是删除，而是把不再活跃的整个工作区移出主界面。

要做到这一点，必须先有一个总归档 root，作为唯一真相源。

否则 UI 只能自己猜哪些工作区“像是被归档了”，这会直接长出第二状态层。

## Goal

- 定义 `/Users/admin/msgcode-workspaces/.archive/`
- 定义 archive/restore 作为目录移动
- 定义同名冲突失败口径

## Goal / Non-Goals

### Goal

- 定义总工作区 archive root
- 定义 archive / restore 作为目录移动
- 定义主界面只读活跃工作区
- 定义同名冲突失败口径

### Non-Goals

- 不做自动归档
- 不做 archive 索引表
- 不做工作区重命名向导

## Plan

- [ ] 钉死总工作区 archive root 路径
- [ ] 钉死主界面活跃工作区读取边界
- [ ] 钉死 workspace archive / restore 的目录移动口径
- [ ] 钉死同名冲突失败口径

## Acceptance Criteria

1. archive root 路径被钉死。
2. 主界面活跃工作区读取边界被钉死。
3. restore 的冲突策略被钉死。

## Notes

- archive root 第一版建议是 `/Users/admin/msgcode-workspaces/.archive/`
- UI 文案可以叫“项目归档”，但文件系统保持简单路径
- 恢复若遇到根目录同名工作区，先直接失败，不做隐式重命名
