## Context

- 用户已经明确要求“尽量一次性安装到位”，希望减少模型之后自行摸索环境。
- 仅改 runner 不够；如果安装链不默认提供现代 Bash，主链会继续停留在“环境碰运气”。
- 目前仓库里还没有一条正式、收口的“托管 Bash 默认安装”合同，只有外部参考资产。

## Goal / Non-Goals

- Goal:
  - 在正式安装资产里把 Homebrew `bash` 变成默认依赖
  - 提供最小验证步骤，证明托管 Bash 已装好且可被主链使用
  - 让安装合同与运行时合同一一对应
- Non-Goals:
  - 不顺手把所有机器 bootstrap 平台化
  - 不新增多 profile 安装系统，除非当前正式安装入口确实需要最小拆分
  - 不改 runner 逻辑本身

## Plan

- [ ] 盘点当前正式安装入口；若缺失，则新增最小安装资产，不额外造平台
- [ ] 默认加入 Homebrew `bash`
- [ ] 增加最小 doctor / smoke 命令，验证 bash 路径与版本
- [ ] 把安装输出与 runner 合同对齐，明确主链只认托管 Bash

## Acceptance Criteria

1. 正式安装入口默认包含 Homebrew `bash`。
2. 安装完成后，存在明确验证命令证明：
   - 托管 Bash 路径存在
   - 版本是现代 Bash
3. 文档或脚本中明确写出：`bash` 工具依赖该托管 Bash。
4. 不引入新的安装编排层；仍保持薄安装面。

## Notes

- 这里的“安装资产”可以是最小 `Brewfile`、bootstrap 脚本，或仓库已存在的等价入口；关键是要有正式真相源。
- 若当前仓库尚无正式 bootstrap 入口，优先新增最薄版本，只解决 Bash 合同，不顺带扩成大而全安装器。
