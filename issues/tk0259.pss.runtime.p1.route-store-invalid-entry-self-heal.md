## Context

- 真实 `routes.json` 里存在一条 key 为 `[object Object]` 的脏记录。
- 这条记录只有：
  - `chatGuid`
  - `createdAt`
  - `updatedAt`
- 它缺 `workspacePath`、`status`、`botType` 等关键字段，但旧的 `loadRoutes()` 不会清掉它，`getActiveRoutes()` 还会把它带出来。

## Structure Review

- `routes.json` 是 route 真相源。
- 既然它是单一真相源，就不能容忍半条 route 长期留盘。
- 不需要新层；只要把 RouteStore 的读写口径收紧：
  - 读时自愈
  - 写时拒绝无效输入

## Goal / Non-Goals

### Goal

- `loadRoutes()` 自动删除缺关键字段的脏 route 并落盘。
- `setRoute()` 拒绝写入无效 RouteEntry。
- 不影响正常 `/bind`、`/where`、`getActiveRoutes()` 主链。

### Non-Goals

- 不改 route schema。
- 不新增 route 修复任务系统。
- 不自动猜测脏条目应该映射到哪个 workspace。

## Plan

- [x] 给 RouteStore 增加最小 RouteEntry 形状校验
- [x] `loadRoutes()` 读时删除脏条目
- [x] `setRoute()` 写时拒绝无效 RouteEntry
- [x] 补单测并用当前自愈逻辑清掉真实 `[object Object]` 脏记录

## Acceptance Criteria

1. 缺关键字段的 route 条目会在 `loadRoutes()` 时被删除并落盘。
2. `setRoute()` 不再接受半条 RouteEntry。
3. 既有 RouteStore 单测继续通过。

## Result

- `/Users/admin/GitProjects/msgcode/src/routes/store.ts` 新增最小形状校验。
- `loadRoutes()` 现在会删除缺 `workspacePath/status/botType` 等关键字段的条目。
- `setRoute()` 现在会拒绝写入无效 RouteEntry。
- 当前真实 `~/.config/msgcode/routes.json` 里的 `[object Object]` 脏记录已被清掉。

## Notes

- 这刀只收口真相源，不追溯脏记录最早由哪次历史写入造成。
- 真实 `routes.json` 里还保留 `test-chat-guid` 这样的历史测试 route；它有完整结构，所以不属于本单的“无效条目”范围。
