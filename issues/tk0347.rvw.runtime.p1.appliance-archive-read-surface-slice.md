# 任务：appliance archive 读面切片

## Context

archive 原型已经独立成页，但之前只有模拟数据。

主线真正缺的是一条正式读面：

- 工作区 archive root 读什么
- archived threads 读什么
- 缺目录或坏文件时怎么降级

这条读面一旦成立，UI 后面就不用继续靠脑补。

## Goal / Non-Goals

### Goal

- 增加 `msgcode appliance archive --workspace ... --json`
- 只读 `.archive/` 下的归档工作区
- 只读 `<workspace>/.msgcode/archived-threads/` 下的归档线程
- 缺目录时返回空，不报错
- 坏项时返回 warning，不崩

### Non-Goals

- 不实现 workspace restore
- 不实现 thread restore
- 不移动 session jsonl
- 不改 archive 页 UI

## Plan

- [x] 增加 `msgcode appliance archive --workspace ... --json`
- [x] 只读 `.archive/` 下的归档工作区
- [x] 只读 `<workspace>/.msgcode/archived-threads/`
- [x] 缺目录时降级为空
- [x] 坏项时给 warning

## Acceptance Criteria

1. archive 命令能输出归档工作区与归档线程。
2. 缺 archive 目录时降级为空。
3. 非目录项和坏线程文件会产生 warning。
4. UI 后续可以只读这条主链，不再硬编码模拟数据。
