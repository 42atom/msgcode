# 任务：thread front matter 标题合同

## Context

用户可见的 thread 标题不能由 UI 自己发明。

否则很快就会变成：

- UI 截断第一条消息
- UI 维护标题缓存
- UI 再做一层 thread 名称表

这和 file-first 冲突。

## Goal / Non-Goals

### Goal

- 定义 thread front matter 最小字段
- 说明哪个字段供 UI 显示标题
- 说明 `chatId/transport` 为什么也应在 front matter 里

### Non-Goals

- 不改正文 turn 格式
- 不实现 thread 文件迁移
- 不设计二级标题系统

## Plan

- [ ] 定义 front matter 最小字段集合
- [ ] 说明标题字段的唯一来源
- [ ] 说明 UI 只读 front matter，不自拼标题

## Acceptance Criteria

1. front matter 最小字段清楚。
2. 标题字段来源清楚。
3. UI 不再需要截断第一条消息拼标题。

## Notes

- `title` 是用户层标题真相源。
- `transport` 是来源属性，不再与 thread 并列成一级心智。
