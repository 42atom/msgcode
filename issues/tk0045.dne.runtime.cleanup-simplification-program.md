## Context

当前分支已经把多条关键主链跑通：

- `read_file + bash` 的 skill 主链已通
- schedule 的 create / trigger / remove 已通
- finish supervisor 已接入统一结束口

但工作区仍堆积了几类“认知噪音”：

1. 一批已完成但尚未提交的 runtime skill 文案与测试改动
2. 一批未跟踪的中间计划/issue 草稿，来自已经被后续工作覆盖的中间阶段
3. 少量历史文件与退役口径仍可能制造阅读噪音

这些问题本身不一定阻断运行时，但会持续提高认知负担，影响后续派单、验收与维护效率。

## Goal / Non-Goals

- Goal: 清点并分类当前工作区中的改动、草稿和历史残影
- Goal: 只保留对当前正式口径仍有价值的文件
- Goal: 把 runtime skill 相关整理结果收成一笔清晰提交
- Goal: 让分支状态从“多笔中间草稿堆积”收敛成“少量可解释改动”
- Non-Goals: 不修改已跑通的 scheduler / browser / supervisor 运行时逻辑
- Non-Goals: 不顺手重构 agent-backend
- Non-Goals: 不扩到新的功能开发
