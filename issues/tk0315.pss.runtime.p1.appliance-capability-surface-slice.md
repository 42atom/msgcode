# 任务：appliance capabilities 读面切片

## Context

Settings 页的“智能体能力配置”原型里有很多能力位：

1. 大脑模型
2. 视觉模型
3. 语音输出
4. 听觉识别
5. 画图模型
6. 视频模型
7. 音乐模型
8. 联网搜索

但当前仓库真正有统一真相源的，并不覆盖全部。

## Goal / Non-Goals

### Goal

- 增加最小 `msgcode appliance capabilities --workspace <...> --json`
- 让它只输出已有硬事实支撑的能力位
- 无统一真相源的能力位，明确空态

### Non-Goals

- 不实现 provider 编辑
- 不实现连接测试
- 不做账号/API Key 管理
- 不把 image/video/music/search 伪装成已配置
- 不做 settings 写回

## Plan

- [x] 新增 workspace-capabilities 读面
- [x] 增加 appliance capabilities CLI
- [x] 补黑盒测试
- [x] 做一轮圆桌复核

## Acceptance Criteria

1. `appliance capabilities` 能稳定输出 JSON。
2. 大脑/视觉/语音/ASR 只在有硬事实时显示为已配置。
3. image/video/music/search 没有统一真相源时保持空态。
4. tmux 工作区不会被误装成“已配置能力中心”。

## Notes

- 这刀是只读面，不是 provider 控制面。
- UI 后面应只消费这里的空/实状态，不得自行补默认值。
