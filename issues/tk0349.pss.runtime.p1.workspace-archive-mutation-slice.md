# 任务：workspace archive / restore 切片

## Context

现在 archive 已经有正式读面。

如果继续让 UI 自己移动工作区目录，就会直接破坏 file-first 和单一主链。

所以这刀只做一件事：

- 给工作区 archive / restore 一个正式 CLI mutation 入口

## Goal / Non-Goals

### Goal

- 增加 `msgcode appliance archive-workspace --workspace ... --json`
- 增加 `msgcode appliance restore-workspace --workspace ... --json`
- archive / restore 只通过目录移动完成
- 同名冲突直接失败

### Non-Goals

- 不实现 thread archive / restore
- 不做 archive 批量动作
- 不做自动归档
- 不做 UI 页面改动

## Plan

- [x] 增加 workspace archive mutation
- [x] 增加 workspace restore mutation
- [x] 补同名冲突失败路径
- [x] 补 CLI contract test

## Acceptance Criteria

1. 工作区能归档到 `.archive/`。
2. 归档工作区能恢复回活跃根目录。
3. 同名冲突直接失败，不做隐式重命名。
4. UI 不需要直接改目录。

## Verify

`PATH="$HOME/.bun/bin:$PATH" bun test test/p7-tk0338-appliance-archive-contract.test.ts test/p7-tk0338-appliance-workspace-archive-restore.test.ts && ./node_modules/.bin/tsc --noEmit && npm run docs:check`

- restore 仍然只认目录移动
- 这刀不碰 thread archive
