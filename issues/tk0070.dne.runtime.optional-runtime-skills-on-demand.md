## Context

当前 repo 的正式 skill 真相源只有 `src/skills/runtime/`，模型默认只知道 `~/.config/msgcode/skills/index.json`。这会带来两个问题：

- 新增扩展型 skill 时，容易被误加进常驻主索引，污染默认上下文
- 如果完全不接到运行时，skill 又只是 repo 内静态文件，模型无法按需发现

本轮要收口的是一个更薄的形态：

- 基础 skill 继续常驻 `runtime`
- 扩展 skill 进入 repo 内置 `optional`
- 运行时同步到 `~/.config/msgcode/skills/optional/`
- 主索引找不到时，再按需查看 `optional/index.json`

## Goal / Non-Goals

### Goal

- 新增 repo 内置 `optional skills` 层
- 同步到运行时目录，但不并入默认常驻索引
- 补最小提示词约束，让模型按需查看 `optional/index.json`
- 内置这 5 个 skill：
  - `twitter-media`
  - `veo-video`
  - `screenshot`
  - `scrapling`
  - `reactions`

### Non-Goals

- 不做 skill marketplace
- 不做第二套技能平台
- 不做自动意图路由器
- 不把 optional skills 默认塞进模型主上下文
- 不补不存在的 reaction runtime 能力主链

- `msgcode start/init` 后运行时存在 `~/.config/msgcode/skills/optional/index.json`
- 主索引 `~/.config/msgcode/skills/index.json` 不包含这 5 个 optional skill
- 提示词明确：主索引无匹配时，再按需查看 optional 索引
- 测试锁住“已同步但未并入主索引”的语义
