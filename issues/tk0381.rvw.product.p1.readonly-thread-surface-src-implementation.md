# 任务：Readonly Thread Surface Src Implementation

## Context

`tk0377` 已经把正式主线钉死到只读线程面，但它本身不承载代码落点和验证。

当前又要求 `src` 成为真实结构落点，所以需要一张更薄的实现单，只收实现和证据，不回头把 `ui-protype` 当现役实现面。

## 为什么 tk0377 不能直接承载实现

`tk0377` 是主线单，只负责把边界钉死。

它不适合直接承载实现，因为它没有：

- 代码落点
- 验证命令
- 实现证据

而且 `tk0378` 还写在 `ui-protype` 上。现在 `ui-protype` 已经回到原型研究面，不能再拿来当正式 UI 落点。

所以这里单独起一张最薄实现单，只收 `src`。

## Goal / Non-Goals

### Goal

1. 在 `src/ui/main-window` 落正式 UI 第一刀
2. 保留主窗口三栏骨架
3. 保留 `settings` 一级入口 affordance
4. 保留未来宿主桥接的最薄占位

### Non-Goals

1. 不改 `ui-protype`
2. 不做发消息
3. 不做 archive 内容页
4. 不做 settings 内容页
5. 不做 Electron host bridge 实现
6. 不做风格重做

## Plan

- [ ] 新建 `src/ui/main-window/readonly-thread-surface.ts`
- [ ] 新建 `src/ui/README.md`
- [ ] 补最小测试，锁住三栏骨架、settings affordance 和 bridge 占位
- [ ] 保持 `ui-protype` 不动

## Acceptance Criteria

1. `src` 下存在正式 UI 第一刀的真实结构落点。
2. 三栏骨架、settings 一级入口 affordance、host bridge 占位都能从源码里看出来。
3. 没有把 archive、新聊天、composer/send 搬进来。
4. `ui-protype` 没被改成现役实现面。

## Notes

- 这张单只收 `src`，不回写 `ui-protype`。
- 结构优先，样式从简。
- bridge 只留位置，不做实现。
