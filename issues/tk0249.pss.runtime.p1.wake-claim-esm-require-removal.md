## Context

- 现场日志存在真实告警：`推进任务状态失败 [error=require is not defined]`
- `wake-claim.ts` 已是 ESM import 风格，但 `getStaleClaims()` 里还残留 `require("fs")`
- 这条路径会被 startup catch-up / heartbeat 消费链路触发

## Goal

- 清掉 `wake-claim.ts` 里的 CommonJS 残留
- 在 Node ESM 路径上锁住 stale-claim 扫描可正常运行

## Non-Goals

- 不改 wake claim 语义
- 不改 catch-up / heartbeat 策略
- 不补新的兼容层

## Goal / Non-Goals

- Goal:
  - `getStaleClaims()` 在 Node ESM 下可直接运行
  - heartbeat 相关推进链不再因这处 `require` 崩掉
- Non-Goals:
  - 不改 wake 文件合同
  - 不扩 wake 功能
  - 不动已通过的 claim/retention 设计

## Plan

- [x] 清掉 `wake-claim.ts` 的 `require("fs")`
- [x] 补 Node ESM 子进程回归
- [x] 跑 wake 定向验证

## Acceptance Criteria

1. `getStaleClaims()` 在 Node ESM 路径下不再报 `require is not defined`。
2. 既有 wake claim / catch-up 回归继续通过。
3. 不引入新的 compat 层。

## Notes

- 这是单点 ESM 漂移，不是 wake 设计缺陷。
- 最薄修法就是改成现成的 `node:fs` import。

- wake claim / catch-up / retention 相关既有回归继续通过
