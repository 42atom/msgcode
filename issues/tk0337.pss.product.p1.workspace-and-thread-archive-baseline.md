# 任务：工作区与线程归档基线

## Context

现在主界面已经收口到：

- 左边看工作区和线程
- 中间看当前线程
- 右边看工作状况

接下来需要一个同样薄的归档主链：

- 不重要的工作区可以收进总归档
- 不重要的线程可以收进当前工作区归档
- 主界面默认不显示归档内容
- 归档后仍可查看和恢复

同时必须和 `/clear` 明确分开：

- `/clear` 只清短期上下文，并开启新线程
- `archive` 是把内容从活跃面移出，但不删除历史

## Goal / Non-Goals

### Goal

- 定义总工作区归档目录
- 定义工作区内线程归档目录
- 定义主界面、历史归档页、恢复入口的显示原则
- 定义 archive 与 `/clear` 的边界
- 拆成后续可排期的小任务

### Non-Goals

- 不做自动归档策略
- 不做物理删除
- 不做数据库索引
- 不做标签系统
- 不做归档搜索平台
- 不做历史聊天再摘要

## Plan

- [x] 收 archive 的文件真相源
- [x] 收 `/clear` 与 archive 的边界
- [x] 收主界面如何隐藏归档内容
- [x] 收历史归档页最小入口
- [x] 拆出 workspace archive / thread archive / history surface 三张任务

## Acceptance Criteria

1. 归档被定义为“移动到归档目录”，不是删除，不是 `clear`。
2. 总工作区和线程都各自有单一真相源目录。
3. 主界面、历史页、恢复动作的边界清楚，不需要新增中间控制面。

## Notes

- 按 Unix 口径，优先用“目录移动”表达归档，不先造归档表。
- 第一版只做活跃与归档两层，不做更多状态。
- 若恢复时发生同名冲突，第一版应显式失败，不偷偷重命名。
