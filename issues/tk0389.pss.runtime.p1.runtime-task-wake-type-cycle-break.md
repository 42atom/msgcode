# TK0389：Runtime Task Wake Type Cycle Break

## Context

当前循环是类型层循环，不是运行时强耦合：

- `task-types.ts` 以 type-only 方式依赖 `wake-consume.ts` 和 `work-continuity.ts`
- `wake-consume.ts` 又 type-only 依赖 `task-types.ts`
- `work-continuity.ts` 也 type-only 依赖 `task-types.ts`

这说明共享接口和记录类型的位置不对。

## Goal / Non-Goals

### Goal

- 把共享任务/唤醒胶水类型拆出
- 让 `task-types.ts` 不再反向 import `wake-consume.ts / work-continuity.ts`

### Non-Goals

- 不改 heartbeat/supervisor 行为
- 不做 work-continuity 大瘦身

## Acceptance Criteria

1. `codegraph` 不再报告 runtime 三文件循环
2. 相关最小测试与 `tsc --noEmit` 通过

## Plan

1. 梳理 `task-types.ts` 中只为胶水依赖引入的类型
2. 拆到共享类型文件
3. 更新三文件导入
4. 复验 cycle 与最小测试

## Notes

- 这刀只改类型依赖，不改业务流程
- 共享 capsule 与 task document 状态类型已收进 `work-capsule-types.ts`
