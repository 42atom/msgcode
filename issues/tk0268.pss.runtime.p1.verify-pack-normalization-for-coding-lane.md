# 任务：Coding Lane 的最小 Verify Pack

## Context

仓库当前的确定性验证工具已经存在，但入口分散：

- `bun test`
- `tsc --noEmit`
- browser / live smoke
- issue `verify`

这会让 Agent 每次都临时猜“这一轮该跑什么”。

## Goal / Non-Goals

### Goal

- 把 coding lane 的验证入口收成统一最小包
- 明确 `test / types / e2e / custom verify` 的边界

### Non-Goals

- 不把全量 CI 流水线搬进 runtime
- 不对非代码任务强制套用同一套验证包

## Plan

- [x] 盘点现有 verify 入口与能力
- [x] 定义 coding lane verify pack 的最小结构
- [x] 明确何时跑 `test/types/e2e/custom`

## Acceptance Criteria

1. coding lane verify pack 边界清楚，不依赖临时猜测。
2. browser/live smoke 被当成正式 `e2e` 面，而不是“人工补充”。
3. 不新增重型测试编排平台。

## Notes

- 这张单只定义并收口最小验证包，不追求全仓测试统一平台。
- `e2e` 在本仓库里必须包含 browser 和真实 live smoke，不只是前端截图回归。
