# 任务：0362 最小实现切片

## Parent Task

- `tk0362`

## Context

现有 `.msgcode/evidence/` 用法已经有主链，但只覆盖 verify：

- `src/runtime/heartbeat-tick.ts` 会写 `<workspace>/.msgcode/evidence/<taskId>.json`
- `src/runtime/subagent.ts` 会读取 `<taskId>.json`，把最近一次失败摘要带回子代理上下文
- `src/cli/verify.ts` 会把 verify 结果写回 dispatch `result.evidence`，并把 `fullOutputPath` 写到 `evidenceRefs`
- `src/runners/bash-runner.ts` 在输出过大时会把完整 stdout/stderr 写到 `artifacts/logs/*.log`

现在缺的不是“再造一个证据系统”，而是：

1. `.msgcode/evidence/` 的写入逻辑还散在 `heartbeat-tick.ts` 私有函数里
2. `verify.ts` 没有直接写 `<taskId>.json`，CLI 路径与 heartbeat 路径不统一
3. 失败快照边界虽已在 `0362` 说明，但还没有最小实现入口去落 `verify-*.json`
4. provider / inbound 失败目前主要停在日志或异常里，尚无统一快照

按奥卡姆收口，第一刀只做 verify 失败切片，不同时开 provider / inbound。

## 已有证据

### 已有物理证据

- verify 任务级最后一次结果：`.msgcode/evidence/<taskId>.json`
- verify 大输出全文：`artifacts/logs/*.log`
- dispatch 级结构化结果：`.msgcode/dispatch/*.json` 里的 `result.evidence` 与 `evidenceRefs`

### 只有日志/上下文，还没有统一快照

- provider HTTP 非 2xx / 非法响应
- inbound payload 解析失败 / 路由失配 / 资源下载失败
- 非 verify 触发的 browser 失败

## Goal / Non-Goals

### Goal

- 抽出共享 evidence helper，结束 `heartbeat-tick.ts` 私有落盘
- 让 `verify.ts` 与 heartbeat 复用同一套 `.msgcode/evidence/` 写入
- 在 verify 失败时额外落一份受控快照：`verify-<timestamp>-<shortid>.json`
- 快照只保存高价值摘要和 artifact 引用，并执行 0362 约定的大小/保留边界

### Non-Goals

- 不做 provider 快照
- 不做 inbound 快照
- 不做 telemetry ledger
- 不碰 strong-state
- 不新增 server、数据库、控制面
- 不做全量 request/response dump

## Snapshot Shape

```json
{
  "kind": "verify",
  "taskId": "tk0362",
  "timestamp": "2026-03-21T00:00:00.000Z",
  "exitCode": 2,
  "command": "npm test -- --bail",
  "durationMs": 1200,
  "stdoutTail": "...",
  "stderrTail": "...",
  "artifactRefs": [
    "/abs/workspace/artifacts/logs/bash-output-xxx.log"
  ]
}
```

约束：

- 单文件默认上限 `64KB`
- 只保留 tail / excerpt，不落全量正文
- 大内容继续走 `artifacts/logs/*`，快照只留路径引用
- 快照窗口按 `0362`：最近 `20` 份或最近 `7` 天

## Why This Slice First

- 现成真相源已经在 verify 上最完整：`0269`、`heartbeat-tick.ts`、`bash-runner.ts` 都已到位
- `provider` / `inbound` 还没统一入口，直接一起做会逼出第二层抽象
- 先把共享 helper 收好，后续真要扩到 `provider` / `inbound`，也只是复用同一目录和裁剪规则，不需要新面

## Plan

- [ ] 抽共享 evidence helper，保留现有 `<taskId>.json` 合同
- [ ] 让 `verify.ts` 与 `heartbeat-tick.ts` 共用 helper
- [ ] verify 失败新增 `verify-*.json` 物理快照
- [ ] 增加快照裁剪与保留窗口清理
- [ ] 验证 `subagent.ts` 继续能消费 `<taskId>.json`，无合同漂移

## Acceptance Criteria

1. `.msgcode/evidence/` 仍是唯一失败快照目录，不新增别的 evidence 根目录。
2. `heartbeat-tick.ts` 与 `verify.ts` 使用同一套 helper，不再各写各的。
3. verify 失败会有受控 `verify-*.json` 快照；成功不额外落历史快照。
4. `<taskId>.json` 继续只代表最后一次任务级证据，`subagent.ts` 不需要改读取协议。
5. 不新增 server、数据库、控制面。
