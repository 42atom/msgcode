## Context

- 这张单只收第一刀：把 wake 从“提醒文本/内存事件”收成正式文件合同。

## Goal / Non-Goals

- Goal:
  - 固定 wake job 文件落点与最小字段
  - 固定 wake record 文件落点、状态与最小字段
  - 明确 `mode=now|next-heartbeat` 与 `latePolicy` 边界
- Non-Goals:
  - 不先做 heartbeat consume
  - 不先做 work capsule 组装
  - 不先做 startup catch-up
  - 不先做 vitals policy

## Plan

- [x] 定义 wake job 最小字段：`id/kind/schedule/mode/taskId/enabled/createdAt/updatedAt`
- [x] 定义 wake record 最小字段：`id/jobId/status/path/taskId/hint/scheduledAt/claimedAt/completedAt/failedAt/latePolicy`
- [x] 固定 wake record 状态：`pending/claimed/done/failed/expired`
- [x] 补最小回归：文件命名、字段边界、mode 边界

## Acceptance Criteria

1. 仓库内存在正式 wake job / wake record 文件合同。
2. `mode` 只允许 `now` / `next-heartbeat`。
3. wake record 状态边界固定，不靠内存补脑。
4. schedule 到 wake 的映射可以只靠文件重建。

## Notes

- 这张单只钉文件真相，不抢后续 consume/capsule 的决定权。
