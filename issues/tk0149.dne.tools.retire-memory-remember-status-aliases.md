## Context

`msgcode memory add/stats` 已经是正式 canonical 子命令，但 `memory remember/status` 仍在 CLI 解析期被静默映射到 canonical 主链，同时 `help-docs` 合同继续暴露 alias。

这让 `memory --help`、`help-docs --json`、direct invoke 三个面继续分叉：help 看不到，合同写 alias，执行还能成功。

## Goal / Non-Goals

- Goal: 退役 `memory remember/status` 历史 alias
- Goal: direct invoke 返回显式 retired 提示，不再静默归一化到 `add/stats`
- Goal: `help-docs` 合同只保留 canonical `memory add/stats`
- Non-Goals: 本轮不改 memory 正式业务实现
- Non-Goals: 本轮不改 `memory add/search/index/get/stats` 的参数与输出

## Plan

- [x] 创建 Plan 文档，冻结 `memory remember/status` 退役决策
- [x] 停止在 CLI 解析期静默映射 `memory remember/status`
- [x] 新增隐藏 retired 子命令并移除合同 alias 字段
- [x] 更新相关测试并执行 targeted tests、`npx tsc --noEmit` 与 `npm run docs:check`
- [x] 更新 issue notes 与 `docs/CHANGELOG.md`

## Acceptance Criteria

1. `msgcode memory --help` 仍只公开 `add/search/index/get/stats`
2. `msgcode memory remember ...` direct invoke 返回 retired 提示，不再成功落到 `memory add`
3. `msgcode memory status ...` direct invoke 返回 retired 提示，不再成功落到 `memory stats`
4. `help-docs --json` 导出的 memory canonical 合同不再带 `remember/status` alias
