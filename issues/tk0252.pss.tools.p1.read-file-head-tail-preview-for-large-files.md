## Context

- `tk0251` 已把 `tool_result` replay 爆窗收住
- 真实 Feishu smoke 里，`read_file` 对大文件已成功执行，但模型仍拿不到尾部事实，最后一行问题会答错
- 当前 large `read_file` 只读头部预览，尾部完全丢失

## Goal

- 让 large `read_file` 截断结果同时暴露 head 和 tail
- 让 replay 预览在 512 chars 内仍保留尾部事实

## Non-Goals

- 不新增 `read_file tail` 新参数
- 不重做 file tool 接口
- 不把文件阅读问题转成 bash-only

## Goal / Non-Goals

- Goal:
  - 大文件时 `read_file` 返回 head + tail 结构
  - previewText 保留 head/tail 且长度受控
- Non-Goals:
  - 不扩新的文件读取 API
  - 不改小文件 inline 行为
  - 不引入新的上下文控制层

## Plan

- [x] 大文件读取时同时读取 head/tail 片段
- [x] 截断 preview 改为结构化 head/tail 预览
- [x] 补专属回归

## Acceptance Criteria

1. large `read_file` 时，返回内容包含 head 和 tail。
2. `buildReadFilePreviewText()` 在截断模式下仍保留 tail 事实。
3. preview 长度可直接进入 replay，不再把 tail 挤掉。

## Notes

- 这条修的是工具合同缺口，不是模型温度问题。
- 最薄办法不是加参数，而是把现有截断预览从“只有头”改成“头尾都给”。

## Result

- large `read_file` 现在会读取 head + tail 两段
- 截断 preview 改成 compact `head/tail` 结构
- 尾部事实可以进入 tool replay
