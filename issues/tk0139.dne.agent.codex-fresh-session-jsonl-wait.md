## Context

`msgcode subagent run codex --watch` 在新建 tmux/Codex 会话时，过早要求 `~/.codex/sessions/**/rollout-*.jsonl` 已经存在。真实 `test-real` 验收里，这会导致：

- fresh session 刚启动时直接报 `SUBAGENT_DELEGATE_FAILED`
- 任务文本根本没被发送到 Codex
- 只有等会话手动热起来后，后续任务才会恢复正常

这不是 Codex 能力不足，而是 delegate 侧在新会话上抢跑了。

## Goal / Non-Goals

- Goal: Codex fresh-session 下，delegate 前允许短暂等待 JSONL 就绪。
- Goal: 真实 `test-real` 工作区里，最小文件创建与项目级贪吃蛇 HTML 游戏任务都能通过 `--watch` 完成。
- Goal: 不新增新控制层，只修正 delegate 时序。
- Non-Goals: 不改 `subagent` 命令协议；不实现队列/调度器；不改 claude-code 路径。

## Plan

- [x] 在 `src/tmux/responder.ts` 的 codex 分支里，把 JSONL 查找改为短等待轮询
