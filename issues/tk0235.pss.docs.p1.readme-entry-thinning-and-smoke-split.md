## Context

根 `README.md` 当前同时承担产品入口、上手手册、维护者约束和渠道 smoke 基座四种角色，结果是首页信息流倒挂：

- 读者还没跑起来，就先被长段哲学和维护规则拦住
- 真机 smoke 基座混在首页，和普通使用者无关
- `README.md`、`docs/README.md`、`test/README.md` 三处重复同一套 smoke 细节

## Goal / Non-Goals

- Goal:
  - 把根 `README.md` 收口成真正的第一入口：先跑起来，再理解主链
  - 吸收这次 review 的三点：减轻哲学重量、拆开开发规约与核心用法、补最小主管道图
  - 把飞书真机 smoke 基座拆成单独文档，并让 `docs/README.md` 与 `test/README.md` 只保留入口链接
- Non-Goals:
  - 不重写产品定位
  - 不新增新的架构控制层
  - 不新开第二套 smoke 协议
  - 不把 README 变成维护手册
