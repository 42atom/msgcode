# 任务：失败证据快照边界

## Parent Task

- `tk0361`

## Context

现有证据面已经有两条主线：

- `verify.ts` 会把命令、退出码、stdout/stderr tail 与 `fullOutputPath` 组合成最小验证证据
- `heartbeat-tick.ts` 已把任务级验证结果写到 `<workspace>/.msgcode/evidence/<taskId>.json`

缺口不在“完全没有 evidence”，而在“哪些失败值得落第二份快照还没钉死”。

如果这一步不先收口，后面最容易走偏成：

- 为了排障方便去做全量 payload dump
- 把日志、证据、遥测混成同一层
- 让 `.msgcode/evidence/` 长成无限增长的垃圾桶

## Goal / Non-Goals

### Goal

- 定义失败快照的准入边界
- 钉死路径、命名、大小上限和保留窗口
- 明确显式 debug 下允许扩展到什么程度

### Non-Goals

- 不写代码
- 不做 telemetry ledger
- 不做 strong-state
- 不做 UI
- 不做全量 payload dump

## 结论

默认只为三类失败落快照：

1. 难复现
2. 跨边界
3. 排障高价值

没同时命中这些特征的失败，优先留在现有命令 tail、结构化日志或原生错误里，不额外落盘。

## Worth Saving

### 1. verify / shell / browser 失败

- 命令退出码非 0，且现有 stdout/stderr tail 足以定位不了问题时
- 浏览器自动化失败，但仅靠测试报告或终端尾部无法复现时
- 保留内容：
  - command
  - exitCode
  - durationMs
  - clipped stdout/stderr tail
  - 相关 artifact / output path

### 2. 模型或外部 API 跨边界失败

- provider 非 2xx
- 返回体结构不符，导致运行时无法继续
- 请求已离开本地，失败现场难再重放
- 保留内容：
  - provider / route
  - http status
  - requestId 或等价 trace id
  - 经过裁剪和脱敏的 body excerpt
  - 错误码 / 错误消息

### 3. 入站协议或路由失败

- Feishu 等入站 payload 解析失败、签名校验失败、路由失配
- 正常日志里没有足够现场时才允许补快照
- 保留内容：
  - source
  - route / event kind
  - 脱敏后的 payload excerpt
  - 解析错误

## Not Worth Saving

- 普通成功结果
- 可从现有日志直接复原的本地失败
- 纯业务阻塞、普通等待态
- 大体积原始请求/响应全量内容
- 与排障无关的模型长文本输出

## Path And Naming

- 根目录固定：`<workspace>/.msgcode/evidence/`
- 任务级 verify 证据继续复用现有文件：`<taskId>.json`
- 额外失败快照统一命名为：
  - `<kind>-<YYYYMMDDTHHMMSSZ>-<shortid>.json`
- `kind` 只允许：
  - `verify`
  - `runner`
  - `provider`
  - `inbound`

## Size And Retention

- 默认单文件上限：`64KB`
- 显式 debug 下单文件上限：`256KB`
- 默认保留窗口：最近 `20` 份快照或最近 `7` 天，先命中哪个就删最老的
- 同一任务的 `<taskId>.json` 继续只保留最后一次任务级证据，不演化成历史仓

## Explicit Debug

只有显式 debug 开关打开时，才允许额外落这些内容：

- 更长的 stdout/stderr tail
- 更长的 provider / inbound excerpt
- 更多 artifact refs

即便在 debug 下，也不允许：

- 全量 request / response dump
- 未脱敏 header / token / cookie
- 二进制大 payload 直接入盘

## Guardrails

- `.msgcode/evidence/` 只放失败现场，不放统计账本
- 快照不是新的状态真相源，只服务排障和下一轮修复
- 大内容一律裁剪；需要全量原文时，只保存外部 artifact 路径引用
