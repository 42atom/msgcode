## Context

`ghost_*` 已经作为正式桌面能力面接入 Agent，但仓库里仍保留 legacy `desktop` 的三类主链入口：

- Tool Bus 正式工具 `desktop`
- `/desktop rpc ...` slash 路由
- `/tool allow` 中的遗留显式工具 `desktop`

这让系统继续同时维护两套路径，也让 `msgcode-desktopctl` 旧桥仍占着现役代码结构。按照冻结设计，`desktop` 不应再是正式能力面。

## Goal / Non-Goals

- Goal: 让现役桌面能力面只剩 `ghost_*`，删除 legacy `desktop` 的正式工具与 slash 主链。
- Goal: 让 parser/Tool Bus 对 `desktop` 回到真实不存在状态，不再维护遗留 compat 入口。
- Goal: 保留历史文档与代码存档到 `.trash`，方便回查。
- Non-Goals: 不实现新的 desktop provider/platform。
- Non-Goals: 不改 `ghost_*` 的工具面与 skill。
- Non-Goals: 不清理所有历史文档发布记录，只处理现役帮助与主链文案。

## Plan

- [x] 建立 issue / plan，明确退役边界与回滚口径。
- [x] 删除 `desktop` 的 Tool Bus / ToolName / manifest 主链。
- [x] 删除 `/desktop` slash 识别、分发与 `/tool allow` 遗留显式入口。
- [x] 将 legacy desktop route/runner 实现移到 `.trash`，不再留在现役源码目录。
- [x] 更新 README / CHANGELOG / 直接相关测试并完成验证。

## Acceptance Criteria

1. `desktop` 不再是 `ToolName`、`TOOL_MANIFESTS` 或 Tool Bus 的正式工具。
2. `/desktop` 不再是可识别 slash 命令；系统对它返回通用 unknown command 行为。
3. `/tool allow` 不再列出或接受 `desktop`。
4. 现役桌面能力面只剩 `ghost_*`；相关测试与文档同步。
5. 通过直接相关 `bun test`、`npx tsc --noEmit`、`npm run docs:check`。
