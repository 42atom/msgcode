# 任务：App Bundle Host Assembly

## Context

`build artifact` 不是最终 app。

它还需要被一个极薄宿主壳承接，装成用户可启动的本地 Desktop App bundle。

这一步必须单独收口，因为它同时受 3 条边界约束：

- `pl0274`：Electron 只做宿主
- `rf0000`：Electron 不是第二主脑
- `rf0001`：当前 UI 约束仍然存在

所以这里缺的不是“再讨论要不要 Electron”，而是把宿主装配边界写薄、写死。

## Goal / Non-Goals

### Goal

- 定义 app bundle host assembly 的输入只来自 build artifact
- 定义宿主壳最小职责
- 定义 app bundle 输出面与启动面
- 定义 app bundle 与 dmg 链的切口

### Non-Goals

- 不重定义 build artifact 结构
- 不设计签名、公证、notarization 流程
- 不扩授权、更新、跨平台
- 不写 Electron 实现代码

## Plan

- [ ] 明确 app bundle 只能消费 artifact，不直接消费源码树
- [ ] 明确宿主壳只负责哪些最小职责
- [ ] 明确 app bundle 的最小输出与验收
- [ ] 明确为什么这一步不能和 dmg 链合并

## Acceptance Criteria

1. 能明确说明 app bundle 的输入只来自 `tk0371` 所定义的 artifact。
2. 能明确说明宿主壳只做 host assembly，不承接第二主脑职责。
3. 能明确区分 app bundle 完成与 dmg 分发完成不是一回事。
4. 文档不扩到授权中心、更新中心、跨平台安装器。

## Notes

- 这张单是 `tk0370 / pl0362` 的第 2 个子产物切片。
- 它的目标是把“能启动的 app bundle”从 artifact 与 dmg 之间单独切出来。
