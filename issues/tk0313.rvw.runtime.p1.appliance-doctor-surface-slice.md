# 任务：appliance doctor 读面切片

## Context

Settings 页已经冻结了 `诊断` 板块。

当前现状：

- 现有 probe 主链已经成立
- `hall` 里已经包含一份 runtime 摘要
- 但还没有单独的 doctor 只读出口

如果直接拿 `hall` 顶上，会把门厅和设置页混成一个大接口。

## Goal / Non-Goals

### Goal

- 增加最小 `msgcode appliance doctor --workspace <...> --json`
- 输出：
  - runtime summary
  - runtime categories
  - appVersion / configPath / logPath

### Non-Goals

- 不新增 probe
- 不改 probe 聚合规则
- 不新增 doctor 状态机
- 不做设置页整页总线

## Plan

- [x] 复用现有 `runAllProbes()`
- [x] 抽一层最薄 runtime surface
- [x] 增加 `appliance doctor`
- [x] 增加黑盒测试

## Acceptance Criteria

1. `appliance doctor` 能稳定输出设置页诊断读面。
2. 不新增第二套 probe 逻辑。
3. `hall` 与 `doctor` 共享同一份 runtime 事实，不产生分叉。

## Notes

- 这刀只做 read surface。
- 若以后需要更细的诊断页动作，再单开切片，不在这里顺手做。
