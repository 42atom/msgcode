# 任务：Desktop Main Window Readonly Thread Surface Mainline

## Context

Desktop App 下一刀只开一条最薄主线：

- 左栏接 `workspace-tree`
- 正文和右栏接 `thread`
- UI 只保留 `selectedWorkspace` / `selectedThreadId` / `loading-error`

不做写面，不开第二状态层，不重做壳。

## Goal / Non-Goals

### Goal

- 为 Desktop App 主窗口开一张现役执行单
- 只联通 `workspace-tree` 和 `thread` 两个正式读面
- 把 UI 状态收死到 `selectedWorkspace / selectedThreadId / loading-error`
- 正式 UI 落地到 `src`，不再继续堆 `ui-protype`
- 明确这刀不做的旁支

### Non-Goals

- 不做发消息
- 不做新建聊天
- 不做 archive
- 不做 settings
- 不做美学重做
- 不做 Electron 大壳
- 不新增 UI 缓存层
- 不直接 import runtime 内部模块

## Plan

- [x] 确认主窗口只消费 `workspace-tree` 和 `thread` 两个正式读面
- [x] 左栏只承载工作区树选择，不再挂 archive / new chat / settings 入口
- [x] 正文和右栏只承载当前选中线程的只读展示
- [x] UI 瞬时状态只保留 `selectedWorkspace / selectedThreadId / loading-error`

## Acceptance Criteria

1. 仓库里存在一张现役任务单，专门承载主窗口只读线程面联通。
2. 文档明确说明为什么 `tk0263 / tk0293 / tk0355 / tk0356 / rf0001 / pl0211 / main.html` 不能直接承载这刀职责。
3. 范围被收死到 `workspace-tree + thread` 两个正式读面。
4. UI 瞬时状态被收死到 `selectedWorkspace / selectedThreadId / loading-error`。
5. 非范围明确排除发消息、新建聊天、archive、settings、美学重做和 Electron 大壳。

## Notes

### 为什么现有文件不能直接承载这个职责

### `tk0263`

它是方向单。

它回答：

- 为什么需要 App shell
- 为什么 Electron 只能做薄壳
- Main UI 最小面大概包含什么

它不回答：

- 主窗口这刀到底接哪两个正式读面
- UI 只保留哪几个瞬时状态
- 哪些原型元素这一刀必须删出范围

继续往 `tk0263` 里塞，会把方向判断和现役执行混成一张单。

### `tk0293`

它是数据合同单。

它回答：

- 左中右各读什么文件
- 哪些区域缺真相源必须降级

它不回答：

- Desktop App 主窗口这一刀的执行顺序
- 只读联通时 UI 该砍到什么程度
- 哪些功能明确不做

而且 `tk0293` 的范围更宽，包含 work status、日历、people、邻居降级口径。直接拿来承载这刀，会把“合同”误当“执行单”。

### `tk0355`

它只承载左栏读面：

- `msgcode appliance workspace-tree --json`

它不承载主窗口正文和右栏，也不约束 UI 状态。

### `tk0356`

它只承载选中线程读面：

- `msgcode appliance thread --workspace ... --thread-id ... --json`

它不承载左栏树，也不承载 Desktop App 主窗口收口。

### `rf0001`

它是参考，不是执行单。

而且里面还保留了：

- settings
- new chat
- thread / new chat 边界参考

这份文档的职责是记录当前 UI 约束，不是替这刀做任务拆解。

### `pl0211`

它是主线导航，不是真相源，也不是单任务执行计划。

它可以指向这刀，但不能代替这刀。

### `ui-protype/main.html`

它是原型，不是真相源，也超出本刀范围。

当前原型里仍有：

- settings
- archive
- new chat
- send
- base / neighbor 视图
- 顶栏状态簇

如果直接拿它承载这刀，范围一定会再次膨胀。

正式 UI 这刀只认 `src` 里的落点，不回头把现役实现堆回 `ui-protype`。

## Links

### 输入

这刀只承认两个正式输入：

1. `msgcode appliance workspace-tree --json`
2. `msgcode appliance thread --workspace <...> --thread-id <...> --json`

UI 端不直接读 `src/runtime/*`，不自己扫工作区文件。

### 范围

只做 3 件事：

1. 左栏接 `workspace-tree`
2. 选中线程后，正文和右栏同读 `thread`
3. UI 状态只保留：
   - `selectedWorkspace`
   - `selectedThreadId`
   - `loading-error`

### 非范围

明确不做：

1. 发消息
2. 新建聊天
3. archive
4. settings
5. 美学重做
6. Electron 大壳
7. 新的 UI 缓存层
8. 直接 import runtime 内部模块

### 一句话

这刀不是再谈方向，也不是再补 runtime。

这刀只把主窗口读面接通，并把 UI 收成只读线程面。

### 当前 review 口径

这张单的 readonly 主线已经实际落地，并由：

- `tk0381` 承接正式 `src` 骨架
- `tk0391` 承接 host bridge
- `tk0403` 把桌面 thread 从 readonly 推到最薄 writable

所以它不再是现役 `tdo`，现在只等 review 确认这条阶段主线已经完成并被后续 writable 主线接续。
