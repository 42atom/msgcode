## Context

`msgcode skill` 已经是显式 retired compat shell，但 `skills` 仍通过 CLI 解析期归一化和 commander alias 继续映射到同一实现。

虽然它不是第二套逻辑，但仍保留了一个隐藏复数根词，和当前“显式 canonical / 显式 retired，不保留多余词法兼容层”的方向不一致。

## Goal / Non-Goals

- Goal: 退役 `skills` 复数 alias 根入口
- Goal: 停止 `skills -> skill` 静默映射与 commander alias
- Goal: 保持 `msgcode skills ...` direct invoke 仍返回显式 retired 提示
- Non-Goals: 本轮不改 `msgcode skill` retired 文案主旨
- Non-Goals: 本轮不恢复任何 skill CLI 正式能力

## Plan

- [x] 创建 Plan 文档，冻结 `skills` alias 退役决策
- [x] 停止在 CLI 解析期静默映射 `skills -> skill`
- [x] 将 `skills` 从 commander alias 改为独立 retired compat root
- [x] 更新相关测试并执行 targeted tests、`npx tsc --noEmit` 与 `npm run docs:check`
- [x] 更新 issue notes 与 `docs/CHANGELOG.md`

## Acceptance Criteria

1. `msgcode --help` 仍不公开 `skill/skills`
2. `msgcode skill ...` 保持 retired 提示
3. `msgcode skills ...` 不再通过 alias/归一化偷偷映射，但 direct invoke 仍返回显式 retired 提示
