## Context

本地 `main` 已完成 merge，但仓库自带的正式发布门槛 `npm run test:all` 仍未通过，当前阻断有两类：

1. `npm run bdd` 失败 3 条，其中 2 条来自已退役的 `/persona` 场景，和当前仓库明确的 persona 退役锁冲突。
2. `npm run docs:check` 失败一批，根因不是单个 issue 漏填，而是 `scripts/check-doc-sync.ts` 的校验口径比当前 AGENTS 协议更严，尤其把 `plan_doc` / `links` / `docs/tasks` 回链都当成硬门。
3. `/model codex` 在 `local-only` 下的拒绝提示没有同步到当前协议里仍然保留的 `/policy egress-allowed` 兼容口径，BDD 也因此有一条真实失败。

## Goal / Non-Goals

### Goal

- 让 `npm run bdd` 与当前真实功能边界一致
- 让 `scripts/check-doc-sync.ts` 只校验当前协议明确要求的门槛
- 修正 tmux/local-only 拒绝提示词，使其与新旧协议兼容文案一致
- 让 `npm run test:all` 回到可通过状态

### Non-Goals

- 不恢复 persona 功能
- 不批量补写几十份历史 issue/task 文档
- 不新增新的文档系统或兼容层
- 不修改当前 backend lanes 命令协议
