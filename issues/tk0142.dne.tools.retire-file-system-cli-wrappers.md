## Context

当前 `msgcode` 仍公开两组没有新增真实能力的 CLI 包装层：

- `msgcode file find/read/write/delete/move/copy`
- `msgcode system info/env`

它们本质上是在把原生 Unix/macOS shell 已经稳定提供的文件与系统能力，再包一层私有命令、私有错误码和私有 JSON 合同。这会带来三类偏差：

- LLM 与用户要多记一套 `msgcode file/system` 方言，增加 token 与认知负担
- 原生 `stderr/exit code` 被 envelope 化后，物理反馈更钝
- skill / prompt / help-docs 会继续把“本地文件与系统壳操作”描述成 msgcode 的正式能力，而不是 Unix 的正式能力

## Goal / Non-Goals

- Goal: 将 `file` 与 `system` 从公开 CLI 主链与 `help-docs` 正式合同移除
- Goal: 保留最薄 retired compat shell，显式引导回原生 shell / 原生工具
- Goal: 更新 prompt、AGENTS 与 runtime file skill，写死“本地文件/系统操作直接用原生能力”
- Non-Goals: 本轮不改 `browser`、`memory`、`thread`、`todo`、`schedule` 等桥接型 CLI
- Non-Goals: 本轮不重构 msgcode 原生工具总线
- Non-Goals: 本轮不删除历史归档文档，只改现役主链与当前说明书
