## Context

- vitals 在这条主线上只能是投影，不是新的裁判层；它只该影响消费策略。

## Goal / Non-Goals

- Goal:
  - 给 wake consume 接上 `defer / degrade` 两级策略
  - 固定 vitals 只读真相源投影
  - 明确高压下哪些 wake 可推迟，哪些只能降级
- Non-Goals:
  - 不做 kill/restart 自动动作
  - 不改 issue / dispatch / wake record 真相
  - 不引入单独 vitals 控制面

## Plan

- [x] 定义 wake consume 上的 vitals snapshot 与策略映射
- [x] 锁 Phase 1 只做 `defer / degrade`
- [x] 明确哪些 wake 可延后、哪些只能降级提示
- [x] 补 BDD/回归：高压下策略变化但真相文件不漂移

## Acceptance Criteria

1. 高压下 wake consume 会 `defer / degrade`，而不是硬顶到底。
2. vitals 只影响消费策略，不抢真相源。
3. Phase 1 不引入 kill/restart 自动动作。
4. 行为可被 BDD/回归锁住，不靠叙事描述。
