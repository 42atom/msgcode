# 任务：负向 Smoke 的真实错误通过口径

## Context

当前主线已经明确收口到：

- 暴露真实能力
- 工具返回真实结果
- 错误优先进入模型上下文
- 只保留安全 / 预算 / 物理边界的硬拦截

在这个前提下，负向 smoke 若继续把“入口成功拦截”当成默认通过标准，就会和主线设计相冲突。

## Goal / Non-Goals

### Goal

- 定义负向 smoke / BDD 的正式通过口径
- 明确哪些失败应当：
  - 进入真实执行
- 返回真实底层错误
- 回给模型
- 明确只有哪三类边界允许入口 fail-closed

### Non-Goals

- 不新增新的 guard / interceptor
- 不重写工具总线
- 不扩大为完整测试框架改造

## Plan

- [x] 盘点当前负向测试里仍假定“入口拦截即通过”的断言
- [x] 定义新的默认通过口径：真实错误进入日志与模型上下文
- [x] 列出仅保留入口 fail-closed 的三类边界
- [x] 补最小合同测试或 smoke 说明

## Acceptance Criteria

1. 非硬边界失败的默认通过条件改为：真实执行发生，真实错误返回，模型可见。
2. 仅安全 / 预算 / 工具启动前物理不可达允许入口 fail-closed。
3. 至少有一组明确例子，例如：
   - 不存在命令 -> `command not found`
   - 不存在文件 -> `ENOENT`
4. 测试口径不再把“拦截成功”当成默认正确行为。

## Notes

- 这张单收的是测试与验收口径，不是再造权限层。
- 核心判断标准是“是否忠实暴露真实错误”，不是“系统是否更像一个守门员”。
