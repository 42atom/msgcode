# 任务：appliance profile mutation 实现切片

## Context

`0322` 已经把边界钉死：

- 当前只允许 `我的资料` 可写
- 其他 settings 分组继续只读
- UI 不得直接改真相文件

现在缺的只是最小正式写入口。

## Goal / Non-Goals

### Goal

- 增加 `msgcode appliance set-profile --workspace ...`
- 只支持四个显式字段：
  - `--name`
  - `--organization-name`
  - `--city`
  - `--soul`
- 写完后回读 profile surface 返回
- 若中途写失败，诚实返回已落盘 `changedFiles`

### Non-Goals

- 不做 `set-general`
- 不做 `set-capabilities`
- 不做通用 `set-setting`
- 不承诺跨文件事务
- 不做回滚器

## Plan

- [x] 在 `src/cli/appliance.ts` 增加 `set-profile`
- [x] 只写 `config.json` / `ORG.md` / `SOUL.md`
- [x] 写完回读 `workspace-profile`
- [x] 补部分成功回归锁
- [x] 做一轮圆桌复核

## Acceptance Criteria

1. `set-profile` 是当前唯一 settings mutation 入口。
2. 写后返回 `changedFiles + profile`，不靠 UI 自己拼成功态。
3. 中途写失败时，返回 error，并诚实保留已落盘文件列表。
4. 未提供任何 mutation 参数时返回真实错误。

## Notes

- 这刀故意不做事务。
- 重点是诚实，不是假装全成或全失败。
