# 任务：线程归档与 /clear 边界合同

## Context

当前线程真相源已经很清楚：

- 活跃线程落在 `.msgcode/threads/`
- `/clear` 只清短期上下文并开启新线程

所以 archive 不该去篡改 `/clear` 的含义。

更稳的做法是单独引入 `archived-threads/`，把 archive 收成一次文件移动。

## Goal / Non-Goals

### Goal

- 定义 `<workspace>/.msgcode/archived-threads/`
- 定义 thread markdown 的 archive/restore
- 定义 session jsonl 第一版不搬动
- 定义 `/clear` 与 archive 的清晰边界

### Non-Goals

- 不做 session archive
- 不做自动归档
- 不做历史摘要二次生成

## Plan

- [ ] 定义 archived-threads 路径
- [ ] 定义 thread markdown 的 archive / restore
- [ ] 定义 session jsonl 第一版保持不动
- [ ] 定义 `/clear` 与 archive 的边界说明

## Acceptance Criteria

1. active 线程和 archived 线程的目录边界明确。
2. `/clear` 不再与 archive 混淆。
3. restore 仍然只是文件移动，不新增中间层。

## Notes

- 第一版 archive 单位先只认 thread markdown 文件
- archive 页面若要回看原始会话，可通过 thread front matter 里的 `chatId` 去找 session jsonl
- 同名冲突继续按 fail-closed 处理
