## Context

`getToolsForLlm()` 当前仍会在 workspace 显式配置了 `tooling.allow` 后，偷偷再并上 `[read_file, bash, help_docs]`。这导致“配置看到的工具面”和“模型真实拿到的工具面”不是同一套真相源，也让 `tooling.allow` 失去精确控制意义。

## Goal / Non-Goals

### Goal

- 让 `tooling.allow` 成为 `getToolsForLlm()` 的唯一真相源
- 仅在 workspace 缺少 `tooling.allow` 时回退到 `DEFAULT_WORKSPACE_CONFIG`
- 更新相关回归锁，删除对隐藏基线的依赖

### Non-Goals

- 不调整 `DEFAULT_WORKSPACE_CONFIG["tooling.allow"]` 默认值
- 不改变 `vision/mem` 的默认 suppress 策略
- 不新增新的工具层、恢复层或 help fallback

- 相关测试、`tsc`、`docs:check` 全绿
