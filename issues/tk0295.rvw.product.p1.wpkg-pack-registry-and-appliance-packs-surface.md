# 任务：wpkg 注册表与主界面 packs 可见面

## Context

`0290` 已经定义了：

- `wpkg` 是带 `manifest.json` 的 zip 包
- 安装后写 `packs.json / sites.json`

但现状还是：

- `appliance hall` 里的 `packs` 为空
- 只有 `sites` 有最小合同

所以需要先补一张更薄的实施前切片，只回答：

- pack 安装后怎么注册
- 主界面怎么读到它

## Goal / Non-Goals

### Goal

- 定义 pack 安装目录约定
- 定义 `packs.json` 最小形态
- 定义 `hall.packs` 的最小消费面
- 定义“默认内置 / 用户安装”分组口径

### Non-Goals

- 不实现 pack 安装器
- 不实现 agent skill 发现
- 不实现授权 / 支付 / 市场
- 不实现包下载源

## Plan

- [ ] 固定 pack 安装目录与最小落盘约定
- [ ] 固定 `packs.json` 最小字段
- [ ] 固定主界面 `packs` 读取合同
- [ ] 固定“默认内置 / 用户安装”分组口径
- [ ] 明确哪些能力继续留到后续切片

## Acceptance Criteria

1. `wpkg` 安装后最少写什么，被讲清楚。
2. 主界面如何看见已安装 pack，被讲清楚。
3. 当前不做的事情被明确留后，不借这张单偷长更多层。

## Notes

- 这张单是 `0290` 的第一条实施前切片。
- 当前只收口 pack 注册表和可见面，不碰安装器本体。
- `appliance.ts` 现有 `packs: []` 缺口已经被实现切片补上，细节见 `rp0301`。
