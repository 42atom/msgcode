## Context

`browser gmail-readonly` 已经退出公开命令面和 `help-docs` 正式合同，但当前仓库仍保留一条真实可执行的 compat 入口：

- `msgcode browser gmail-readonly ...`
- `msgcode browser-gmail-readonly ...`

这与“历史壳坚决封死”的收口原则冲突。既然 Gmail 只读验收已退出主回归和活跃任务索引，这条 compat 入口也不该继续执行真实业务流。

## Goal / Non-Goals

- Goal: 退役 `browser gmail-readonly` 与 `browser-gmail-readonly` 兼容入口
- Goal: direct invoke 返回显式 retired 提示
- Goal: 更新 browser 合同测试与 changelog
- Non-Goals: 本轮不改 browser core 正式命令
- Non-Goals: 本轮不删除 `src/browser/gmail-readonly.ts` 历史文件

## Plan

- [x] 创建 Plan 文档，冻结 Gmail readonly compat 入口退役决策
- [x] 将 `src/cli/browser.ts` 中 compat 入口改为 retired shell
- [x] 保留 CLI 解析期映射，但 direct invoke 只返回 retired 提示
- [x] 更新相关测试并执行 targeted tests、`npx tsc --noEmit` 与 `npm run docs:check`
- [x] 更新 issue notes 与 `docs/CHANGELOG.md`

## Acceptance Criteria

1. `msgcode browser gmail-readonly --help` 不再执行历史业务流，只显示 retired 提示
2. `msgcode browser-gmail-readonly --help` 显示 retired 提示
3. `help-docs --json` 与 `browser --help` 继续不暴露 `gmail-readonly`
