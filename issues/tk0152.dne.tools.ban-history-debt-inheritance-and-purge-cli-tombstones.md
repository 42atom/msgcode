## Context

前几轮虽然已经把一批历史命令从公开合同和 prompt 主链移除，但代码里仍残留大量“体面墓碑”：

- retired compat root
- hidden retired subcommand
- 专门给死命令准备的提示文案与 envelope

这类实现不再提供真实能力，却继续占据命令树和代码结构，属于“用面向对象的方式继承历史债务”。

## Goal / Non-Goals

- Goal: 把“不要用面向对象的方式去继承历史债务”写进仓库规则
- Goal: 删除 retired CLI tombstone，让已死命令直接返回 parser 原生 `unknown command`
- Goal: 更新测试口径，从 retired 提示改成 unknown command
- Non-Goals: 本轮不恢复任何已退役能力
- Non-Goals: 本轮不重写历史 issue / plan，只追加新的真相源

## Plan

- [x] 将新铁律写入 `AGENTS.md`
- [x] 从 `src/cli.ts` 删除 retired root 装载逻辑，只保留现役命令树
- [x] 删除 active 模块中的 hidden retired subcommand / compat root 实现
- [x] 将纯 tombstone 文件移出 `src/cli/`
- [x] 更新相关测试并执行 targeted tests、`npx tsc --noEmit` 与 `npm run docs:check`
- [x] 更新 issue notes 与 `docs/CHANGELOG.md`

## Acceptance Criteria

1. 已死命令不再拥有专门 `Command` 节点或 compat shell
2. `msgcode file/system/web/media/skill/skills/jobs/gen-image/gen-audio/browser-gmail-readonly` 直接报 unknown command
3. `msgcode browser gmail-readonly`、`msgcode memory remember/status` 直接报 unknown command
4. `AGENTS.md` 明确写入“不要用面向对象的方式去继承历史债务”
