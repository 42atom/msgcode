# 任务：Local Audio Output and Playback Policy Baseline

## Context

当前语音支线先要把本机音频输出站住，用于：

- TTS 输出
- BDD 验证播放行为
- 后续 settings / runtime 的最小统一口径

这一步不需要“播放管理器”，只需要一个薄执行口和清楚的中断策略。

## Goal / Non-Goals

## Goal

- 定义本机 `output_channel` 的最小真相
- 定义播放策略：`replace / queue / drop`
- 定义最薄执行口：`play / stop / status`
- 固定默认策略：TTS 回复默认 `replace`

## Non-Goals

- 不做输入通道
- 不做 HomePod bridge
- 不做统一音频平台
- 不写音频实现代码
- 不引入播放管理器/调度器/supervisor

## Baseline

### output_channel

- `system-default`
- `audio-device:<device-id>`
- `none`

### playback_policy

- `replace`
- `queue`
- `drop`

默认：

- TTS / agent 回复 = `replace`

### 最薄执行口

- `play(path, outputChannel, policy)`
- `stop()`
- `getStatus()`

## UI Placement

- 首页只显示：
  - `语音外放设备：系统音响/耳机/HomePod/静音`
- `settings.html -> general` 新增 `会话音频` 区块，包含：
  - `语音接收设备`
  - `语音外放设备`
  - `播放策略`
- 第一刀里 `语音接收设备` 默认 `系统默认`

## Plan

1. 先把 `output_channel` 与 `playback_policy` 收成 settings / CLI / runtime 共用口径
2. 明确 `replace` 是默认策略
3. 明确后续实现只允许先做薄执行口，不允许先长出播放管理器

## Acceptance Criteria

1. 文档明确存在 `output_channel` 真相
2. 文档明确存在 `replace / queue / drop` 三种策略
3. 文档明确默认策略为 `replace`
4. 文档明确后续只做 `play / stop / status` 最薄执行口

## Notes

- 这张单是行为口径，不是实现单
- 先把输出做稳，再讨论输入和 HomePod
