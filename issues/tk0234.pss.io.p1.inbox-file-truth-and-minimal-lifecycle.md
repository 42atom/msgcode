# Goal

给 I/O 系统补一个人类可读、可巡检、带生命周期的原始请求面：

- `.msgcode/inbox/`
- 文件名可直接 `ls`
- 内容保留原始请求详情

## Proposed Shape

文件名模式：

- `rq0001.new.<transport>.<slug>.md`
- `rq0001.triaged.<transport>.<slug>.md`

最小 front matter：

```yaml
---
owner: agent
assignee: codex
reviewer: user
transport: feishu
request_id: <transport-side-id>
related_task: ""
created_at: <iso>
---
```

## Minimal Lifecycle

- `new`
  - 原始请求刚落盘
- `triaged`
  - 已拆成任务，或已确认无需拆任务

## Non-Goals

- 不做 inbox 平台
- 不做自动优先级系统
- 不做 dispatch/wake/memory 文件名重构
- 不做复杂状态机

## Acceptance

1. 原始 I/O 请求可以落成 inbox 文件。
2. inbox 文件能从 `new` 推进到 `triaged`。
3. 文件名本身足够支持 `ls` 巡检。
4. 不改变 dispatch / wake / memory 现有协议面。
