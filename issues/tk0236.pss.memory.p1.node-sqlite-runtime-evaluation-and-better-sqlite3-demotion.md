# Context

- 今天的本机探针已确认：
  - `node:sqlite` 可用
  - `DatabaseSync(..., { allowExtension: true })` 可加载 `sqlite-vec`
  - `RETURNING` / `readOnly` / 文件库 `WAL` / `busy_timeout` 可工作
- 这张单开始时，仓库仍直接依赖：
  - `src/memory/store.ts` -> `better-sqlite3 + sqlite-vec`
  - `src/cli/todo.ts` -> 历史 `todo.db` 导入路径
- `node:sqlite` 当前仍带 `ExperimentalWarning`，因此不宜无证据直接切主链。

## Goal / Non-Goals

- Goal:
  - 评估并尽量完成 `node:sqlite` 对 `better-sqlite3` 的替换
  - 减少一个 native 依赖，同时不破坏 memory 主链
  - 让 memory store 与历史 `todo.db` 导入路径收口到同一 SQLite 运行时适配层
- Non-Goals:
  - 不把 SQLite 路径 async 化
  - 不重写 memory schema
  - 不顺手扩大到别的存储抽象层

## Plan

1. [x] 新增单一 sqlite 运行时适配层
2. [x] 让 `src/memory/store.ts` 通过适配层打开数据库，并保留 `sqlite-vec`
3. [x] 让 `src/cli/todo.ts` 的历史 `todo.db` 导入路径复用同一适配层
4. [x] 补 Node-only 验证，确认：
   - [x] 扩展加载
   - [x] schema 初始化
   - [x] `busy_timeout`
   - [x] 文件库 `WAL`
   - [x] 向量查询主链
5. [x] 删除 `better-sqlite3` 与对应边界代码，并同步依赖锁文件

## Acceptance Criteria

1. `node:sqlite` 能在仓库主链里成功加载 `sqlite-vec`
2. memory store 关键链路可跑通
3. `todo.db` 历史导入链路可跑通
4. `better-sqlite3` 若仍保留，必须有明确保留理由；若不保留，则依赖和边界测试一起删干净
