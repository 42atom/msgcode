# 任务：快反馈循环的验证证据写回

## Context

现在验证能力已经存在，但失败证据往往只停在本轮上下文或日志里。

这会带来两个问题：

- 下一轮修复要重复定位
- 人和 Agent 都难快速判断“最后一次到底失败在哪里”

## Goal / Non-Goals

### Goal

- 定义验证证据的最小写回口径
- 缩短下一轮修复路径

### Non-Goals

- 不做系统代替 Agent 判定完成
- 不做复杂历史聚合平台

## Plan

- [x] 盘点当前 evidence 写回入口
- [x] 定义最后一次验证结果的最小字段
- [x] 明确写回位置和读取方式

## Acceptance Criteria

1. 至少能写回：命令、退出码、stderr/stdout tail、关键产物路径。
2. 证据面服务下一轮修复，不承载状态真相。
3. 不引入新的自动裁决层。

## Notes

- 这张单只做“最后一次验证结果”的最小写回，不做历史聚合大平台。
- 证据面应该服务下一轮修复，而不是替代 `issues/` 状态真相。
