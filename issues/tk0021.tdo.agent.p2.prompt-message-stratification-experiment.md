## Context

- 当前执行链路主要把规则、记忆摘要、最近窗口和当前请求拼进同一条 system/user 序列中。
- 外部实测反馈表明，采用“system 上下夹规则，中间用多条 user 消息分层承载记忆/近期剧情/重要表格”的格式后，长程追忆与稳定性明显提升，胡言乱语体感下降。
- msgcode 当前尚未验证这种“消息分层重排”对工具调用正确率、长上下文追忆和幻觉率的实际影响。

## Goal / Non-Goals

### Goals

- 为 msgcode 设计一版最小可验证的 prompt 消息分层实验方案。
- 保持工具协议和硬边界仍由 system 承载，不把权限/工具规则降级成 user 消息。
- 明确 A/B 验证指标，判断该实验是否值得进入正式主链。

### Non-Goals

- 本单不直接改正式 prompt 主链。
- 本单不扩到浏览器底座、工具桥接或记忆系统实现重构。
- 本单不讨论角色扮演文本质量本身，只关注消息结构对稳定性的影响。

3. Plan 包含可执行的验证指标，而不是只写主观体感。
