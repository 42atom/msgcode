## Context

`ghost_*` 已成为唯一现役桌面能力面。是否要为高风险动作新增 confirm gate 已经有了明确结论：不新增执行拦截层，优先用 skill / prompt 约束模型在高风险语义下先向用户确认。

当前缺口是，这条原则还没有被明确写入 `ghost` skill、系统 prompt 和 README。若不补，后续很容易又回到“为安全感再加一层 gate”的老路。

## Goal / Non-Goals

- Goal: 把“高风险 ghost 动作先询问用户”固化到说明书与 prompt。
- Goal: 明确系统不新增 confirm gate，能力面保持完整。
- Goal: 让 README 口径与 skill/prompt 一致。
- Non-Goals: 不修改 `ghost-mcp-client`、Tool Bus 或 tool-loop 行为。
- Non-Goals: 不新增 token、审批层、supervisor 或 policy engine。
