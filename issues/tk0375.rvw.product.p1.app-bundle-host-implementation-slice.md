# 任务：App Bundle Host Implementation Slice

## Context

Desktop packaging 前两层已经分开：

- `tk0374` 负责把 artifact 合同落成 `bundle-root`
- `tk0372` 负责定义 app host 只消费 `bundle-root`
- `pl0362` 负责固定 `artifact -> app -> dmg` 顺序

但仓库里还缺 app bundle host 的现役实现单。

如果继续只停在合同，会有两个问题：

1. 下游无法证明 `.app` 真能只消费 `bundle-root`
2. `artifact -> app` 的切口仍然停留在文档，不能形成本地可启动骨架

所以这次实现只推进一刀：

- 只把 `bundle-root` 装成 `.app`
- 不进入 `dmg`

## 为什么现有文件不能直接承载实现职责

`tk0372` 只定义 app bundle host assembly 合同。

它回答：

- 输入只认 `bundle-root`
- 宿主只做 host assembly
- 输出只到本地 `.app`

它不承载实现推进状态。

如果把实现直接塞回 `tk0372`，会把：

- 合同变更
- 实现进度
- 验证证据

重新混成一张单。

`tk0374` 只承载 build artifact implementation。

它的边界到：

- `bundle-root/runtime`
- `bundle-root/bin/msgcode`
- `bundle-root/bootstrap`
- `bundle-root/appliance.manifest`

不能再往下替 app host 背实现状态，否则 artifact 和 host 的切口又糊掉。

`pl0362` 是导航主线。

它负责：

- 固定 `artifact -> app -> dmg`
- 约束子任务顺序

plan 不是实现单，不能代替当前 slice 的状态真相。

所以这次必须单开 1 张最薄实现单，只承载：

- `bundle-root -> .app`

## Goal / Non-Goals

### Goal

- 只消费 `tk0374` 产出的 `bundle-root`
- 产出本地可启动的 `.app` 目录骨架
- 宿主只做装配和启动转发

### Non-Goals

- 不碰 `bundle-root` 合同
- 不碰 dmg、签名、公证
- 不碰授权、更新、跨平台安装器
- 不引入第二主脑

## Implementation Slice

最小实现只做：

1. 复制 `bundle-root` 到 `.app/Contents/Resources/`
2. 生成 `Info.plist`
3. 生成 `Contents/MacOS/` 启动入口
4. 保证启动入口只转发到 bundled `msgcode`

输入只认：

1. `tk0374` 产出的 `bundle-root`

输出只认：

1. 本地 `.app` 目录骨架

## Plan

- [x] 补一张最薄 app host implementation task，单独承载实现状态
- [x] 增加只消费 `bundle-root` 的 `.app` 装配入口
- [x] 生成最小宿主骨架：`Info.plist`、`Contents/MacOS/`、`Contents/Resources/bundle-root`
- [x] 用最小测试验证宿主只转发到 app 内 bundled launcher

## Acceptance Criteria

1. `.app` 输入只认 `bundle-root`
2. `.app` 内存在稳定启动入口
3. 启动入口执行时只回到 app 内 bundled runtime
4. 不要求 dmg、签名、公证成立

## Notes

- 这是 Desktop packaging 的第 2 张实现切片。
- 这张单只承载 app bundle host implementation，不替 dmg 链占位。
