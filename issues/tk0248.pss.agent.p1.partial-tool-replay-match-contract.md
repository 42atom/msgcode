## Context

- 现场日志存在真实失败：`invalid params, tool call and result not match (2013)`
- 当前主链若一轮里模型发出多个工具调用，但本轮只实际执行了其中一部分，就会把整轮 tool_call 原样回灌
- 这会让 replay 中的 tool_call 数量大于 tool_result 数量

## Goal

- 让 replay 配对保持一一对应
- 保持当前工具执行与失败返回策略不变

## Non-Goals

- 不改模型选工具策略
- 不新增恢复提示或二次裁判层
- 不改变首个工具失败就收口的当前行为

## Goal / Non-Goals

- Goal:
  - OpenAI / MiniMax replay 时都只回灌本轮真正执行过的工具调用
  - 多工具失败或 quota 提前中断时不再制造配对不一致
- Non-Goals:
  - 不改变工具执行顺序
  - 不强行把一轮剩余未执行工具继续跑完
  - 不改新协议，只修 replay 口径

## Plan

- [x] 过滤 replay assistant/tool_use 到已执行集合
- [x] 补 OpenAI 回归
- [x] 补 MiniMax 回归

## Acceptance Criteria

1. OpenAI 路径 replay 时，assistant `tool_calls` 数量与本轮 `tool` result 数量一致。
2. MiniMax 路径 replay 时，assistant `tool_use` 与 user `tool_result` 一一对应。
3. 不改当前“首个工具失败就收口本轮”的执行策略。

## Notes

- 这次是 replay 配对合同错误，不是工具执行器本身错误。
- 最薄修法是过滤 replay 集合，不是追加恢复提示。

## Result

- OpenAI replay 现在只保留本轮真实已执行的 `tool_calls`
- MiniMax replay 现在只保留本轮真实已执行的 `tool_use`
- 失败或 quota 提前收口时，不再把未执行调用错误地一并回灌
