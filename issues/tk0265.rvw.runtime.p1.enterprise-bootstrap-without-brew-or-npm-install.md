# 任务：Appliance 安装链去 Brew / 去 npm install

## Context

当前正式安装面还依赖：

- Homebrew
- `bootstrap/Brewfile`
- `npm install`
- 源码态 `tsx`

这不适合 appliance “自包含交付”。

## Goal / Non-Goals

### Goal

- 定义 appliance 交付包结构
- 定义安装入口、启动入口、升级入口
- 定义首次启动初始化入口
- 明确哪些依赖必须被随包携带

### Non-Goals

- 不立即实现 DMG/PKG/Installer
- 不直接改用户版开发流程
- 不先决定 App 技术栈

## Plan

- [x] 盘点当前 bootstrap 依赖链
- [x] 明确 appliance 最小安装包结构
- [x] 定义“安装后可启动”的验收流程
- [x] 明确 install / first-run-init / upgrade 三段结构
- [x] 落地 compiled/runtime 入口优先合同
- [x] 落地 `msgcode init --workspace` first-run 骨架
- [x] 落地 install / first-run-init / upgrade 脚本骨架

## Acceptance Criteria

1. 能说清 appliance 如何在不执行 `brew` / `npm install` 的情况下启动。
2. 能列出必须随包交付的运行时组件。
3. 能明确区分开发者 bootstrap 与 appliance bootstrap。
4. 有正式安装验收口径。
