# Goal

把 memory 第一阶段主链收成：

- append log 是唯一真相
- sqlite / chunks / summary 都只是派生层
- 任何派生层都能从 append log 重建

## Non-Goals

- 不重做整套 memory 检索
- 不扩 vector / hybrid 策略
- 不做“长期记忆平台”
- 不把摘要升格成真相源

## Plan

- [x] 盘清当前 append 落盘面：
  - [x] memory/*.md
  - [x] thread/session 追加面
  - [x] 哪些事实还没有稳定 append
- [x] 明确 rebuild 合同：
  - [x] sqlite 能删后重建
  - [x] chunks 能重切
  - [x] summary 只能派生生成
- [x] 补最小实现或验证：
  - [x] 至少一条 append -> index rebuild 证据
  - [x] 至少一条“删索引不丢真相”的证据
- [ ] 把后续深水区拆开：
  - [ ] retrieval
  - [ ] summarization
  - [ ] cross-agent sharing

## Acceptance

1. append log 被明确定义为 memory 真相根。
2. sqlite/index 被明确降为派生层，不承载唯一真相。
3. 删除索引后，系统仍可从 append log 重建派生层。
4. 后续检索、摘要、共享都不再和真相层混写。

## Notes

- 这张单承接的是用户已经明确给出的原则：
  - append 不能被压缩和摘要抹杀
- 这张单不是“再讲一遍理念”，而是把理念收成可验证主线。

## Findings

- 当前稳定的 memory append 真相面，只有 `<workspace>/memory/*.md`。
- `msgcode memory add` 直接向当天 `memory/YYYY-MM-DD.md` 追加文本；这是唯一已落地、可编辑、可追溯的长期记忆写入口。
- `~/.config/msgcode/memory/index.sqlite` 只承载派生索引：documents / chunks / FTS / vec；运行时检索读它，但不会自动重建它。
- `thread/session` 追加面存在，但当前承载的是会话连续性与短中期上下文，不是 memory 真相根。
- 已补最小回归：
  - [`/Users/admin/GitProjects/msgcode/test/p7-tk0228-memory-append-rebuild-contract.test.ts`](/Users/admin/GitProjects/msgcode/test/p7-tk0228-memory-append-rebuild-contract.test.ts)
  - 证明链路为：`memory add -> memory index -> 删除 index.sqlite -> memory index -> memory search`
  - 删除索引后，append 文件仍在；重建后检索可恢复。
