# tk0218: 子代理主链 Live 验证（Feishu）

## 目标

在飞书真实通道里，验证主脑会“按经理方式”调用子代理完成任务，而不是自己亲自干活；并且能留下可审计的文件真相源（dispatch/subagent records），可据此验收。

## 前置

- 走 workspace：`/Users/admin/msgcode-workspaces/test-real`
  - 该 workspace 已有 `.msgcode/config.json`，可用于飞书通道调试。
- persona 文档存在：
  - `docs/protocol/personas/frontend-builder.md`
  - `docs/protocol/personas/code-reviewer.md`

## 测试用例（强制委派）

用飞书以“用户身份”发一条消息，明确要求“必须委派子代理”，避免模型按“小任务亲做”跳过 subagent 链：

### 消息模板（复制用）

把 `<TOKEN>` 换成一次性 token（例如 `tk0218-subagent-<timestamp>`）：

```
哥，做一次子代理主链验收（必须走 subagent，不允许亲做）。

工作区固定：/Users/admin/msgcode-workspaces/test-real
产物目录：/Users/admin/msgcode-workspaces/test-real/live-subagent-<TOKEN>/

任务（第一单，仅此一单）：
1) 委派子代理（persona=frontend-builder）创建最小 HTML 页面 index.html
   - 明确画布：800x600
   - 背景：草绿色（不要换成你自选的别的绿）
   - 文案：粗体黑字 “Im here ！”（标点空格保持原样）
   - 居中
2) 子代理完成后，你负责验收：用证据证明上述四条都满足（读文件/grep/任何可核验方式）
3) （可选）再委派第二个子代理（persona=code-reviewer）对产物做一次 review，只提问题不改代码

回传只要 6 行：
- 本轮是否委派：是/否（必须是）
- 使用的 persona id（必须来自 persona 文档，不准编）
- dispatch 记录路径（.msgcode/dispatch/*.json）
- subagent 记录路径（.msgcode/subagents/*.json）
- 产物路径
- 验收证据摘要（1-2 句）

最后带上标记 <TOKEN>
```

## 验收（硬标准）

### A. 主脑管理（核心）

验收重点是“管理工作是否成立”，不以执行臂的具体能力作为唯一判断；即使子代理失败，只要主脑完整履行派单/监督/验收/回写/如实沟通，也应被视为主脑管理能力达标（任务结果可判 failed，但管理流程要成立）。

- 派单决策证据：dispatch 中能看到主脑明确选择“委派而非亲做”的理由（至少一条关键句）。
- 合同锋利证据：dispatch 的 `taskCard` 里 `goal/acceptance/artifact/cwd/constraints` 不空泛，且包含本次 `<TOKEN>`。
- Persona 证据：`dispatch.persona` 必须来自 `docs/protocol/personas/*.md` 现存 id，不得编造；最好记录选择理由。
- 监督证据：dispatch 有状态迁移与时间戳（`pending -> running -> completed/failed`）；若 watch 超时则应保持 `running` 且可继续监督（有 `subagentTaskId`）。
- 验收证据：主脑在宣告完成/失败前做过可核验检查（命令输出摘要、文件 digest、路径清单等），写入 `dispatch.result.evidence[]`（或等价字段）。
- 回写证据：子任务 issue 文件名状态有推进（`tdo/doi -> pss/rvw` 或失败时进入 `bkd/cand`），并能反向链接到对应 dispatch/subagent 记录。

### B. 文件真相源

在 `test-real` workspace 内，必须出现并可对照：

- `.msgcode/dispatch/*.json`
  - 关联本次 `<TOKEN>`（例如 goal/summary/metadata 中包含 token）
  - `status` 进入 `completed`（或若超时则为 `running`，且有可继续监督的 `subagentTaskId`）
  - 含 `persona` 与 `taskCard` 字段（若实现支持）
- `.msgcode/subagents/*.json`
  - 能看到子代理 run 记录与 `persona`（若实现支持）

### C. 产物

- `index.html` 存在于 `live-subagent-<TOKEN>/index.html`
- 文件内容满足：800x600、草绿色背景、黑色粗体文字 `Im here ！`、居中

## 非目标

- 不要求接入 `tk0204` heartbeat/alarm/reflection
- 不要求并发、多 subagent 编排平台化
- 不要求把此用例写成自动化测试（本单是 live 验收用例）
