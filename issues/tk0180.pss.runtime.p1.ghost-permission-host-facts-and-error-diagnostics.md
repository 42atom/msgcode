## Context

当前 `ghost_screenshot` 在 Terminal 手动调用可用，但在 launchd 托管的 msgcode daemon 路径下会失败。历史 run-core 事件与 tools-bus 结构化日志只记录 `errorCode`，缺少原始错误文本，导致模型只能“猜”成未授权，用户无法用证据闭环定位是“权限宿主不同”还是其它根因。

## Goal / Non-Goals

- Goal: 在不新增控制层/拦截层的前提下，把 ghost 失败的**原始错误事实**写进可观测面，并提供一个薄的 CLI 命令输出 daemon 宿主事实并打开系统设置权限页。
- Non-Goals: 不做全局 confirm gate；不做自动授权；不做 desktop manager/platform；不重构 tool-loop 主链。

## Plan

- [x] tools-bus telemetry 事件补齐 `errorMessage`（结构化事实，不加教学文案）
- [x] tool-loop actionJournal 补齐 `errorMessage`（截断）并透传到 run:tool details
- [x] 新增 `msgcode ghost permissions`：输出 daemon 宿主事实并可 `--open` 打开系统设置页
- [x] 更新 `docs/CHANGELOG.md`
- [x] 跑回归：`bun test` + `npx tsc --noEmit` + `npm run docs:check`

## Acceptance Criteria

1. 当工具失败时，`~/.config/msgcode/log/msgcode.log` 的 `Tool Bus: FAILURE ...` 元数据包含 `errorMessage` 字段（截断允许）。
2. `run-events.jsonl` 的 `run:tool` 事件 `details` 中包含 `errorMessage`（若有）。
3. `msgcode ghost permissions --open` 能打开 Accessibility + Screen Recording 两个系统设置页面，并输出 daemon label/plist/programArguments 事实。

## Notes

- logs: `~/.config/msgcode/log/msgcode.log`
- run events: `~/.config/msgcode/run-core/run-events.jsonl`
