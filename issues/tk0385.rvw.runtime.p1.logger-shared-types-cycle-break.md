# TK0385：Logger Shared Types Cycle Break

## Context

`codegraph` 已确认：

- `src/logger/index.ts`
- `src/logger/console-transport.ts`
- `src/logger/file-transport.ts`

存在真实循环依赖。

这组循环不是业务流程互相调用，而是共享类型和接口被挂在 `index.ts`，导致 transport 反向依赖聚合入口。

## Goal / Non-Goals

### Goal

- 把 `LogLevel / LogEntry / Transport / TransportOptions` 从 `index.ts` 拆到共享类型文件
- 让 `console-transport.ts` 与 `file-transport.ts` 不再 import `index.ts`
- 保持 `src/logger/index.ts` 的对外导出面继续可用

### Non-Goals

- 不改 logger 行为
- 不重写 transport 实现
- 不处理 `tmux` 和 `runtime` 的其他循环

## Acceptance Criteria

1. `logger` 三文件循环消失
2. 外部继续可以从 `src/logger/index.ts` 使用原有类型/类导出
3. `tsc --noEmit` 通过
4. `codegraph` 循环查询不再包含 logger 这一组

## Plan

1. 新建 `src/logger/types.ts`
2. 把共享类型与接口迁过去
3. `index.ts` 回导出共享类型
4. transport 改为直接 import `types.ts`
5. 用 `tsc`、logger 邻近测试和 `codegraph` 复验

## Notes

- 这刀只收共享类型切口，不改 logger 行为
- `codegraph` 增量 hook 当前已确认口径过期，因此循环复验使用全量重建后的图结果
- `tmux` 与 `runtime` 其他循环保持不动
