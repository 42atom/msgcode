# 任务：people pending add 与读面过滤切片

## Context

当前 `people` 主线已经有：

- `people-save`
- `character-identity/*.csv`
- `people-pending.json` 读取
- `channel + chatId + senderId` 键合同

但还缺两处实现：

1. 没有正式 pending 写入口
2. 已认领 pending 仍会继续显示

## Goal / Non-Goals

### Goal

- 增加 `msgcode appliance people-pending-add`
- 只写 `.msgcode/people-pending.json`
- people 读面按唯一键自动隐藏已认领 pending

### Non-Goals

- 不实现飞书自动导入
- 不删除 pending 文件里的旧条目
- 不做 tags
- 不做全局人物中心

## Plan

- [x] 增加 pending 正式写口
- [x] pending 写入口按 `channel + chatId + senderId` upsert
- [x] people 读面自动隐藏已认领 pending
- [x] 用现有测试文件补黑盒锁

## Acceptance Criteria

1. `people-pending-add` 可创建或更新 `people-pending.json`。
2. people 读面不会继续显示已存在于 CSV 的 pending 项。
3. 仍然只有两份真相源：
   - `character-identity/*.csv`
   - `people-pending.json`

## Notes

- `people-pending-add` 至少要求：
  - `channel`
  - `chatId`
  - `senderId`
  - `username` 或 `displayName`
- `seenAt` 可选；未提供时由 runtime 写当前时间。
- 读面过滤只做差集，不修改 pending 文件本体。

## Review Notes

- 圆桌结论：这刀可以过。
- 保留点：
  - 不发明 `people.json`
  - 不让 UI 直写或删除 pending
  - 不实现自动匹配推荐
- 风险提醒：
  - 现阶段仍需要导入侧自己提供 `chatId + senderId`
  - 真正飞书拉取仍应另开切片，不在本单偷塞
