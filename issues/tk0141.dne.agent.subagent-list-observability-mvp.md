## Context

当前 `subagent` 已有：

- `run`
- `status`
- `stop`

这已经足够跑单任务主链，但主脑和用户仍然需要记住 `taskId` 才能继续观察任务。  
在“一个主脑 + 一个子例程”的薄模型下，最值钱的补充不是 queue / orchestrator，而是一个最小观测原语：

- `list`

这条设计也可借鉴 Node 异步最佳实践：

- 明确句柄
- 提交与观察分离
- 调用方编排
- 背压诚实暴露

而不是引入新的池子控制面。

## Goal / Non-Goals

### Goal

- 增加 `msgcode subagent list`
- 允许按 workspace / client 查看当前任务清单
- 让主脑和用户无需死记 `taskId`
- 同步 `help-docs` 与 `subagent` skill 文案

### Non-Goals

- 不做多任务队列
- 不做并发调度器
- 不做自动聚合器
- 不新增 `tail`（当前 `status` 已含 paneTail）

4. 不引入 queue / parallel / orchestrate 等新命令或新状态层。
