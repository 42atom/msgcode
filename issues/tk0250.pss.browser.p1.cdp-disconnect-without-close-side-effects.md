## Context

- 现场日志存在真实 daemon `unhandledRejection`：`Protocol error (Network.setCacheDisabled): Internal server error, session closed`
- 当前 browser runner 通过 `connectOverCDP()` 连接共享 Chrome
- 但各个 `finally` 里统一直接 `browser.close()`，这对共享 CDP 连接过重

## Goal

- 共享 CDP 连接完成后优先只断本地连接
- 保持 browser tool 的对外合同不变

## Non-Goals

- 不改 tabs/instances 合同
- 不重写 Patchright runner
- 不引入 browser daemon

## Goal / Non-Goals

- Goal:
  - 给 `connectOverCDP` 浏览器连接提供薄的 disconnect 路径
  - 避免 finally 里的 `browser.close()` 触发额外远端副作用
- Non-Goals:
  - 不改变浏览器操作语义
  - 不处理业务层 tab 不存在等正常错误
  - 不补第二套 browser 运行时

## Plan

- [x] 抽一个 CDP disconnect helper
- [x] 让 tabs.* finally 统一走 helper
- [x] 补一条“优先断 connection，不调用 browser.close”回归

## Acceptance Criteria

1. `tabs.*` 这类 `connectOverCDP` 路径在 finally 时优先断本地 connection。
2. 不再直接依赖 `browser.close()` 作为共享 CDP 连接的默认收尾方式。
3. browser runner 现有定向回归继续通过。

## Notes

- 这次修的是连接收尾方式，不是浏览器操作语义。
- 最薄修法是抽一个 disconnect helper，统一给现有 finally 复用。

## Result

- Patchright CDP 连接现在优先断本地 `_connection`
- `tabs.open/list/snapshot/text/action/eval` 不再默认直接 `browser.close()`
- browser runner 定向回归已锁住“优先断 connection，不调 browser.close”
