# 任务：people-pending 键合同

## Context

当前 People 线已经有：

- 已知人物 CSV
- `people-save`

但 `people-pending.json` 还只有读取口径，没有稳定写键。

如果现在直接做“已认领自动隐藏”，会有两个风险：

- 同一个 `senderId` 在不同 chat 混在一起
- 读面开始替缺字段猜测

## Goal / Non-Goals

### Goal

- 固定 pending 与 CSV 对齐的唯一键
- 固定最小字段
- 固定读面何时允许隐藏 pending

### Non-Goals

- 不实现导入器
- 不做迁移脚本
- 不实现 UI 逻辑

## Plan

- [x] 固定 pending 最小键字段
- [x] 固定 pending 与 CSV 的唯一键对齐
- [x] 固定读面允许隐藏 pending 的前置条件

## Acceptance Criteria

1. pending 的唯一键与 CSV 唯一键一致。
2. 读面不会因为字段不全就猜测已认领关系。
3. 后续过滤实现有明确前置条件。

## Notes

- 本轮已经把读面收成 `channel + chatId + senderId`。
- 旧 pending 条目不再接受，只给 warning，不做兼容猜测。
- 目标仍然是把后续过滤实现的“前提”钉死，不让 runtime 自作聪明。
