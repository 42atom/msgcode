# 任务：wpkg 切到 tar.zst 包壳

## Context

`0290` 已经把 `wpkg` 的协议口径收成：

- 带 `manifest.json` 的 `tar.zst` 包

但当前安装主链和测试还停在旧事实：

- 安装器仍按 zip + `ditto -x -k`
- 测试仍用 `/usr/bin/zip` 造包

这会让协议和实现重新漂开。

## Goal / Non-Goals

### Goal

- 把 wpkg 安装主链切到 `tar.zst`
- 把测试造包方式切到 `tar.zst`
- 把相关切片文档同步到当前事实

### Non-Goals

- 不改 manifest 合同
- 不改 packs/sites 注册表
- 不实现升级、卸载、签名、市场
- 不新增新的包格式层

## Plan

- [x] 改安装主链，从 zip 解包切到 tar.zst 解包
- [x] 改测试造包方式，从 zip 切到 tar.zst
- [x] 同步切片文档里的旧 zip/ditto 描述
- [x] 跑最小安装回归、类型检查和 docs gate

## Acceptance Criteria

1. `workspace-wpkg-install.ts` 不再用 `ditto -x -k` 解 zip。
2. `p7-tk0305-wpkg-install-register.test.ts` 不再用 `/usr/bin/zip` 造包。
3. `WPKG / tk0290 / rp0305` 等真相文档和当前实现一致。
4. 最小安装回归、`tsc`、`docs:check` 通过。

## Notes

- 这刀只换包壳，不换主链。
- 当前宿主使用 `zstd -d -c | tar -xf -` 这条最薄且已验证可用的解包路径。
