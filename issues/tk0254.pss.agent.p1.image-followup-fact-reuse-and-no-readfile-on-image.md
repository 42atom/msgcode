## Context

- 2026-03-19 family 真实线程里，首轮图片问题能得到报告解读
- 后续追问“左眼数据也有啊”“你帮我把它整理成一份md记录”时，模型退回“我没有视觉能力”
- 同一段日志里还出现了 `fail:read_file -> ok:bash -> ok:ghost_screenshot -> fail:read_file`
- 根因更像 exec 合同不够锋利，而不是缺新的状态层：最近窗口里其实已经有图片附件路径和上一轮图像分析结论

## Goal

- 让图片后的文本追问优先复用最近已提取的图片事实
- 禁止对 png/jpg 这类图片本体调用 `read_file`

## Non-Goals

- 不新增图片记忆层
- 不改附件落盘协议
- 不改 tool bus 或 vision provider

## Goal / Non-Goals

- Goal:
  - 图片 follow-up 优先复用最近窗口里已有的图片事实
  - 禁止对图片本体误用 `read_file`
- Non-Goals:
  - 不新增任何图片状态层
  - 不改现有附件收取或落盘主链

## Plan

- [x] 在 exec prompt 增加图片 follow-up 复用规则
- [x] 在 exec prompt 增加图片文件禁止 `read_file` 规则
- [x] 补回归并验证

## Acceptance Criteria

1. `buildExecSystemPrompt` 明确包含图片 follow-up 复用规则。
2. `buildExecSystemPrompt` 明确包含图片文件禁止 `read_file` 的规则。
3. 相关 prompt 拆分回归通过。

## Result

- exec prompt 现在明确要求：
  - 如果最近窗口已经有附件信息、`[图片摘要]` 或上一轮图片分析结论，图片 follow-up 优先复用这些事实继续回答
  - 不能因为本轮没有新附件就退回“我没有视觉能力”
  - `png/jpg/jpeg/webp` 不是 `read_file` 的目标，图片本体不能用 `read_file`
- 这刀只收提示词合同，没有新加 runtime 层，保持单一主链
