## Context

- 正式协议、目录协议、AIDOCS 表层整理已经完成。
- 但历史 markdown 里仍有大量旧回链：
  - `issues/000x-*.md`
  - `docs/plan/plan-YYMMDD-*.md`
  - 个别旧目录口径
- 这些残留不阻断主链，但会误导检索和交接。

## Goal / Non-Goals

- Goal:
  - 清理高价值历史回链，让文档优先指向当前 `tk/pl/rs/rf/rp` 真相源
  - 对 `docs/tasks/`、`docs/archive/`、`issues/`、`AIDOCS/` 做一轮历史索引巡检
  - 形成“现役必须精确、历史尽量不误导”的最低标准
- Non-Goals:
  - 不重写历史正文业务内容
  - 不追求把所有 archive 文本完全现代化
  - 不修改业务代码

## Plan

- [x] 盘点旧回链类型与高频命中
- [x] 先修高价值 issue / plan 回链
- [x] 再修 `docs/tasks/` / `docs/archive/` 历史索引
- [x] 最后清 `AIDOCS/` 中仍误导现役入口的文本

## Acceptance Criteria

1. 高价值文档不再指向 `issues/000x-*.md` 或 `docs/plan/plan-YYMMDD-*.md`。

3. `npm run docs:check` 通过。
