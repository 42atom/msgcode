## Context

`src/tools/bus.ts` 里剩余最厚的两块是：

1. `desktop` 的旧 `subcommand` 兼容主链，仍在 bus 内部查找 `msgcode-desktopctl`、维护 session、拼装命令行参数
2. `validateToolArgs()` 中最厚的 `browser` 手写前置裁判，和 runner 自身的 bad args 能力重复

这让 bus 同时承担了网关、历史兼容、执行适配三类职责，继续偏离“validate -> execute -> return”的薄主链。

## Goal / Non-Goals

- Goal: 删除 `desktop` 旧 `subcommand` 主链，只保留 `method + params` RPC 主链
- Goal: 将 desktop session / desktopctl 发现 / artifact 解析移出 `src/tools/bus.ts`
- Goal: 删除 `validateToolArgs()` 中 `browser` 的厚手写裁判，只保留最小必需校验
- Non-Goals: 本轮不重做 manifest 全体系
- Non-Goals: 本轮不改 desktop / browser / feishu 的能力边界
- Non-Goals: 本轮不重做 tool-loop
