## Context

- 这张单把跨 agent 共享从“抽象平台想象”拉回到文件真相与路径边界。

## Goal / Non-Goals

- Goal:
  - 明确同路径共享、异路径隔离的 file-truth 边界
  - 解决 `workspaceId` 由目录同名导致的串味问题
- Non-Goals:
  - 不做共享服务
  - 不做权限中台

## Plan

- [x] 找出当前跨 agent 串味的真实根因
- [x] 把 `workspaceId` 改成绝对路径导出的稳定键
- [x] 用回归锁住同路径共享、异路径隔离

## Acceptance Criteria

1. 同一路径跨 agent 共享同一份 append 真相。
2. 不同路径即使目录同名也不会串味。
3. 没有新增共享服务或数据库真相源。

## Notes

- 这轮只收第一性边界，不扩更厚的授权面。

## Links

- Related: `issues/tk0228.pss.memory.p1.append-log-truth-and-rebuild-contract.md`

# Goal

把跨 agent 共享单独拆出来：

- 哪些 memory 文件可共享
- 共享后如何保持 file truth
- 不让 sqlite/index 反客为主

## Non-Goals

- 不先做共享服务
- 不先做权限中台
- 不把 memory 设计成数据库优先

## Progress

- 已确认当前最硬的主链缺口不是“少一个共享层”，而是 `workspaceId` 曾用 `basename(workspacePath)`：
  - 不同路径但同名目录会撞 ID
  - 检索可能串味
- 第一刀已落地：
  - `workspaceId` 改为从绝对路径导出的稳定内部键
  - 同一路径跨 agent 一致
  - 不同路径即使目录同名也不再撞 ID
- 协议已写明：
  - 同一 `workspacePath` 才共享 append 真相
  - summary / dispatch / sqlite 派生层都不算共享真相

## Result

- cross-agent sharing 的第一轮边界已收住：
  - 同一 `workspacePath` 下共享同一份 `memory/*.md`
  - 共享身份由绝对路径导出的稳定内部键承载
  - 不同路径即使目录同名，也不会在索引和检索里串味
- 这轮没有新增共享服务，也没有引入数据库优先真相源。
