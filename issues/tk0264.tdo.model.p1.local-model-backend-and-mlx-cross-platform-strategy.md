# 任务：本地模型后端与 MLX 跨平台策略

## Context

当前系统里，本地模型主链与 `LM Studio / OMLX / MLX` 相关能力明显偏 macOS 环境。

如果企业版要进入：

- 自包含交付
- App 形态
- 跨平台兼容

那这条边界必须先说清。

## Goal / Non-Goals

### Goal

- 盘点当前本地模型依赖哪些 macOS / MLX 条件
- 区分：
  - mac-only 能力
  - 可替换能力
  - 必须抽象的能力边界
- 给出跨平台本地模型路线建议

### Non-Goals

- 不立即改 chat/provider 主链
- 不直接接入新的模型平台
- 不把问题扩成完整模型控制面

## Plan

- [ ] 盘点当前本地模型与视觉路径的系统依赖
- [ ] 明确 MLX/mac 与跨平台路线的冲突点
- [ ] 输出企业版建议能力矩阵

## Acceptance Criteria

1. 能说清当前哪些能力被 macOS / MLX 锁死。
2. 能区分短期保留、短期替换、长期抽象三类。
3. 不用“以后再看”回避跨平台边界。

## Notes

- 这张单的目标是把模型后端边界说清，不是立刻平台化 provider。
- 需要优先保留当前可运行主链，再讨论跨平台抽象。
