# 任务：appliance thread 读面切片

## Context

现在已经有：

- `appliance workspace-tree`：左树
- `appliance threads`：单工作区总览 + 当前线程

但主界面中间聊天区真正需要的是：

- 指定 `threadId` 的正文
- 同时带出右栏继续要用的 `people / workStatus / schedules`

如果继续复用 `threads` 总览去硬拼，会把“总览”和“正文”两种职责搅在一起。

## Goal / Non-Goals

### Goal

- 增加 `msgcode appliance thread --workspace ... --thread-id ... --json`
- 只读指定线程正文
- 继续沿用现有 `status.log / people / schedules` 真相源
- 缺线程时返回正式 JSON 错误

### Non-Goals

- 不实现 thread 写面
- 不改当前 thread front matter 合同
- 不改 archive 页
- 不在 UI 里新增第二状态层

## Plan

- [x] 增加 `msgcode appliance thread --workspace ... --thread-id ... --json`
- [x] 复用现有 thread markdown 解析
- [x] 复用现有 `status.log / people / schedules`
- [x] 缺线程时报 `APPLIANCE_THREAD_MISSING`
- [x] 增加黑盒合同测试

## Acceptance Criteria

1. 给定合法 `threadId` 时，能返回对应线程正文和右栏所需数据。
2. `config.json` 当前选中线程不影响按 `threadId` 读取。
3. 缺线程时，命令返回 JSON Envelope 错误，不抛原生栈。
4. people 和 schedules 仍按现有工作区真相源读取，不额外造索引。
