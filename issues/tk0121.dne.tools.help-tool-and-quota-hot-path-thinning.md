## Context

Issue 0119 已指出当前骨架仍有三块主链纯度问题：

1. help 系统存在，但还没进入模型主探索路径
2. 工具索引里仍有“写改文件优先用 bash”这类硬编码折返
3. 配额边界仍在 tool-loop 热路径里直接生成中文系统话术

用户明确要求继续参考《CLI is All Agents Need》的核心骨架收口 runtime，不停留在审计层。

## Goal / Non-Goals

### Goal

- 将 `help-docs` 接成模型可直接调用的原生工具
- 去掉工具索引里对 bash 的硬编码映射，改为显式引导模型先查 help
- 将 quota 命中从“热路径中文系统文案”改为“结构化事实 + 机器标记”

### Non-Goals

- 不把整套工具面重做成单一 `run(command)`
- 不在本轮改动 `write_file/edit_file` 默认暴露策略
- 不新增新的 manager / orchestrator / controller

## Plan

- [x] 新增 `help_docs` 工具合同与执行实现，复用现有 `help-docs --json` 真相源
- [x] 将 `help_docs` 加入默认 workspace 工具面与模型工具索引
- [x] 去掉工具索引中的“写改文件优先用 bash”硬编码，改为“不会就先用 help_docs 查合同”
- [x] 为 quota 命中新增结构化字段，热路径 answer 退化为机器标记
- [x] 更新回归测试、0119 Notes 与 CHANGELOG

## Acceptance Criteria

1. 模型默认工具面中可见并可执行 `help_docs`
2. `help_docs` 能返回过滤后的 CLI 合同，供模型在主链中自发现命令
3. `tool-loop` 触发工具/步骤 quota 时，不再在热路径拼中文说明，而是返回结构化 quota 事实
4. 相关回归、类型检查与 docs 校验全部通过
