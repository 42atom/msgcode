# 任务：Readonly Thread Surface Host Bridge Slice

## Context

`tk0381` 已把正式 UI 骨架落到 `src/ui/main-window/thread-surface.ts`，但当前代码图上它仍是 dead-unresolved：

- 没有 preload 桥
- 没有 main/renderer 接线
- 没有正式的 `window.msgcodeReadonlySurface.runCommand` 注入

这意味着正式 UI 结构已经有了，但它还不是宿主里真的可运行的一刀。

## Goal / Non-Goals

### Goal

1. 给 readonly thread surface 落最薄 host bridge。
2. 只暴露两个正式读面：
   - `workspace-tree`
   - `thread`
3. 让 renderer 通过受控桥接读取数据，而不是直接 import runtime 内部模块。

### Non-Goals

1. 不做发消息。
2. 不做 archive / people / settings 内容页。
3. 不做新的本地服务层。
4. 不做 Electron 大壳工程扩张。
5. 不改 `ui-protype`。

## Plan

- [x] 找到 main / preload / renderer 的最薄代码落点。
- [x] 给 renderer 暴露 `window.msgcodeReadonlySurface.runCommand` 或等价薄接口。
- [x] 让 readonly thread surface 只通过这条桥读取 `workspace-tree` 与 `thread`。
- [x] 补最小测试，锁住桥接存在与读面边界。

## Acceptance Criteria

1. `src/ui/main-window/thread-surface.ts` 不再是死骨架。
2. 宿主桥只读，不承载写面。
3. renderer 不直接 import `src/runtime/*`。
4. 没有引入 REST / GraphQL / WebSocket 中间层。
5. 最小验证通过。

## Notes

### 为什么不能继续塞进 tk0377

`tk0377` 是主线单，只负责范围和边界，不适合承载宿主实现细节。

### 为什么不能继续塞进 tk0381

`tk0381` 的职责已经写死为：

- `src/ui` 三栏骨架
- settings 一级入口 affordance
- host bridge 占位

它明确没有进入 host bridge 实现。

继续把桥接实现塞回 `tk0381`，会把“正式 UI 骨架”和“宿主接线”重新混成一张单。
