## Context

- 当前系统仍把 `vision` 当成 LLM 可直接调用的正式工具：
  - `tooling.allow` 默认暴露 `vision`
  - `TOOL_MANIFESTS` 暴露 `vision`
  - `/tool allow` 也把 `vision` 当成用户可配置工具
- 这会把详细视觉任务继续锁在 runtime 里，导致系统不断吸收供应商差异：
  - GPT 原生看图
  - GLM MCP vision server
  - 本地 LM Studio / 本地视觉脚本
- 最近图片表格事故再次证明：问题不只是模型能力，而是系统仍在替模型决定视觉任务执行方式、重试方式和供应商路径。
- 仓库里还存在同类越权点：
  - `src/listener.ts` 在图片-only 场景仍会伪造 `"请用一句话概括主要内容。"`
  - `src/media/pipeline.ts` / `src/runners/asr.ts` 仍默认强制中文转写

## Goal / Non-Goals

- Goal: 系统层只保留“收图、落盘、预览摘要”这条图片主链。
- Goal: 详细视觉理解迁到 skill 层，按供应商能力编写说明书，不再把供应商差异写进 runtime 主链。
- Goal: LLM 不再把 `vision` 当作正式可调用工具，而是通过 skill 决定走 GPT 原生看图、GLM MCP、本地 LM Studio 等路径。
- Goal: 为后续多视觉供应商并存提供稳定边界。
- Non-Goals: 本轮不做新的视觉供应商编排平台。
- Non-Goals: 本轮不顺手把 ASR 全部重构掉，只记录为同类后续风险。
- Non-Goals: 本轮不要求一次性删除所有现有 `vision` 实现，可分阶段降级/退场。

## Plan

- [x] 创建 Plan 文档，冻结“预览摘要 vs skill 详细视觉”边界
- [x] 收口系统边界：停止把 `vision` 作为 LLM 正式工具暴露
- [x] 设计 runtime vision skill 方案：provider-neutral 入口 + provider-specific skill
- [x] 清理图片-only 场景的伪造问题注入
- [x] 设计迁移步骤：旧 workspace / `/tool allow vision` / 旧测试的兼容处理
- [x] 记录 ASR 等同类越权点，形成后续清理队列

## Acceptance Criteria

1. 系统层的图片职责被明确收口为：收图、落盘、预览摘要。
2. 详细视觉任务不再依赖 runtime `vision` 工具暴露给 LLM。
3. 方案能同时容纳 GPT 原生看图、GLM MCP、本地视觉脚本三类供应商。
4. 方案不新增新的视觉控制层或供应商编排平台。
5. 迁移后，用户和模型面对的是 skill 说明书，而不是不断变化的 runtime 视觉合同。

  - 2026-03-09 排障补记：文本追问旧图片时，模型曾在本轮 `toolCallCount=0` 的情况下复述“vision 崩溃”旧失败文案。已把“不许编造工具成功/失败、不许在缺少当前附件路径时假装已读图”的硬约束补进执行核提示词与工具索引
  - 2026-03-09 排障补记：重启后再次追问旧图片时，`minimax` 在 `read_file` 成功后返回 `invalid_request_error: context window exceeds limit (2013)`。根因是 `tool_result` 原文整段回灌模型没有截断；已收口为回灌前统一裁成 4000 字符，避免大文件/长技能说明直接顶爆上下文
  - 2026-03-10 排障补记：`local-vision-lmstudio` 一次失败不是模型本身挂，而是模型根据过度装配的 skill 文案，自己拼出了 `~/.config/msgcode/skills/local-vision-lmstudio/analyze_image.py` 伪路径。已把 vision skill 文案改成更接近 API 文档的“真实调用合同”，并移除系统提示里“统一走 main.sh”这一层过度暗示
  - 2026-03-10 排障补记：图片表格抽取再次暴露 runtime `vision` 的输出预算过小。LM Studio 日志显示同一请求 `finish_reason=length`、`completion_tokens=511`，而 `src/runners/vision.ts` 仍写死 `max_tokens: 500`，导致模型只来得及吐出前半段表格。已将视觉输出上限提升到 2048，并补测试锁防止回退
