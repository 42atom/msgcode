# Goal

把 memory retrieval 收成单独一刀：

- 读取时清楚区分 append 真相与 sqlite 索引
- listener/search 读索引，但不把索引说成真相
- 缺索引时的行为边界明确

## Non-Goals

- 不改 append 落盘格式
- 不扩 vector / hybrid 策略
- 不做大一统 memory service

## Result

- retrieval 主链已收成：
  - append 真相仍在 `<workspace>/memory/*.md`
  - listener / `memory search` 继续只读 sqlite 索引
  - 但“索引缺失”不再伪装成普通无结果
- 已落地边界：
  - `memory search --json` 在 workspace 下存在 memory 文件、但无索引记录时，返回 `warning` 和重建建议
  - listener 在同样场景下继续 fail-open，但会把 `memory 索引缺失` 写进 debug/warn，而不是只写“无搜索结果”
