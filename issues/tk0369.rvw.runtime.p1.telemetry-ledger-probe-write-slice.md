# 任务：probe telemetry writer

## Parent Task

- `tk0363`

## Context

现有 telemetry ledger 合同已经固定：

- 单一日账本：`<workspace>/.msgcode/telemetry/telemetry-YYYY-MM-DD.ndjson`
- append-only
- 最小字段
- 默认保留 7 天

`tk0367` 已把 tool bus 的稳定单一写点接进 ledger。
`tk0368` 已把 verify 主链的最薄写侧接进同一份 ledger。

这一刀只收 probe 主链的最薄写侧：

- `runAllProbes()`
- `runSingleProbe()`

不把 `runner / provider / inbound` 一起拉进来，不做 reader。

## Goal / Non-Goals

### Goal

- 让 probe 主链在单一稳定写点 append probe summary 到同一份日账本
- 保持现有 `status.log`、工具 ring buffer、failure snapshot 主链不变

### Non-Goals

- 不做 ledger reader
- 不改 `status.log`
- 不做 dashboard
- 不扩到 `runner / provider / inbound`
- 不碰 `tool` / `verify` writer slice

## Acceptance Criteria

1. `runAllProbes()` 和 `runSingleProbe()` 结束后，会把 probe summary 追加到 `telemetry-YYYY-MM-DD.ndjson`。
2. `TelemetryLedgerKind` 仍然是同一份合同，不新增第二套账本。
3. 现有 `tool` ring buffer 与 `/toolstats` 读法不变。
4. 不新增 server、数据库、控制面。

## Plan

- [ ] 确认现有 `tk0363` / `tk0367` / `tk0368` 不能直接承载 probe writer slice
- [ ] 新建最薄 `tk0369` 实现单
- [ ] 复用 `telemetry-ledger.ts` 接入 probe 主链
- [ ] 补最小回归，覆盖 `runAllProbes()` 与 `runSingleProbe()`

## Notes

- 这张单只收 probe writer，不做 reader。
- `runner / provider / inbound` 另拆子单，不回头把这张单写胖。
