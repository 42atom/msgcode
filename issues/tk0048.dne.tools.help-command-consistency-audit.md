## Context

- `src/routes/cmd-info.ts` 中的 `/help` 为群聊内主帮助入口。
- 真实可执行命令分散在 `src/routes/commands.ts`（路由命令）和 `src/handlers.ts`（会话/语音命令）。
- 用户要求核对 `/help` 是否与真实命令一致，并评估是否存在可简化命令面。

## Goal / Non-Goals

- Goal: 找出 `/help` 与真实命令面的不一致项，并修复低风险的帮助文案漂移。
- Goal: 给出命令简化的最小收口建议，但不擅自删除兼容命令。
- Non-Goals: 本轮不做大范围命令重命名，不改变已有命令行为，不合并多条入口链路。

## Plan

- [x] 梳理 `/help` 文案、路由层命令和处理器层命令的实际集合
- [x] 识别文案遗漏、参数错误与隐藏命令
- [x] 更新 `/help` 文案，补充回归测试
- [x] 输出审计结论与简化建议

## Acceptance Criteria

1. `/help` 中列出的群聊命令均可在代码中定位到真实处理入口。
2. 关键真实命令不再被 `/help` 漏掉。
3. 回归测试覆盖新增的帮助命令枚举。
