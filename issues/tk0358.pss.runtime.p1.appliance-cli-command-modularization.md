# 任务：appliance CLI command modularization

## Context

目前 runtime 已经按职责拆开，但 CLI 入口层没有跟上。

最近几条功能线：

- capability mutation
- people pending
- workspace tree
- thread surface

都继续往 `src/cli/appliance.ts` 里堆，已经明显失控。

## Goal / Non-Goals

### Goal

- 让 `appliance.ts` 只保留：
  - `new Command("appliance")`
  - 子命令注册
  - help docs 对接
- 把业务逻辑按命令域拆到子模块

### Non-Goals

- 不改 runtime 真相源
- 不改 Envelope 合同
- 不顺手重构其他 CLI

## Plan

- [x] 盘点现有 appliance 子命令边界
- [x] 拆 profile 子命令模块
- [x] 拆 capabilities 子命令模块
- [x] 拆 people 子命令模块
- [x] 拆 archive/thread 子命令模块
- [x] 保持 `appliance.ts` 只做注册

## Acceptance Criteria

1. `src/cli/appliance.ts` 不再承载大段业务逻辑。
2. profile/capabilities/people/archive/thread 至少按域分文件。
3. 现有 CLI 合同和测试行为不变。

## Notes

- 这刀收结构，不收功能。
- 不允许趁拆分顺手改动用户可见合同。
- 验证：
  - `bun test test/p7-tk0322-appliance-set-profile.test.ts test/p7-tk0315-appliance-capabilities-contract.test.ts test/p7-tk0296-appliance-people-contract.test.ts test/p7-tk0331-appliance-people-save.test.ts test/p7-tk0338-appliance-archive-contract.test.ts test/p7-tk0338-appliance-workspace-archive-restore.test.ts test/p7-tk0339-appliance-thread-archive-restore.test.ts test/p7-tk0355-appliance-workspace-tree-contract.test.ts test/p7-tk0356-appliance-thread-contract.test.ts`
