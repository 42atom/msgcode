## Context

当前 `normalizeLegacyCliArgs()` 里只剩两条历史兼容改写：

- `browser gmail-readonly -> browser-gmail-readonly`
- `file send -> file-send`

它们已经不再执行业务逻辑，只是把历史入口先改写到 retired compat root，再返回迁移提示。但这仍然是一层多余 CLI 预处理。

## Goal / Non-Goals

- Goal: 删除 `normalizeLegacyCliArgs()` 里最后两条兼容改写
- Goal: `browser gmail-readonly` / `file send` 直接由各自模块内隐藏 retired 子命令处理
- Goal: 保持 `browser-gmail-readonly` / `file-send` 旧根入口仍返回显式 retired 提示
- Non-Goals: 本轮不恢复任何历史业务能力
- Non-Goals: 本轮不改 browser/file 正式公开合同
