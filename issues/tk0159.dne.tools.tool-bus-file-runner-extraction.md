## Context

`src/tools/bus.ts` 里剩余最厚的一块是 `read_file / write_file / edit_file`：

- 路径解析
- `fs_scope` 越界拒绝
- 二进制探测
- 大文件 preview
- 文件读写本体
- `edit_file` 补丁语义与简写兼容

这已经不是总线闸门，而是文件域的完整执行实现。

## Goal / Non-Goals

- Goal: 把文件工具实现与边界处理外移到 runner
- Goal: 保持 `fs_scope`、binary guard、大文件 preview、edit patch 语义不变
- Goal: bus 只保留工具 case、preview 组装与结构化结果返回
- Non-Goals: 本轮不改 manifest
- Non-Goals: 本轮不改文件工具对外参数/结果合同

## Acceptance Criteria

1. bus 中不再直接包含 read/write/edit 的文件 IO 与 `fs_scope` 处理细节
2. `read_file` 二进制/大文件/边界行为不回归
3. `edit_file` 补丁模式与 oldText/newText 简写兼容不回归
4. 相关测试、`npx tsc --noEmit`、`npm run docs:check` 通过
