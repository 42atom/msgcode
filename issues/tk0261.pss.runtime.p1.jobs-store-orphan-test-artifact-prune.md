## Context

- 真实 `~/.config/msgcode/cron/jobs.json` 里还残留一条：
  - `id = test:job`
  - `route.chatGuid = test-chat`
- 它不是当前 schedule 真相源投影出来的 job，也没有对应 route 真相。
- 这类测试残留继续留在 `jobs.json`，会污染：
  - scheduler 启动
  - probe / info
  - 对真实 runtime 状态的判断

## Structure Review

- `jobs.json` 既然是执行投影，就不该长期留测试垃圾。
- 这不是调度编排问题，不需要新控制层。
- 最薄收口就是：启动前删掉“明显测试 job + 没有 route 真相”的残留。

## Goal / Non-Goals

### Goal

- 删除没有 route 真相的明显测试 job。
- 保留仍有 route 真相的测试 job，避免误删测试隔离场景。
- 让 scheduler 启动前先把 `jobs.json` 收回真实执行面。

### Non-Goals

- 不重写 job store 格式。
- 不删除正常 orphaned job。
- 不扩展更多猜测式清理规则。

## Plan

- [x] 在启动链接入 test-only job prune
- [x] 补专属回归，锁住删与不删两类行为
- [x] 清掉真实 `test:job` 残留

## Acceptance Criteria

1. `test:*` 或 `test-` route 的明显测试 job，在没有 route 真相时会被删除并落盘。
2. 仍有 route 真相的测试 job 不会被误删。
3. scheduler 启动前会执行这次清理。
