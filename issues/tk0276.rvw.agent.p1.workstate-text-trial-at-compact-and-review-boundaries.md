# 任务：WORKSTATE 纯文本试跑

## Context

`WORKSTATE` 协议已经存在，但现役主链还不知道：

- 什么时候该写
- 写到哪里
- 哪些任务不该写

先用 prompt 合同试跑，比先改 runtime 更薄。

## Goal / Non-Goals

### Goal

- 让 agent 在关键边界按协议写 `WORKSTATE`
- 先验证 `WORKSTATE` 是否真能帮助续接

### Non-Goals

- 不做 runtime 自动写盘
- 不做 graph/context 自动重建
- 不把 `WORKSTATE` 变成第二真相源

## Plan

- [x] 明确试跑边界：只在 compact 前和 `doi -> rvw` 前
- [x] 把写入位置、模板来源、覆盖规则写进 prompt 合同
- [x] 明确哪些轻任务不写 `WORKSTATE`

## Acceptance Criteria

1. agent 知道何时写 `WORKSTATE`。
2. agent 知道写入路径和模板来源。
3. 试跑仍然停留在纯文本协议层，不引入自动化控制面。
