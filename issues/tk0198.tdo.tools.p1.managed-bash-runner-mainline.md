## Context

- 当前 `bash` 工具对外名义是 Bash，但运行时仍通过 `shell: true` 执行。
- 在当前 macOS 环境里，这条链实际落到 `/bin/sh`，不是显式 Bash。
- 用户目标已经明确：一次性把工具装到位，减少模型后续自己试探 shell 差异。
- 因此第一刀应该先把运行时合同收口成“托管现代 Bash”，而不是继续放任 shell 漂移。

## Goal / Non-Goals

- Goal:
  - `bash` runner 显式执行 Homebrew Bash
  - 缺失时返回真实物理边界错误
  - 增加测试锁住 shell 路径与错误行为
- Non-Goals:
  - 不引入 `/bin/sh`、`zsh`、系统 `/bin/bash` 3.2 的自动 fallback
  - 不改工具名
  - 不在本单处理 bootstrap 文档与 prompt 文案

## Plan

- [ ] 抽出托管 Bash 路径解析逻辑，只承认 Homebrew Bash 位置
- [ ] `bash-runner` 改为 `shell: false + <bash> -lc <command>` 显式执行
- [ ] 缺失托管 Bash 时返回可诊断错误，包含建议安装路径或命令
- [ ] 增加回归测试：锁住执行路径、版本特征和缺失错误
- [ ] 验证 Tool Bus 返回结构不回归

## Acceptance Criteria

1. [src/runners/bash-runner.ts](/Users/admin/GitProjects/msgcode/src/runners/bash-runner.ts) 不再使用 `shell: true`。
2. `bash` 工具只承认 `/opt/homebrew/bin/bash` 或 `/usr/local/bin/bash`。
3. 托管 Bash 缺失时，工具失败且错误可诊断，不发生 shell 自动回退。
4. 测试覆盖至少包含：
   - 执行器使用的 shell 路径
   - `BASH_VERSION` 为托管 Bash 返回
   - 缺失托管 Bash 的失败路径
