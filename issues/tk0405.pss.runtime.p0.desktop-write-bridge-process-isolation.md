# 任务：Desktop Write Bridge Process Isolation

## Context

当前真相是：

1. renderer 调 `sendThreadInput()`
2. main process 直接进 runtime
3. runtime 再起 agent 执行

这让 Electron main process 背了重执行链。

## Goal / Non-Goals

### Goal

把这条主链收成：

- renderer -> `sendThreadInput()`
- main process -> 快速写入 user turn
- detached CLI child -> agent 执行 + assistant writeback

### Non-Goals

1. 不做流式
2. 不做中断
3. 不改右栏
4. 不改 composer 行为合同

## Plan

1. 先补 CLI 写桥入口
2. 再让 Electron main 改走 detached spawn
3. 最后补最小验证

## Acceptance Criteria

1. `sendThreadInput()` 签名不变
2. Electron main process 不再直接承载 agent 执行
3. 子进程复用现有 CLI/runtime 壳
4. 保持“先写 user、立即返回、异步补 assistant”

## Notes

### 必做

1. 保持 `sendThreadInput({ workspacePath, threadId, text })`
2. 主进程不再直接 `getHandler("agent-backend").handle()`
3. 子进程复用现有 CLI/runtime 壳，不另造 worker 平台
4. 保持“先写 user、立即返回、异步补 assistant”
