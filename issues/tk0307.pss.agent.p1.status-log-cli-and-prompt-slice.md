# 任务：status.log CLI 与 prompt 切片

## Context

`0292` 已收成共享近况簿，`0294` 已收成 append/tail 运行时合同。

现状还缺一刀：

- 主脑没有正式 CLI 出口去读/写 `status.log`
- prompt 里还没把 `status.log` 接进主链

## Goal / Non-Goals

### Goal

- 增加最小 `msgcode status-log add`
- 增加最小 `msgcode status-log tail`
- 把合同接进 `help-docs`
- 把 `status.log` 读写口径接进 `agents-prompt`

### Non-Goals

- 不实现自动 hook
- 不实现定时写入
- 不新增去重器
- 不让 UI 变成第二真相源

## Plan

- [x] 增加 `status-log` CLI
- [x] 增加 add/tail 黑盒测试
- [x] 增加 prompt 合同锁
- [x] 做一轮圆桌复核

## Acceptance Criteria

1. 主脑不再需要手写 `status.log` 原始行。
2. 读取共享近况有正式 CLI 合同，而不是临时 `tail` 拼接。
3. 这刀只补主链入口，不长新机制。
