## Context

当前 `vision` 处于一个不干净状态：

- `DEFAULT_WORKSPACE_CONFIG["tooling.allow"]` 默认包含 `vision`
- 但 `getToolsForLlm()` 与 `canExecuteTool()` 又会通过 `filterDefaultLlmTools()` 静默把它抹掉

这意味着 `tooling.allow` 不是单一真相源，运行时仍存在“配置允许、执行层偷偷再过滤”的第二裁判。

## Goal / Non-Goals

### Goal

- 让 `tooling.allow` 重新成为 vision 工具暴露的唯一真相源
- 如果默认不想暴露 `vision`，则在默认配置中显式移除，而不是运行时暗中过滤
- 保留 `vision` 的显式 opt-in 能力：workspace 明确允许时，LLM 真实看到并能调用

### Non-Goals

- 本轮不处理 `mem`
- 本轮不重构 vision skill 文案
- 本轮不新增任何工具层/策略层

## Plan

- [x] 从默认 `tooling.allow` 中移除 `vision`
- [x] 移除 `vision` 的隐藏 suppress 路径
- [x] 保持 `mem` 暂时不变
- [x] 更新相关测试，改锁“默认无 vision、显式 allow 则真暴露”
- [x] 更新 CHANGELOG

## Acceptance Criteria

- 默认配置下 `policy.allow` 与 `getToolsForLlm()` 都不再包含 `vision`
- workspace 显式 `tooling.allow=["vision", ...]` 时，`getToolsForLlm()` 返回 `vision`
- `canExecuteTool()` 对显式 allow 的 `vision` 不再额外拒绝
- 测试、`tsc`、`docs:check` 全通过
