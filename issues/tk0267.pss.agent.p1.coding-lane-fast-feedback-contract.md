# 任务：Coding Lane 快反馈合同

## Context

当前 Agent 已有执行能力，也能手动跑验证。

缺口是：

- 对代码任务来说，“改代码后立即验证”还不是正式合同

## Goal / Non-Goals

### Goal

- 把 coding lane 的默认执行节奏写清楚
- 明确哪些验证属于默认反馈内环
- 明确停止条件只保留安全 / 预算 / 物理边界

### Non-Goals

- 不做系统自动循环重试
- 不改 runtime 执行器

## Plan

- [x] 盘点当前 prompt / skill / AGENTS 里的相关口径
- [x] 写清代码任务默认反馈循环
- [x] 对齐 `tk0266` 的负向错误口径

## Acceptance Criteria

1. coding lane 默认节奏明确为：改代码 -> 验证 -> 看真实错误 -> 再修。
2. 默认验证面至少覆盖：`test / types / e2e / custom verify`。
3. 停止条件不扩展为新的系统裁判层。

## Notes

- 这张单先收 Agent 合同，不先写自动修复器。
- `tk0266` 的真实错误口径应作为本单前提，而不是平行冲突口径。
