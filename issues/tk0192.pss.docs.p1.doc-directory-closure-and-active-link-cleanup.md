## Context

文档文件名协议与迁移后协议收尾已基本完成，但目录层和正文层还需要最后一刀收口：

- `docs/plan/` 已经是正式文档主目录
- `docs/design/`、`docs/notes/` 已退役，但目录壳和说明仍可能继续制造“现役入口”错觉
- 现役文档正文里仍可能残留 `docs/design/`、`docs/notes/` 历史路径
- `docs:check` 虽已恢复通过，但还缺“禁止退役目录重新回流”的硬门

这一步不是再改文件名，而是把目录协议收成单一主链。

## Goal / Non-Goals

- Goal:
  - 让 `docs/plan/` 成为唯一正式 plan/research/ref/report 入口
  - 清掉现役文档中的旧目录路径
  - 让 `docs:check` 明确阻止 `docs/design/`、`docs/notes/` 回流
- Non-Goals:
  - 不再做第二轮大规模 rename
  - 不要求一次性重写 archive 历史正文
  - 不修改与文档协议无关的代码逻辑

## Plan

- [x] 收口目录协议：决定 `docs/design/`、`docs/notes/` 是删除还是只留退役 README
- [x] 清理现役文档旧路径：优先 `issues/`、`docs/tasks/`、`docs/plan/`、入口文档
- [x] 收紧 `docs:check`：禁止新增退役目录引用与退役目录新文件
- [x] 验收：目录、正文、校验三层一致

## Acceptance Criteria

1. `docs/plan/` 是唯一正式文档入口，`docs/design/`、`docs/notes/` 不再被描述为现役目录。
2. 现役文档中的 `docs/design/`、`docs/notes/` 历史路径清零或仅剩明确退役说明。
3. `docs:check` 对退役目录回流 fail-closed。
4. `npm run docs:check` 继续通过。
