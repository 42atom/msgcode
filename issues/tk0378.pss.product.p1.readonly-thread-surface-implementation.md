# 任务：Readonly Thread Surface Implementation

## Context

`tk0377` 已经把这刀的主线钉死：

- 左栏接 `workspace-tree`
- 正文和右栏接 `thread`
- UI 只保留 `selectedWorkspace / selectedThreadId / loadingError`

但仓库里还没有一张承载实现状态和验证证据的现役 task。

如果继续把实现塞回 `tk0377`，会把：

- 主线边界
- 代码落点
- 验证结果

重新混成一张单。

## 为什么现有文件不能直接承载实现职责

`tk0377` 是主线单，只钉边界和为什么要开这一刀，不承载代码落点、验证命令和实现证据。

`tk0293` 是数据合同单，只定义读什么，不负责把页面改到只读。

`tk0355` 和 `tk0356` 是 runtime 读面单，只保证 CLI 合同成立，不负责主窗口状态收口。

`rf0001` 是参考，不是执行单。

`ui-protype/main.html` 是页面文件，不该混入任务边界和验收说明。

所以这刀需要单独一张很薄的实现单，把“该改哪、改到什么程度、怎么验”钉死。

## Goal / Non-Goals

### Goal

1. 左栏只消费 `msgcode appliance workspace-tree --json`
2. 正文和右栏只消费 `msgcode appliance thread --workspace <workspace> --thread-id <threadId> --json`
3. UI 本地只保留：
   - `selectedWorkspace`
   - `selectedThreadId`
   - `loadingError`

### Non-Goals

1. 发消息
2. 新建聊天
3. archive
4. settings 内容
5. 额外本地缓存
6. 排序/拖拽
7. Electron 大壳

## Plan

- [x] 把 `main.html` 收成只读线程面三栏骨架
- [x] 把 `main.js` 收成 `workspace-tree -> thread` 两步读面联通
- [x] 删除假数据、composer、新聊天、archive 入口
- [x] settings 只保留一级入口 affordance，不把内容并回主窗口
- [x] 增加最小测试锁住只读边界

## Acceptance Criteria

1. `main.html` 不再保留 archive、new chat、composer/send 入口；`settings` 只保留独立入口 affordance。
2. `main.js` 只围绕 `workspace-tree` 和 `thread` 两条读面组织状态与渲染。
3. 至少有 1 个最小测试锁住“只读收口”边界。

## Notes

- `main.js` 仍留在原文件，没有内联进 `main.html`。
- 这刀只收实现边界，不重做样式系统。
- 当前页面通过 `window.msgcodeReadonlySurface.runCommand` 取两条正式读面；桥接缺失时直接报真实错误，不再回退假数据。
- 已约定的一级用户入口属于用户侧真相；实现切片不得静默删除，只能保留 affordance 或单独迁移说明。
