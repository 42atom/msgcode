## Context

- 当前文档和提示词都频繁使用 `bash` 这个词，但没有把“到底是哪一份 Bash”写死。
- 只要文案继续模糊，模型就会继续把 `sh`、`bash`、`zsh` 当成可互换概念。
- 用户的目标不是给模型更多选择，而是减少试探空间。

  - 把“先检查 shell 合同再写脚本”的口径写进 agent 提示
- Non-Goals:
  - 不新增新的 tool 面或 skill
  - 不重写整份 README
  - 不扩展成通用 shell 教程
