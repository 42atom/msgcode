# 任务：工作区 people 写边界基线

## Context

当前已成立的是：

- `.msgcode/character-identity/*.csv`
- `.msgcode/people-pending.json`
- `msgcode appliance people --workspace <...> --json`

但原型还多了两类动作：

- 存入通讯录
- 编辑统一称谓 / 备注 / tags

如果不先收写边界，后面很容易出现：

- UI 直接改 CSV
- `pending` 和已知人物各写各的
- 为了 tags 新起一套 `people.json`

## Goal / Non-Goals

### Goal

- 固定 people 第一版唯一键
- 固定第一版真正允许写的字段
- 固定 `pending` 与通讯录的关系
- 固定最薄 CLI 写入口

### Non-Goals

- 不实现飞书拉取
- 不实现自动匹配
- 不实现 tags 真相源
- 不实现全局人物中心

## Plan

- [ ] 固定唯一键与目标 CSV 路径
- [ ] 固定第一版可写字段
- [ ] 固定 pending 过滤而非双写删除的规则
- [ ] 固定最薄 `people save` CLI 合同

## Acceptance Criteria

1. 第一版 people 写入不需要 UI 直接改文件。
2. 不再出现 `people.json` 这类新人物中心。
3. `pending` 不会和已知人物长期重复显示。

## Notes

- 这张单只收写边界，不直接上实现。
- 目标是做成最薄的 append/update 主链，不加人物平台。
