## Context

虽然我们已经删掉了静默改参和最终答复自动重试，但 `tool-loop` 与 `runAgentChat()` 仍然会在最终交付前调用 `sanitizeLmStudioOutput()`。这会继续删除 `<think>`、工具协议片段、元叙事和部分 JSON-ish 内容，相当于系统还在替模型重写最后交付。

## Goal / Non-Goals

### Goal

- 让最终交付给用户的答案不再经过系统清洗
- 保留清洗器给错误 snippet / 诊断等内部用途
- 更新回归锁，明确用户看到的是模型原样输出

### Non-Goals

- 本轮不删除 `sanitizeLmStudioOutput()` 模块本身
- 本轮不改错误提示中的 snippet 清洗
- 本轮不动 `runVerifyPhase` / `finish supervisor`

## Plan

- [x] 新建 `0111` issue 与对应 plan，冻结边界
- [x] 移除 `tool-loop` 与 `runAgentChat` 返回路径上的 `sanitizeLmStudioOutput()`
- [x] 更新行为锁，改为验证 `<think>` / tool_call 协议片段不再被系统抹掉
- [x] 跑定向测试、类型检查和 docs:check

## Acceptance Criteria

- `tool-loop` 返回的 `answer` 为模型原样输出
- `runAgentChat()` 返回的聊天结果为模型原样输出
- 清洗器仍只用于错误 snippet / 兼容辅助，不再干预最终交付
