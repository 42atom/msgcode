# 任务：Provider failure evidence handoff

## Context

当前 provider 失败会：

- 进入运行日志
- 进入 listener 失败分支
- 结束本轮 run

但不会进入模型下一轮可见上下文。

结果是：

- 模型看不到刚刚发生的上游失败
- 下一轮仍按旧认知继续回答
- 用户知道失败，模型却不知道失败

## Goal / Non-Goals

## Goal

- 把 provider 失败证据沉淀为可引用事实
- 让下一轮模型能看到最近一次相关 provider 失败
- 保持 fail-fast，不掩盖原始错误

## Non-Goals

- 不新增自动 retry
- 不新增 fallback
- 不新增 repair middleware
- 不把 provider 错误改写成成功结果
- 不引入新的控制层或 supervisor

## Plan

1. 定义最小失败证据字段：
   - `provider`
   - `model`
   - `upstream_status`
   - `error_type`
   - `message`
   - `request_id`
   - `run_id`
   - `occurred_at`
2. 明确第一刀落盘位置与读取位置
3. 明确只回流最近一次相关 provider 失败
4. 明确回流内容是只读事实，不附带系统代判

## Acceptance Criteria

- 失败证据字段固定、够薄、够用
- 下一轮模型能看到最近一次 provider 失败事实
- 第一刀不改变 provider 请求主链
- 不引入 retry / fallback / repair

## Notes

- 这是 evidence handoff，不是错误恢复层。
- 失败依旧先暴露；新增的只是“把失败事实回流给模型”。
