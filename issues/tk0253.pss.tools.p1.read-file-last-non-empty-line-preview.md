## Context

- `tk0252` 已把 large `read_file` 从 head-only 改成 head+tail
- 2026-03-19 真实 Feishu smoke 中，模型不再爆窗，但对“最后一条非空行”仍只回 token，没有稳定带出 tail line
- 根因不是 `read_file` 失败，而是 provider 仍要从 tail 文本自行推理最后一条有效行

## Goal

- 在截断 preview 里直接暴露 `lastNonEmptyLine`
- 保持 replay 预算不变，仍不超过 512 chars

## Non-Goals

- 不改小文件 inline 行为
- 不扩 `read_file` 参数
- 不加新的文件工具或控制层

## Goal / Non-Goals

- Goal:
  - truncated preview 显式给出 `lastNonEmptyLine`
  - 保持 replay 限额内的 head/tail 结构
- Non-Goals:
  - 不改变 `read_file` 返回 schema
  - 不把文件阅读问题转成 bash-only

## Plan

- [x] 在 truncated preview 中提取 tail 的最后一条非空行
- [x] 补专属回归
- [x] 用真实 Feishu smoke 复核

## Acceptance Criteria

1. truncated `read_file` preview 包含 `lastNonEmptyLine`。
2. preview 总长度仍不超过 `512` chars。
3. 真实 Feishu smoke 中，最后一条非空行问题能稳定命中 tail token。

## Notes

- 这刀继续修的是 `read_file` replay 合同，不是 provider 温度。
- 最薄办法不是扩 API，而是把已经读到的 tail 事实显式化。

## Result

- truncated `read_file` preview 现在显式带出 `[lastNonEmptyLine]`
- exec system prompt 增加两条窄规则：
  - `[lastNonEmptyLine]` 是尾部最后一条非空行
  - 文件事实题必须先 `read_file / bash` 再回答
- 真实 Feishu smoke 已从“只回 token / [EOF] / no-tool”收口到“真实 `read_file` 后回出整条 tail line”
