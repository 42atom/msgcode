## Context

`batch1` live 结果已经给出明确证据：

- `skill-live-03`（文件回传）没有优先走 `feishu_send_file`，而是退回 `bash`
- `skill-live-05`（浏览器标题）没有优先走 `browser` 工具，而是退回 `bash`

这说明当前主问题不是运行时主链完全坏掉，而是 **skill 引导还不够强，模型仍把 bash/CLI 当默认路径**。

## Goal / Non-Goals

- Goal: 收紧 `feishu-send-file` skill 文案，明确原生工具优先、禁止绕回 bash/CLI
- Goal: 收紧 `patchright-browser` skill 文案，明确浏览器任务默认优先走 `browser` 工具
- Goal: 同步 runtime index 描述，让全局技能发现层也传达同一口径
- Goal: 如果 skill 文案仍不足以纠偏，用最小提示把执行核也收口到“原生工具优先”
- Goal: 同步用户目录 skill 后，复测 `skill-live-03` / `skill-live-05`
- Non-Goals: 本轮不新增控制层
- Non-Goals: 本轮不新增新的编排/裁判层
