## Context

当前 `msgcode` 仍公开 `web search` 与 `web fetch`：

- `web search` 只是 MiniMax MCP `web_search` 的 CLI 转述
- `web fetch` 只是 `webReader` 的 CLI 转述

它们没有成为 msgcode 独有的桥接能力边界，而是在把上游能力再包一层子命令、错误码和 envelope。这与前面已经退掉的 `file/system` 同属“二手壳”问题。

## Goal / Non-Goals

- Goal: 将 `web search/fetch` 从公开 CLI 主链与 `help-docs` 合同退役
- Goal: 保留最薄 retired compat shell，引导回原生浏览器、bash 或上游原生能力
- Goal: 更新相关测试与 changelog
- Non-Goals: 本轮不处理 browser 原生工具
- Non-Goals: 本轮不新增新的 web 工具层

## Plan

- [x] 创建 Plan 文档，冻结退役 `web` domain 的决策
- [x] 将 `src/cli/web.ts` 收口为 retired compat shell
- [x] 更新 `src/cli.ts` 与 `src/cli/help.ts`，不再公开 `web`
- [x] 更新相关测试并执行 targeted tests、`npx tsc --noEmit` 与 `npm run docs:check`
- [x] 更新 issue notes 与 `docs/CHANGELOG.md`

## Acceptance Criteria

1. `msgcode --help` 不再公开 `web`
2. `msgcode help-docs --json` 不再暴露 `web search/fetch`
3. `msgcode web search ...` 与 `msgcode web fetch ...` direct invoke 返回 retired 提示
