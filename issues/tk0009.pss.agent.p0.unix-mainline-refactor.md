## Context

- `AIDOCS/architecture/personal-agent-unix-audit-260306.md` 已明确指出 5 处多余设计：
  - 前置 route classifier
  - fake-tool recover 补丁链
  - `task-supervisor` 平行执行入口
  - `steering-queue` 隐式恢复副作用
  - `commands.ts` 运行时 God Object
- 用户已给出操刀顺序建议：先删后降后拆。
- 当前仓库存在大量未提交改动，本轮只做可回滚的小步收口，不回退他人修改。

## Goal / Non-Goals

- Goal: 把智能体执行链收口到“一个主智能体 + 一条执行入口 + 一份状态真相源”。
- Goal: 第一阶段先完成 `routed-chat.ts` 的纯减法清理，并验证 agent-first 主链稳定。
- Non-Goals: 本轮不顺手重写 `handlers.ts`；不修改 browser/tool 合同；不调整默认配额策略。

## Plan

- [x] 创建/更新 Plan 文档，冻结“删 → 降 → 拆”顺序
- [x] 第一刀：删除 `src/agent-backend/routed-chat.ts` 中前置 classifier 与 recover 残骸
- [x] 跑最小验收，确认主链仍支持直答和 tool-call
- [x] 第二刀：把 `task-supervisor` 降级为状态机/调度器，执行器改为外部注入
- [x] 第三刀：把 `steering-queue` 恢复改成显式调用
- [x] 第四刀：拆薄 `commands.ts`（Batch 1-4 完成）

## Acceptance Criteria

1. `src/agent-backend/routed-chat.ts` 不再保留前置 route classifier 与本文件内 fake-tool recover 逻辑。
2. 最小验收能证明 agent-first 主链仍可：
   - 无工具直答
   - 需要工具时发起 `tool_calls`
3. issue/plan 中明确记录后续三刀的边界与风险。
