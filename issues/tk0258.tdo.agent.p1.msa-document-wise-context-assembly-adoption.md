# 任务：吸收 MSA 的文档级编码方式，形成 msgcode 上下文拼装计划

## Context

### 背景

用户希望参考 EverMind MSA 项目中 `document-wise RoPE` 与 `online routing & context assembly` 的设计，
为 msgcode 提炼一套更清晰的“文件级编码 / 文档边界 / 全局顺序”方案。

重点不是照搬模型训练细节，而是吸收其核心结构思想：

- 文档内部顺序与文档之间顺序分开表达
- 不把多份资料压成一条扁平长串
- 检索、拼装、生成三段职责清楚

## Goal / Non-Goals

### Goal

- 基于 MSA 官方 GitHub README 的公开信息，提炼可用于 msgcode 的结构原则
- 对照 msgcode 当前 `SOUL / memory / summary / window / attachment` 注入主链，识别差距
- 产出一份最小可落地 plan，指导后续实现

### Non-Goals

- 不实现模型层 RoPE
- 不伪造 MSA 未公开的训练或推理源码细节
- 不新增“大一统 memory platform”或控制面

## Plan

- [ ] 阅读 MSA 公开 README 与官方说明，提炼可借鉴原则
- [ ] 对照 msgcode 当前上下文主链，标出相似点与缺口
- [ ] 输出分阶段 plan，并明确每阶段非目标

## Acceptance Criteria

1. 任务正文明确区分 Goal 与 Non-Goals。
2. 任务能指出 MSA 结构思想里哪些能借鉴、哪些不能直接照搬。
3. 有清晰的阶段化计划，而不是泛泛架构愿景。
