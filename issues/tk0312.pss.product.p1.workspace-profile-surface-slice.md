# 任务：工作区“我的资料”读面切片

## Context

`0263` 已经把 Settings 页冻结成：

1. 我的资料
2. 通用
3. 智能体能力配置
4. 诊断

其中“我的资料”目前只有原型，没有运行时真相源出口。

现有文件已经覆盖了其中两段：

- `SOUL.md`
- `ORG.md`

真正缺的只有：

- 我的称呼

## Goal / Non-Goals

### Goal

- 增加最小 `msgcode appliance profile --workspace <...> --json`
- 让它输出：
  - 我的称呼
  - SOUL
  - 组织名称
  - 位置城市
- 位置城市兼容旧字段 `交税地`

### Non-Goals

- 不做 settings 写入
- 不做“通用 / 诊断 / 能力配置”整页出口
- 不新建 `profile.json`
- 不复制 ORG/SOUL 内容到第二份文件
- 不做仓库级 ORG 字段迁移

## Plan

- [x] 在 workspace config 中只补一个标量字段 `profile.name`
- [x] 新增 workspace-profile 读面
- [x] 新增 appliance profile CLI
- [x] 补黑盒测试
- [x] 做一轮圆桌复核

## Acceptance Criteria

1. `appliance profile` 能输出稳定 JSON。
2. 缺少 `profile.name` / `SOUL.md` / `ORG.md` 时返回 warning 空态。
3. `ORG.md` 只有旧字段 `交税地` 时，仍能映射到 `city`。
4. 没有新建第二份 profile/settings 真相源。

## Notes

- `profile.name` 落在 `<workspace>/.msgcode/config.json`
- SOUL 继续读 `<workspace>/.msgcode/SOUL.md`
- 组织信息继续读 `<workspace>/.msgcode/ORG.md`
- 这条切片是 Settings 页“我的资料”的只读底座，不等于 settings 总线
