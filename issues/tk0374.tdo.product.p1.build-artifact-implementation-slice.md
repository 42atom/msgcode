# 任务：Build Artifact Implementation Slice

## 为什么现有文件不能直接承载实现职责

先把边界说死。

### `tk0370`

它是 packaging assembly 主线。

职责是：

- 收口 `artifact -> app -> dmg` 三段顺序
- 固定输入、输出和阶段边界

它不该直接承载实现。

原因：

- 一旦把 artifact 实现塞回 `tk0370`，主线文档会重新混入下游细节
- `app bundle` 与 `dmg` 的非目标会被实现噪声冲掉

### `tk0371`

它是 artifact assembly 合同。

职责是：

- 定义 artifact 承认哪些输入
- 定义 `bundle-root` 最小结构
- 定义 app host 只能消费 artifact

它回答的是“装什么进去”，不是“第一刀实现怎么落”。

如果直接把实现职责塞进来，会出现两个问题：

- 合同文档和实现推进状态混在一起，后续 review 很难判断是合同变更还是实现变更
- 同一张单会同时背“定义 artifact”和“追踪实现切片”，职责不再单一

### `pl0362`

它是 plan，不是实现单。

职责是：

- 给主线排顺序
- 给子任务切边界

plan 负责导航，不负责承载具体实现切片。

### `tk0265`

它只定义 bootstrap 主线。

职责是：

- 去 `brew`
- 去 `npm install`
- 固定 appliance 安装、初始化、升级入口

它可以作为 artifact 的上游输入合同，但不能直接承载 Desktop build artifact 实现职责。

否则会把：

- appliance bootstrap
- Desktop artifact build

重新揉成一层。

## Goal / Non-Goals

### Goal

- 单独承载 build artifact 的第一张实现单
- 把实现输入限制为 `tk0371` 已定义好的 artifact 合同
- 把实现输出限制为一份可验证的 `bundle-root`
- 把验证限定在 artifact，不越界到 `.app` 或 `dmg`

### Non-Goals

- 不写 app bundle host implementation
- 不写 dmg packaging / signing / notarization
- 不重写 `tk0371` artifact 合同
- 不扩授权、自动更新、跨平台安装器

## Implementation Slice

这张单只做 1 刀：

- 把 `tk0371` 文档里定义好的 artifact 合同，落成一条可执行的 build artifact 生成路径

最小范围只到：

1. 产出 `bundle-root/runtime/`
2. 产出 `bundle-root/bin/msgcode`
3. 产出 `bundle-root/bootstrap/`
4. 产出 `bundle-root/appliance.manifest`

不再多做：

1. `.app` 宿主装配
2. `dmg` 分发容器
3. 签名
4. notarization

## 输入

只承认这 2 类现有输入：

1. `tk0371` 已定义的 runtime 交付面
2. `tk0265` 已定义的 appliance bootstrap 脚本与 manifest 合同

下游不允许直接回读：

1. `src/**`
2. `test/**`
3. `package.json`
4. `bootstrap/Brewfile*`
5. app host / dmg 文档线

## 输出

这张实现单完成时，只认一份 artifact 输出：

- `bundle-root/`

验收只看这份产物是否成立，不提前要求：

- `.app`
- `dmg`
- 签名
- notarization

## Plan

- [ ] 固定 build artifact implementation 的唯一入口与输出目录
- [ ] 落地 `bundle-root` 所需的 runtime、launcher、bootstrap、manifest 生成
- [ ] 验证下游可以只消费 artifact，而不是重新回退到源码树
- [ ] 把实现证据留在对应 report，不把 report 职责混进本 task

## Acceptance Criteria

1. 能明确说明为什么实现单必须独立于 `tk0370 / tk0371 / pl0362 / tk0265`。
2. 实现范围只覆盖 build artifact，不进入 app bundle 或 dmg。
3. 完成后至少能稳定产出一份 `bundle-root`，且内容边界与 `tk0371` 一致。
4. 验证口径只检查 artifact，不把 `.app` / `dmg` 成果提前当成验收条件。

## Notes

- 这是 Desktop packaging 的第 1 张实现切片。
- 这张单只承载 artifact implementation，不替 app bundle 和 dmg 占位。
