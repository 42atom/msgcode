## Context

日志已经证明：

- 飞书入站链拿得到当前 `senderId`
- 当前 bot 没有“获取群成员列表”的工具能力
- 当前应用权限实际上已开，真实接口可返回成员 ID 和姓名

因此问题不在飞书后台权限，而在 msgcode 尚未薄接这一条只读工具能力。

## Goal / Non-Goals

### Goal

- 新增最小 `feishu_list_members` 工具
- 让模型能用真实 `member_id + name` 建立 `character-identity` 对照表
- 在提示词中明确：飞书群聊定向对某人说话时，已知 ID 则应 `@` 对方

### Non-Goals

- 不做飞书成员平台
- 不做全局通讯录
- 不做核心人物识别层
- 不新增 gateway / manager / queue

## Plan

- [x] 新增 `feishu_list_members` 工具实现
- [x] 注册 manifest / bus / types / 默认工具 allow
- [x] 提示词增加飞书群聊定向回复 `@` 规则
- [x] `character-identity` skill 增加通过成员列表初始化通讯录的说明
- [x] 补回归测试
- [x] 更新 changelog

## Acceptance Criteria

- LLM 可调用 `feishu_list_members`
- 工具返回 `senderId + name`
- chatId 可从 workspace 当前会话上下文回填
- 失败时错误信息会提示权限未开或机器人不在群
- 系统提示词明确飞书群聊定向说话时的 `@` 规则
