# 任务：wpkg zstd lazy resolution and extract cleanup

## Context

`workspace-wpkg-install.ts` 现在有两处不够干净：

- 模块加载时就同步执行 `resolveZstdBinary()`
- 解包走 `/bin/sh -c "... | tar ..."`

能跑，但不符合现在这条薄主线。

## Goal / Non-Goals

### Goal

- `zstd` 只在真正安装 wpkg 时解析
- 解包改成显式进程/pipe，不靠 shell 拼接

### Non-Goals

- 不改 `.tar.zst` 主链
- 不改 WPKG manifest
- 不改 packs/sites 注册

## Plan

- [x] 去掉顶层 `const ZSTD_BIN = ...`
- [x] 把 `zstd` 解析收成懒执行
- [x] 改造解包流程，去掉 `/bin/sh -c`
- [x] 保持现有测试通过

## Acceptance Criteria

1. `workspace-wpkg-install.ts` 模块加载不再同步探测 zstd。
2. 解包不再依赖 `/bin/sh -c`。
3. 现有 wpkg 安装主链与测试不回退。

## Notes

- 这是路径清理，不是安装协议重写。
- 不为“更通用”引入新的打包器层。
- 验证：
  - `bun test test/p7-tk0305-wpkg-install-register.test.ts`
  - `./node_modules/.bin/tsc --noEmit`
