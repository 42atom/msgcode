## Context

- 文件 wake 不是把事件从内存搬到磁盘就结束；还要解决双消费、重启续跑和文件泄漏。

## Goal / Non-Goals

- Goal:
  - 补 wake record claim 语义
  - 补 startup catch-up 扫描 overdue/pending 记录
  - 固定终态记录保留窗口与回收时机
  - 定义 poison/incident 升级边界
- Non-Goals:
  - 不先做 task 恢复胶囊
  - 不先改 heartbeat 提示词
  - 不先做全局 GC/数据库

## Plan

- [x] 定义同一 wake record 的单消费者 claim 规则
- [x] 定义 startup catch-up 扫描：stale running / overdue jobs / pending records
- [x] 定义终态保留窗口：默认 `7d`，失败追查可延至 `30d`
- [x] 定义 poison/incident 升级，不允许 `claimed -> stale -> reclaim` 静默循环

## Acceptance Criteria

1. 同一条 wake record 不会被两个消费者同时 claim 成功。
2. 系统重启后能识别 overdue wake，并按 `latePolicy` 处理。
3. 终态 wake record 有明确回收窗口，不形成文件型内存泄漏。
4. poison wake 会升级成可见 incident，而不是无限重试。

## Notes

- 这张单是 wake record 生命周期刀，不替 work capsule 做恢复判断。

## Findings
