# TK0387：Structural Refactor Repair Mini Baseline

## Context

当前 codegraph 复验后，仓库里与运行主链相关的真实结构债主要剩三类：

1. `src/tmux/registry.ts` <-> `src/tmux/session.ts`
2. `src/runtime/work-continuity.ts` <-> `src/runtime/wake-consume.ts` <-> `src/runtime/task-types.ts`
3. `src/tmux/responder.ts` 中 `handleTmuxSend` 认知复杂度过高

`logger` 循环已在 `tk0385` 收掉，因此这条线只继续处理剩余三刀。

## Goal / Non-Goals

### Goal

- 用最小切口收掉剩余结构债
- 每刀单独提交、单独验证
- 不改变外部产品行为

### Non-Goals

- 不发起仓库级重构运动
- 不处理 `AIDOCS/refs` 里的外部参考循环
- 不同时碰 `handlers.ts` 或 `listener.ts`

## Acceptance Criteria

1. `tk0388` 收掉 tmux 两文件循环
2. `tk0389` 收掉 runtime 三文件类型循环
3. `tk0390` 把 `handleTmuxSend` 从超大函数切成更薄的主骨架
4. 每刀都通过最小测试、`tsc --noEmit`、必要的 `codegraph` 复验

## Plan

1. 先做 `tk0388`
2. 再做 `tk0389`
3. 最后做 `tk0390`
4. 三刀做完后，再决定是否需要继续碰 `handlers.ts` / `listener.ts`

## Notes

- 这条线是辅线，不抢正式 UI 主线
- 只处理最小可证实的结构债，不做“顺手整理”
