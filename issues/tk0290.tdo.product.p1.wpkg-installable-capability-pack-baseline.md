# 任务：WPKG 可安装能力包基线

## Context

用户已经把产品主线收口到：

- 本地 appliance
- 主窗口负责设置 / 项目 / 基座入口
- 基座业务内容与主界面解耦

下一步希望：

1. 先把 `wpkg` 做成可安装能力包
2. 安装后主界面能看见
3. 安装后 agent 能发现并使用其中的 skill

但当前明确不做：

1. 市场
2. 分成
3. 支付
4. 开发者收入实现

## Goal / Non-Goals

### Goal

- 定义 `wpkg` 的最小包结构
- 定义安装后写入哪些真相源
- 定义主界面怎样看见已安装 pack
- 定义 agent 怎样发现 pack 内 skill
- 给未来商业字段和授权字段留位置

### Non-Goals

- 不实现 pack 安装器
- 不实现签名、支付、分成
- 不实现开放市场
- 不实现包下载源

## Plan

- [ ] 定义 `wpkg` 最小物理形态
- [ ] 定义 `manifest.json` 最小字段
- [ ] 定义安装后写入的注册表真相源
- [ ] 定义主界面消费口径
- [ ] 定义 agent 发现 skill 的口径
- [ ] 明确第一版不支持项

## Acceptance Criteria

1. `wpkg` 可以被描述成一个明确、简单、可安装的能力包单位。
2. 安装后主界面要读哪些文件、agent 要读哪些文件都讲清楚。
3. 当前协议已经给未来授权/商业字段留位，但不强行实现。

## Notes

- 第一版 `wpkg` 应优先等价于“带 manifest 的 zip 包”，不发明复杂二进制格式。
- 主链必须保持：`pack -> 文件真相源 -> 主界面 / agent`。
- 这张单当前只做协议和计划，不进实现。
