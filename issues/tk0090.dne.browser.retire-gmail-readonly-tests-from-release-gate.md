## Context

当前仓库的发布门槛仍被一批历史回归锁卡住，其中 `gmail-readonly` 相关测试已经不再代表真实需求：

1. `test/p5-7-r7b-gmail-contract.test.ts` 仍把 Gmail 只读合同当作必须维护的 CLI 合同。
2. `test/p5-7-r7b-gmail-readonly.test.ts` 仍把 Gmail 只读业务流当作浏览器域核心验收。
3. `docs/tasks/README.md` 仍把 `P5.7-R7B` 放在当前任务单索引中，`docs/tasks/p5-7-r7b-gmail-readonly-acceptance.md` 仍像活跃任务。

用户已明确：邮件测试不再是准确需求，应退出主回归与当前任务索引。

## Goal / Non-Goals

### Goal

- 将 Gmail 只读测试从主回归中移出
- 将 Gmail 只读验收任务从活跃任务索引中移出
- 保留历史材料，但明确归档状态
- 不让发布门槛继续被过期需求卡住

### Non-Goals

- 不删除 `src/browser/gmail-readonly.ts` 运行时代码
- 不删除 `src/cli/browser.ts` 里的 Gmail 子命令
- 不重写浏览器域整体任务历史
- 不顺手清理所有历史 issue 中的 Gmail 叙述
