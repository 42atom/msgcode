## Context

- 用户已经明确提出：主脑不仅要会响应，还要会周期性回看工作目录、检查未完成任务、监督 subagent、在完成后写日记并回收记忆与 skill 改进。
- 现有仓库已经有 `scheduler`、`jobs`、`/task`、`TaskSupervisor`、`subagent` 等积木，但缺一条正式的“主脑节律”主链。
- 对标结果显示：
  - `pi-mono` 在文件唤醒上很薄
  - `openclaw` 在 heartbeat / cron / subagent 分工上最清楚
  - `hermes-agent` 在学习循环叙事上最完整
- msgcode 需要在不加厚控制面的前提下，把这三块收口成自己的主循环。

## Goal / Non-Goals

- Goal:
  - 为每个 workspace 建立正式 heartbeat 主循环
  - 提供精确定时 alarm/cron 唤醒机制
  - 把任务巡检、subagent 监督、反思回收接进同一条主链
  - 让所有状态继续优先落到文件真相源
- Non-Goals:
  - 不新建一个重型 autonomic platform
  - 不把 `issues/` 状态真相源替换成数据库状态
  - 不在本单先做大而全的学习系统

## Plan

- [ ] 定义 workspace heartbeat 合同：输入、输出、静默 ack、触发频率
- [ ] 明确 heartbeat 物理触发策略：
  - 第一版由统一 heartbeat runner 托管
  - 不为每个 workspace 先起独立常驻进程
  - `fsevents` 只作为后续优化，不作为第一版地基
- [ ] 定义 alarm/cron 合同：只负责唤醒，不负责决策
- [ ] 明确 heartbeat 如何读取：
  - `HEARTBEAT.md`
  - `issues/*.md`
  - 派单记录
  - subagent 状态
- [ ] 明确 `HEARTBEAT.md` 与 `issues/ / .msgcode/wakeups/ / .msgcode/dispatch/` 的真相源边界
- [ ] 落一版 heartbeat 协议模板：
  - 读取顺序
  - 静默 ack 口径
  - `HEARTBEAT.md` 最小结构
- [ ] 落一版 alarm 文件协议：
  - `<workspace>/.msgcode/schedules/*.json`
  - 与 wake job / wake record 的映射
  - missed wake catch-up 规则
  - schedule 修改/删除对既有 wake record 的级联边界
  - deterministic record ID 口径
  - `wake.mode` 的物理降级语义
- [ ] 明确 reflection 最小回收步骤：
  - 日记
  - memory candidate
  - skill candidate
- [ ] 明确 reflection review 口径：
  - candidate 不自动升级成正式 memory / skill
  - 下一轮 heartbeat 优先审核 candidate 的去向
- [ ] 对齐 `/task`、`subagent`、任务看板与主循环的绑定方式

## Acceptance Criteria

1. 主脑存在一条明确 heartbeat 主循环，而不是只靠用户消息驱动。
2. heartbeat 与 alarm/cron 分工明确：
   - heartbeat = 周期 awareness / 巡检 / 推进
   - alarm/cron = 精确定时唤醒
3. heartbeat 默认读取 workspace 文件真相源，而不是另建隐藏状态层。
4. 没有需要动作时，heartbeat 能静默结束，不刷屏。
5. `HEARTBEAT.md` 被明确定义为主脑草稿/提示文件，不承载硬状态真相；与 `issues/` 或 `.msgcode/*` 合同冲突时，一律以后者为准。
6. 任务完成后，存在最小 reflection 流程，至少覆盖：
   - 日记
   - memory candidate
   - skill candidate
7. 存在一版 heartbeat 协议模板和一版 alarm 文件协议，能支撑后续实现接线。
8. alarm 文件协议明确写清：
   - schedule 删除/修改不反向污染已生成 wake record
   - 同一 schedule + 同一时刻生成同一 record ID
   - `wake.mode=now` 忙时自动降级到 `next-heartbeat`
9. reflection candidate 不会自动进入正式 memory / skill；下一轮 heartbeat 必须存在最小审核动作决定其去向。
10. 该主循环能驱动 `tk0202` 与 `tk0203`，使 persona 装配和 doc-first 派单真正转起来。

## Notes

- 这单是主线单。
- 如果没有这条主循环，前面的 persona 与 doc-first dispatch 只能算被动组件，不会自己转。
- 设计目标不是“比别人功能更多”，而是“比别人更薄、更真、更能长期跑”。
