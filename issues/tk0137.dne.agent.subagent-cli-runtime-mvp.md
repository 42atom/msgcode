## Context

当前仓库已经有 `subagent` optional skill，但仍停留在说明书层。用户希望主脑能把任务委派给 `codex` / `claude-code` 子代理，并监控任务是否完成。现有 tmux 主链已具备会话启动、消息发送、pane 读取能力，但没有正式的 `subagent` 程序合同。

## Goal / Non-Goals

- Goal: 提供最小正式 `subagent` CLI/runtime，支持 `run/status/stop`，并复用现有 tmux 执行臂。
- Goal: 让 `help-docs` 和 `subagent` skill 都能指向同一套正式合同。
- Non-Goals: 不做队列、不做自动分配、不做多任务编排器、不做新的控制面。
- Non-Goals: 不在本轮接入新的 LLM tool；先把程序合同立住。

## Plan

- [x] 创建 `subagent` runtime 薄服务，任务状态仅落盘到 workspace 下 `.msgcode/subagents/`
- [x] 增加 `msgcode subagent run/status/stop` 命令
- [x] 接入 `help-docs` 合同与 `subagent` skill 文案
- [x] 增加 targeted tests，覆盖 CLI 合同、task 落盘与 tmux 委派主链

## Acceptance Criteria

1. `msgcode subagent run <client> --goal '...' --watch` 能在 workspace 内创建 task 文件，并复用现有 tmux 会话发送任务。
2. `msgcode subagent status <taskId>` 与 `msgcode subagent stop <taskId>` 能基于同一 task 文件工作。
3. `help-docs --json` 能暴露 `subagent run/status/stop` 正式合同。
4. `subagent` skill 不再声称“还没有正式 runtime 主链”，而是引用当前 CLI 合同。
