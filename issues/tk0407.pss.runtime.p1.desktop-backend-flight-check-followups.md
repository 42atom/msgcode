# 任务：Desktop Backend Flight Check Followups

## Context

桌面线程主链已经成立：

1. thread 可写
2. user turn 先落盘
3. agent 执行走 detached child
4. thread 写入会 push UI 更新

但在继续 UI 驱动开发前，后端仍有四类该收的噪音：

1. Electron bridge 边界
2. Surface 聚合读面
3. Thread input 主链
4. `readonly` 命名债

## Goal / Non-Goals

### Goal

把这四点收成清楚的后续飞行检查队列，避免它们散落在对话里。

### Non-Goals

1. 不在这张单里直接重构所有文件
2. 不追加流式、中断、多渠道平台
3. 不把 UI 设计问题混进这张后端规整单
