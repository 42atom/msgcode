## Context

`0041` 已经收掉 `prompt.ts` 和 `tool-loop.ts` 里的主要流程裁判器，但 `src/agent-backend/routed-chat.ts` 仍残留一层会替 LLM 预判执行路径的入口逻辑：

1. `selectedLevel === "LEVEL_2"` 时，直接走 `runLmStudioChat()`，并返回：
   - `route: "no-tool"`
   - `decisionSource: "degrade"`
2. `options.forceComplexTool` 时，强制走 plan/act/report 三段式，并返回：
   - `route: "complex-tool"`
   - `decisionSource: "router"`
3. 默认主链虽然已改成 tool-loop，但当前文件仍保留：
   - 顶部注释中的 `degrade 强制 no-tool`
   - `toolsAvailable` 死变量
   - `allowNoTool: false` 这类入口层替模型做主的残影

真实历史问题已经证明，入口层只要继续提前判断 “这轮是不是 no-tool / 是否该降级 / 是否该走特殊流程”，就会重新制造随机阻断。OpenClaw 参考口径则是：默认给完整工具面，只按配置/权限过滤，不在消息入口做前置裁判。

## Goal / Non-Goals

### Goal

1. 盘点 `routed-chat.ts` 中仍生效或半残留的前置判断。
2. 删除会替 LLM 做主的入口裁判逻辑。
3. 让 `routed-chat.ts` / `tool-loop.ts` / `lmstudio.ts` 对“何时进入工具链”保持一致。
4. 补测试并完成一轮 schedule create/remove 真机 smoke。

### Non-Goals

1. 不重构整个 `agent-backend`。
2. 不新增新的 review / supervisor / fallback / parser 层。
3. 不回滚 `0041` 已拆掉的松绑成果。
4. 不顺手改 scheduler / browser / memory 等业务链。
