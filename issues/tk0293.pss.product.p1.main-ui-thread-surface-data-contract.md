# 任务：Main UI 线程主视图数据合同

## Context

当前原型已经收口到：

- 场景工作区
- 线程列表
- 当前线程聊天
- 工作状况
- 日历 / 定时任务
- 邻居未读提示

但这些区域现在还主要靠原型假数据撑着。

如果不先把文件真相源钉死，后面实现时很容易：

- 前端自己维护第二套状态
- UI 直接猜数据
- 为了补缺口再长一层中间控制面

## Goal / Non-Goals

### Goal

- 明确主界面每个区域的最小数据来源
- 明确读取优先级与缺失时的降级方式
- 明确哪些信息当前没有真相源，必须留空或隐藏

### Non-Goals

- 不直接实现前端页面
- 不新增数据库
- 不发明 UI 专用缓存层
- 不直接重构会话存储

## Plan

- [ ] 固定左栏“最近 + 当前工作区线程”的文件来源
- [ ] 固定当前线程聊天区的文件来源
- [ ] 固定右栏“工作状况 / 日历”的文件来源
- [ ] 固定邻居未读和人物入口的降级口径

## Acceptance Criteria

1. 左栏、聊天区、右栏各自读什么文件，被明确写清。
2. 不再出现“原型能画，但运行时不知道从哪读”的区域。
3. 缺真相源的区域被明确降级，而不是靠 UI 瞎编。

## Notes

- 这张单只定义读取合同，不新增 UI 专用状态层。
- 当前真实工作区先以 `family` 为例做约束收口。
- 若现有 `threads/*.md` 足够表达线程面，就先复用，不急着发明 `thread-index.json`。
- `family` 工作区对照报告已经补出，后续实现以真实文件边界为准。
- People 真相源缺口已拆成独立切片 `0296`。
- 当前实现已经补出 `appliance threads` 读面，细节见 `rp0300`。
