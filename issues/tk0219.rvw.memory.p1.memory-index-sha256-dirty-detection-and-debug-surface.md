## Context

- `tk0208` 已经确认：
  - truth = `<workspace>/memory/*.md`
  - `index.sqlite` = 可丢弃派生索引
- 但当前实现还有三个明显缺口：
  - `sha256` 没真正算
  - dirty detection 没接
  - 对外 debug/stats 可见性不够

## Goal / Non-Goals

- Goal:
  - 补真实 `sha256`
  - 补最小 dirty detection
  - 补足够用的 debug/stats 可见性
- Non-Goals:
  - 不重写 memory 检索
  - 不把 vector/hybrid 强行升格
  - 不替换 SQLite

## Plan

- [x] 在 memory index 主链里补真实 `sha256`
- [x] 用 `mtime + sha256` 或等价最小策略补 dirty detection
- [x] 明确 `memory stats/debug` 的最小输出面
- [x] 补回归，锁住“索引可重建、不是黑箱真相”

## Acceptance Criteria

1. `memory index` 不再把 `sha256` 写成空字符串。
2. 重复索引时能识别未变化文件，避免无意义重建。
3. AI 或开发者可以直接看到当前索引状态，不必猜 SQLite 里发生了什么。
4. 这单不改变 truth/index 的既有边界判断。

## Notes

- 这单就是把 `tk0208` 的审计结论变成最小实现。
- 目标不是“记忆更高级”，而是“索引更诚实、更可见”。
- 本轮已补：
  - `memory index` 不再写空 `sha256`
  - 重复索引时能直接跳过未变化文件
  - 文件 `mtime` 变了但内容没变时，只刷新 metadata，不重切 chunks
  - `memory stats` 现在能看到：
    - `vectorAvailable`
    - `documentsWithoutSha256`
    - dirty 文件及原因
- dirty 原因当前只收最小集合：
  - `sha_missing`
  - `file_missing`
  - `metadata_stale`
  - `content_changed`
  - `workspace_path_unknown`
- 运行时边界也已收口：
  - `msgcode` / `./bin/msgcode` 是正式 Node 入口
  - `npm test` 只代表 Bun 安全子集
  - 触达 SQLite 的路径不再默认放进 Bun 进程

## Current Position

- 这张单当前已完成第一轮最小实现：
  - `sha256`
  - dirty detection
  - debug/stats 可见性
- 后续若继续，不应再停留在“索引是否诚实”，而应转向：
  - 是否要补 workspace 级 stats
  - 是否要把 `vector/hybrid` 正式接成主链
