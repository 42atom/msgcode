## Context

最近的飞书图片发送排障再次暴露了两处旧漂移：

1. `getToolsForLlm()` 在无 workspace 时仍走手写旧名单，和当前真实工具能力面继续脱节。
2. runtime skill 同步实现和分层文档仍残留 `managed` 语义，但当前主线已经改成更薄的 `runtime / optional / legacy-active / retired`。

这两处不会同时立刻炸掉主链，但都会持续制造“实现已变、心智没变”的复发点。

## Goal / Non-Goals

### Goal

- 收口无 workspace 时的默认工具暴露，改走当前默认配置真相源。
- 收口 runtime skill 同步实现与主文档中的 `managed` 术语。
- 保持现有行为不变，不扩 scope 到 retired skill 清退或飞书日志增强。

### Non-Goals

- 不处理 `pinchtab-browser` 目录退役
- 不处理飞书 `isGroup/chat_type` 日志增强
- 不引入新的 skill 安装层或 marketplace
- 不重构 `legacy-active` 的实际发现逻辑

## Plan

- [x] 把 `tool-loop.ts` 与 `lmstudio.ts` 的无 workspace 工具暴露改为读取 `DEFAULT_WORKSPACE_CONFIG["tooling.allow"]`
- [x] 补回归锁，确保无 workspace 分支可见当前默认工具面，而不是旧硬编码名单
- [x] 将 `runtime-sync.ts` 中的 `managed` 命名收口为 `runtime`
- [x] 同步更新对应测试与主文档（0071 / skill layering plan）
- [x] 记录验证结果并决定是否进入下一轮整理

## Acceptance Criteria

1. 无 workspace 时，工具暴露不再使用手写旧名单。
2. `feishu_send_file` 等当前默认工具可通过无 workspace 暴露路径进入 LLM 可见集合。
3. `runtime-sync.ts` 与测试/文档中不再把 repo runtime skill 称为 `managed`。
4. 不引入新的主链层，不改变现有 runtime/optional 行为。
