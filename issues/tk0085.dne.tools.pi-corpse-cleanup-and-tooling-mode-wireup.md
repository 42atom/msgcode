## Context

当前仓库虽然已经有成熟的 `runAgentToolLoop` / `runMiniMaxAnthropicToolLoop`，但仍保留三块 PI 时代残骸：

- `/pi` 命令仍存在，并读写 `pi.enabled`
- `WorkspaceConfig` 仍保留 `pi.enabled`
- `src/providers/tool-loop.ts` 已退役但文件还在

同时，`tooling.mode` 已有配置字段与 `Tool Bus` 语义，但 `agent-backend` 的工具循环入口没有真正消费它，导致：

- `explicit` 只在执行闸门拒绝工具调用
- 但路由层仍会进入工具循环主链

用户要求：

- 彻底清掉 PI 尸体
- 不重新引入 PI 抽象
- 让 `tooling.mode` 成为当前可理解的唯一运行时工具开关

## Goal / Non-Goals

- Goal: 删除 `/pi` 命令入口与帮助文案
- Goal: 删除 `pi.enabled` 的配置读写
- Goal: 删除 `src/providers/tool-loop.ts`
- Goal: 让 `tooling.mode=explicit` 在 routed/tool-loop 入口直接走 no-tool 主链
- Non-Goals: 不重新引入 PI 包或新工具编排层
- Non-Goals: 不改 `ToolingMode` 枚举定义
- Non-Goals: 不重写 `Tool Bus` 的 allow/requireConfirm 逻辑
