# TK0388：Tmux Registry Session Cycle Break

## Context

当前 `registry.ts` 直接 import `session.ts` 中的 `normalizeRunnerType` 与相关类型，同时 `session.ts` 又直接调用 registry。

这是典型的共享定义挂错地方，而不是业务流程必须互相调用。

## Goal / Non-Goals

### Goal

- 把共享 runner 类型/归一化逻辑从 `session.ts` 拆到更薄的位置
- 让 `registry.ts` 不再依赖 `session.ts`

### Non-Goals

- 不改 tmux 对外行为
- 不改 responder/sender/streamer
- 不顺手瘦 `handleTmuxSend`

## Acceptance Criteria

1. `codegraph` 不再报告 `src/tmux/registry.ts` <-> `src/tmux/session.ts`
2. tmux 邻近测试与 `tsc --noEmit` 通过

## Plan

1. 识别共享 runner 类型与归一化函数
2. 拆到独立共享文件
3. 更新 `registry.ts` 与 `session.ts` 导入
4. 复验 cycle 与最小测试

## Notes

- 这刀只断依赖，不做功能重构
