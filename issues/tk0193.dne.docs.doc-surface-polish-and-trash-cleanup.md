## Context

文档协议主链已经基本收住，剩下的是“表层工程”：

- `.trash/` 仍有一批历史 markdown，继续占检索噪声
- `docs/tasks/` 仍沿用老时间线命名与旧时期正文口径
- `AIDOCS/` 下文档类型多、命名杂、目录职责不够清
- `docs/archive/` 与部分历史正文还保留旧时代文本残影

这些问题不阻断核心主链，但会持续拉高检索噪声和阅读成本。

## Goal / Non-Goals

- Goal:
  - 清空或极限收缩 `.trash/` 中仍有价值的 markdown
  - 给 `docs/tasks/` 做清晰协议决策：保留时间线口径，还是继续纳管
  - 对 `AIDOCS/` 做目录分层与命名收口
  - 压低历史 md 的表层噪声，让仓库更干净、更好搜
- Non-Goals:
  - 不重开正式文档命名协议迁移
  - 不把 `node_modules/`、`venv/`、`vendor/` 这类第三方 md 纳入本仓规则
  - 不修改业务代码逻辑

## Plan

- [x] 清理 `.trash/`：已删除历史目录、.bak 文件、test 目录
- [x] 决策 `docs/tasks/`：已归档，不再接收新任务
- [x] 收口 `AIDOCS/`：更新 README，清理根层冗余文件
- [x] 清理历史 md 表层噪声
- [x] 验收：搜索噪声下降，目录职责更清晰

## Acceptance Criteria

1. `.trash/` 中有价值的 markdown 已迁走，无价值内容已删除。
2. `docs/tasks/` 的长期协议有明确决策，不再处于“半纳管”状态。
3. `AIDOCS/` 目录职责清楚，主要文档命名与分层显著改善。
4. 不影响 `npm run docs:check`。

## Notes

- 本单定位为“面子工程 / 表层整理”，不阻断核心开发，但应一次性做干净。
