# 任务：轻量遥测账本合同

## Parent Task

- `tk0361`

## Context

现在仓库里已经有两种相邻但不同的观测面：

- `status.log`：共享近况簿，只记已确认决定和状态变化
- `src/tools/telemetry.ts`：工具事件的内存 ring buffer，不落盘

缺口是：

- 需要一份真正 append-only、按天轮转、文件化的轻量账本
- 但又不能把它长成第二状态系统或实时监控流

## Goal / Non-Goals

### Goal

- 定义单一 telemetry ledger 合同
- 定义最小字段、轮转和保留窗口
- 钉死和 `status.log` 的边界

### Non-Goals

- 不写代码
- 不做 metrics server
- 不做 dashboard
- 不做 Prometheus
- 不改现有 `status.log` 合同

## 结论

第一版只允许一本账本：

- `<workspace>/.msgcode/telemetry/telemetry-YYYY-MM-DD.ndjson`

不要一开始就拆 `tokens.ndjson / latency.ndjson / errors.ndjson`。
靠 `kind` 与最小字段区分事件即可。

## Ledger Shape

每行一个 JSON 对象，只允许标量和短字符串字段。

最小字段固定为：

- `ts`
- `kind`
- `source`
- `name`
- `ok`
- `durationMs`

可选字段只允许：

- `workspace`
- `errorCode`
- `tokensIn`
- `tokensOut`
- `totalTokens`
- `count`

不允许：

- 大 payload
- 长文本错误堆栈
- 原始请求/响应体
- 二次摘要字段

## Kinds

`kind` 第一版只允许：

- `tool`
- `verify`
- `probe`
- `runner`

是否继续扩展，必须先证明现有四类表达不了。

## Rotation And Retention

- 轮转方式：按天
- 文件名：`telemetry-YYYY-MM-DD.ndjson`
- 默认保留窗口：最近 `7` 天
- 超出窗口直接删最老日账本，不做额外分桶

## Writers

只允许 runtime 自己写：

- tool bus / runner / verify / probe 这类受控写入器

不允许直接写的主体：

- UI
- prompt 文本
- 普通 shell 脚本
- 模型自由输出

## Readers

允许读取汇总或局部窗口的主体：

- `probe`
- 运维排障命令
- 开发调试脚本

默认不允许读全量账本的主体：

- UI 主界面
- 普通线程 prompt
- 任何把它当实时监控流的消费者

读取口径应是：

- tail 最近窗口
- 按天聚合
- 按 `kind` / `ok` / `errorCode` 做最小统计

## Boundary Against Status Log

- `status.log` 回答“当前已经定下了什么”
- telemetry ledger 回答“运行时最近发生了哪些轻量事件”

所以 telemetry ledger 不记录：

- 决策
- 业务状态
- 原文引用行号

而 `status.log` 也不承担：

- tokens
- latency
- errorCode 分布

## Guardrails

- 单一账本，append-only，按天轮转
- 只保留最小字段，不带大 payload
- UI 可以消费汇总结果，但不能把它渲染成实时监控流

## Plan

- [x] 盘点 `status.log` 与内存 telemetry 的边界
- [x] 固定 ledger 文件形态与字段
- [x] 固定读写边界与保留窗口

## Notes

- 这张单只定义合同，不负责实现。
- 若后续实现想拆多本账本，应先回到这张单补证据，而不是直接扩写。

## Acceptance Criteria

1. 默认只有一本 `telemetry-YYYY-MM-DD.ndjson`，不预拆多本。
2. 最小字段、轮转方式、保留窗口和读写边界都已明确。
3. 账本与 `status.log` 的边界清楚，不会互相冒充。
