## Context

`read_file` 当前还保留一条历史补丁：当模型传入相对 `soul/soul.md` 且主路径不存在时，系统会静默改写成 `<workspace>/.msgcode/SOUL.md`。这违反了“不要替 AI 做主、不要静默改参”的主线，也会掩盖真实路径错误。

## Goal / Non-Goals

### Goal

- 删除 `read_file` 中的 SOUL 别名 remap
- 让路径错误按原生 `read_file` 合同失败并回灌模型
- 更新相关回归锁，移除对静默 remap 的保护

### Non-Goals

- 不改 SOUL 固定路径本身
- 不改 `getWorkspaceSoul()` 等显式读取逻辑
- 不新增新的恢复层或路径猜测
