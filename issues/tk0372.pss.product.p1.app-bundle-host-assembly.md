# 任务：App Bundle Host Assembly

## Context

`build artifact` 不是最终 app。

它还需要被一个极薄宿主壳承接，装成用户可启动的本地 Desktop App bundle。

这一步必须单独收口，因为它同时受 3 条边界约束：

- `pl0274`：Electron 只做宿主
- `rf0000`：Electron 不是第二主脑
- `rf0001`：当前 UI 约束仍然存在

所以这里缺的不是“再讨论要不要 Electron”，而是把宿主装配边界写薄、写死。

## Goal / Non-Goals

### Goal

- 定义 app bundle host assembly 的输入只来自 build artifact
- 定义宿主壳最小职责
- 定义 app bundle 输出面与启动面
- 定义 app bundle 与 dmg 链的切口

### Non-Goals

- 不重定义 build artifact 结构
- 不设计签名、公证、notarization 流程
- 不扩授权、更新、跨平台
- 不写 Electron 实现代码

## Plan

- [x] 明确 app bundle 只能消费 artifact，不直接消费源码树
- [x] 明确宿主壳只负责哪些最小职责
- [x] 明确 app bundle 的最小输出与验收
- [x] 明确为什么这一步不能和 dmg 链合并

## 输入面

`tk0372` 只承认一个上游输入：

- `tk0371` 定义好的 `bundle-root`

也就是只认：

- `bundle-root/runtime/`
- `bundle-root/bin/msgcode`
- `bundle-root/bootstrap/`
- `bundle-root/appliance.manifest`

这里不再直接读取：

- `src/**`
- `package.json`
- `bootstrap/Brewfile*`
- 任意开发态 `npm install` / `tsx` 启动链

一句话：

- app bundle 只能装 artifact，不能重新理解源码树

## 宿主壳最小职责

app bundle 在这里的职责只有宿主装配，不新增第二主脑。

Electron host 只做 4 件事：

1. 承接 `bundle-root`，把 runtime 与 launcher 放进本地 `.app` 结构
2. 提供稳定启动入口，拉起 bundled `msgcode` runtime
3. 打开本地主 UI / sidecar 入口，但不接管业务真相
4. 暴露最小宿主元信息，让系统知道这是一份可启动的 Desktop App

它明确不做：

1. 重写 runtime 目录结构
2. 回写 artifact 合同
3. 直接承担任务执行、dispatch、memory、skill 逻辑
4. 在 app bundle 这一层引入授权中心、自动更新中心、签名、公证流程

## 最小输出面

这一步完成后，只要求形成一份本地可启动的 Desktop App bundle。

最小输出可以收口为：

1. 一个 `.app` bundle 外壳
2. 外壳内承接 `bundle-root` 的 runtime/launcher
3. 一个稳定启动面
4. 一个明确的 host 边界：宿主拉起 runtime，UI 继续只读 runtime/file-first 真相

验收重点不是“分发成功”，而是：

- 这份 `.app` 已经可以作为 Electron 宿主壳独立存在
- 下游分发链只需要再消费这份 `.app`

## 与 artifact 的切口

`tk0371` 和 `tk0372` 的切口很硬：

- `tk0371` 回答“要装什么”
- `tk0372` 回答“怎么把它装进宿主壳”

所以 app bundle 这一步不再决定：

- runtime 该编译什么
- bootstrap 该带什么
- manifest 长什么样

这些都以上游 artifact 合同为准。

## 与 dmg 的切口

app bundle 和 dmg 不是同一步。

原因很简单：

1. `.app` 是本地可启动宿主
2. `dmg` 是分发容器

如果把两步揉在一起，会重新混掉两类职责：

1. 宿主装配
2. 分发、签名、公证

所以这里必须先把 `.app` 收干净，再让 `tk0373` 去处理：

- dmg 外壳
- 签名
- notarization

## 排除项

`tk0372` 明确不进入：

1. build artifact 结构重定义
2. dmg 打包、签名、公证
3. 授权中心、更新中心
4. 跨平台安装器
5. 任何 Electron 具体实现代码

## Acceptance Criteria

1. 能明确说明 app bundle 的输入只来自 `tk0371` 所定义的 artifact。
2. 能明确说明宿主壳只做 host assembly，不承接第二主脑职责。
3. 能明确区分 app bundle 完成与 dmg 分发完成不是一回事。
4. 文档不扩到授权中心、更新中心、跨平台安装器。

## Notes

- 这张单是 `tk0370 / pl0362` 的第 2 个子产物切片。
- 它的目标是把“能启动的 app bundle”从 artifact 与 dmg 之间单独切出来。
