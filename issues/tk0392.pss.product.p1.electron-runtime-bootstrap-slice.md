# 任务：Electron Runtime Bootstrap Slice

## Context

当前仓库已经有：

- `.app` bundle host
- `src/ui/main-window/readonly-thread-surface.ts` 正式 UI 骨架

但还没有：

- `electron` 依赖
- main 进程入口
- preload 入口
- renderer 宿主入口

这意味着 `tk0391` 虽然已经立为 host bridge 切片，但现在没有真实 Electron 宿主可接。

## Goal / Non-Goals

### Goal

1. 给仓库补最薄 Electron runtime bootstrap。
2. 只建立：
   - Electron 依赖
   - main 入口
   - preload 入口
   - renderer 宿主入口
3. 让 `tk0391` 后续有真实宿主承载面。

### Non-Goals

1. 不做业务桥接。
2. 不做 `workspace-tree/thread` 读面实现。
3. 不做多页壳。
4. 不做新服务层。
5. 不改 `ui-protype`。

## Plan

- [x] 补最薄 Electron runtime bootstrap task，固定先后顺序 `tk0392 -> tk0391`
- [x] main 只承载窗口启动
- [x] preload 只承载未来桥接占位
- [x] renderer 只承载正式 UI 入口

## Acceptance Criteria

1. 仓库中存在最薄 Electron runtime 入口代码落点。
2. main / preload / renderer 职责分清。
3. 不引入 REST / GraphQL / WebSocket 层。
4. `tk0391` 之后的 host bridge 有真实宿主承载面。

## Notes

### 为什么不能直接继续做 tk0391

`tk0391` 的职责是 bridge，不是 bootstrap。

它假设：

- 已有宿主 main
- 已有 preload
- 已有 renderer 入口

而这些在仓库里当前都不存在。

如果现在直接把 bridge 代码塞进 `tk0391`，只会把“宿主 runtime 建立”和“bridge 实现”重新混成一张单。

### 为什么 tk0375 不能承载

`tk0375` 只承载 `.app` 目录装配，不承载 Electron runtime。

它当前的宿主只是 launcher script，把 bundled `msgcode` 放进 `.app`，还不是 Electron 运行时。
