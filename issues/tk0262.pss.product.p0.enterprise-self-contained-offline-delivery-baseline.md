# 任务：本地 Appliance 自包含交付基线

## Context

当前产品口径已经收口为：

- 面向小微机构本地部署使用
- `Mac mini` appliance
- 单机、单组织、单管理员优先
- 当前现役运行面仍是 `Feishu-first`

但当前仓库默认交付仍依赖：

- `brew`
- `npm install`
- 源码态 `tsx`
- 若干本机预装工具

这不满足 appliance 交付的正式要求：

- 现场安装不能再临时下载一堆依赖
- 机器必须先知道自己服务的是哪个机构
- 能力包与运行边界要有最小合同

## Goal / Non-Goals

### Goal

- 明确本地 appliance “自包含、不下载”的正式定义
- 区分：
  - 离线安装
  - 运行期外网依赖
  - 完全内网运行
- 输出 appliance 的能力分层与依赖清单
- 固定最小机构卡片字段，只允许：
  - 名称
  - 交税地
  - 统一社会信用代码

### Non-Goals

- 不直接实现打包工具
- 不直接修改 runtime
- 不先决定 UI 框架
- 不先做复杂机构对象模型
- 不先做 license / DRM 联网控制面

## Plan

- [x] 盘点当前安装/运行外部依赖
- [x] 定义 appliance 最小核心包与可选扩展包
- [x] 明确“离线安装版”和“完全内网版”的差异
- [x] 固定最小机构卡片协议，先只允许 3 个字段

## Acceptance Criteria

1. 能清楚回答“当前哪些依赖阻止 appliance 自包含交付”。
2. 有明确的 appliance 核心能力边界。
3. 机构身份最小面被固定为 3 个字段，不长成复杂组织模型。
4. 有正式验收口径，而不是抽象愿景。
