## Context

- 用户希望初始化时“一次性安装到位”，除了主链 Bash，还能可选安装一批推荐工具，减少后续环境摸索。
- 但这些推荐工具不应和 `bash` 主链合同绑死；否则默认层会重新变胖，README 与安装现实也容易再次打架。
- 因此需要一条配套但独立的安装单：把推荐工具集做成可选增强入口。

## Goal / Non-Goals

- Goal:
  - 提供一个一键可选的 agent 工具集安装入口
  - 明确区分“默认必装”与“推荐增强”
  - 给出最小 smoke/doctor 验收口径
- Non-Goals:
  - 不把增强包混进默认主链
  - 不做重型 profile 平台或控制面
  - 不改变 `bash` 工具的正式 shell 合同

## Plan

- [x] 从现有参考资产中筛出推荐增强工具，按“agent-first”原则收口成最小清单
- [x] 为增强包提供单独入口，如独立 Brewfile / bootstrap 参数 / 等价薄入口
- [x] 补最小 smoke/doctor 验证，确认关键工具已安装
- [x] 文档明确：这是可选增强，不是默认主链

## Acceptance Criteria

1. 存在单独的可选增强安装入口，不与默认安装混淆。
2. 增强包工具清单有正式真相源，而不是散落在 README。
3. 至少有一组最小验证步骤，确认关键推荐工具已可用。
4. 文档明确区分：
   - 默认主链：托管 Bash
   - 可选增强：agent 工具集
