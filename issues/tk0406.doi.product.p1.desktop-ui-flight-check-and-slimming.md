# 任务：Desktop UI Flight Check And Slimming

## Context

桌面 thread 主链现在已经成立：

1. 可写 thread 有 composer
2. user turn 先落盘再异步跑 agent
3. detached child 承载写桥重执行
4. thread 文件写入能推 UI 更新

但正式 UI 代码仍有三类债：

1. 旧命名残留：`readonly` 已不再完全准确
2. renderer 与 UI shell 存在重复小逻辑
3. 左中右主链虽已成立，但局部职责还不够收口

## Goal / Non-Goals

### Goal

做一次飞行检查式规整：

1. 收命名
2. 去重复
3. 收边职责
4. 保证正式 UI 更像可实现主链

### Non-Goals

1. 不重写 surface 合同
2. 不重写 renderer 架构
3. 不新增 UI 控制层
4. 不顺手做流式、中断、右栏扩展

## Plan

1. 先立规整真相单
2. 再做一刀一刀的小规整
3. 每刀都只处理一类噪音
4. 保持测试与 build 常绿

## Acceptance Criteria

1. 正式 UI 左中右主链不被旧名字误导
2. `renderer/ui shell/main` 的小型重复逻辑减少
3. 不引入发送、更新、选择交互回归
4. 每次规整都可被测试和 build 验证

## Notes

### 必做

1. 只做小规整，不改 surface 合同
2. 先处理共享真相和重复逻辑，再动命名
3. 每刀只处理一类噪音，避免混改
