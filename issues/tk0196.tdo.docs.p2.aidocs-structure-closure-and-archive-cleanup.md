## Context

- 当前仓库正式真相源已经收口到 `issues/` + `docs/plan/`。
- `AIDOCS/` 仍是全仓最杂的区域，主要问题不是文件名，而是目录职责重叠：
  - `refs/` 与 `references/` 重复
  - `design/`、`notes/` 仍混着现役草稿和历史笔记
  - `imessage-compare/`、`p5-7-r3c-data/` 这类历史资料缺少统一归档位
  - `AIDOCS/README.md` 还保留不存在或过时的目录口径

## Goal / Non-Goals

- Goal:
  - 合并重复目录，收清 `AIDOCS` 目录职责
  - 让现役辅助材料、历史参考、一次性数据集分层清楚
  - 修正 `README`，避免继续误导新同学
- Non-Goals:
  - 不把 `AIDOCS` 升格为正式真相源
  - 不重做正式文档命名协议
  - 不重写大批历史正文内容

## Plan

- [ ] 盘点 `AIDOCS` 现役 / 历史 / 冗余目录
- [ ] 合并 `references/` → `refs/`
- [ ] 处理 `imessage-compare/`、`p5-7-r3c-data/` 等历史资料归位
- [ ] 为 `design/`、`notes/` 补清晰的“草稿/笔记”定位
- [ ] 更新 `AIDOCS/README.md`
- [ ] 跑 `npm run docs:check`

## Acceptance Criteria

1. `AIDOCS` 不再存在明显重复职责目录。
2. `README` 与实际目录一致。
3. 历史资料有明确归档位，不再像现役辅助输入。
4. `npm run docs:check` 通过。

## Notes

- 优先做目录与职责收口，文件名只在顺手且低风险时修。
- 原则：现役辅助材料保留；纯历史参考和一次性数据下沉归档。
