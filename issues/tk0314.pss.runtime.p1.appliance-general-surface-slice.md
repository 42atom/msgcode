# 任务：appliance general 读面切片

## Context

Settings 页里“通用”目前原型字段偏多：

1. 默认场景工作区
2. 多语言
3. 日志路径
4. 启动方式
5. 账号连接

但当前仓库真正有硬真相源的只有：

- `WORKSPACE_ROOT`
- msgcode 日志路径
- launchd 启动事实

如果为了原型强行把其他字段也做出来，就会长出第二套 settings 中心。

## Goal / Non-Goals

### Goal

- 增加最小 `msgcode appliance general --workspace <...> --json`
- 让它输出：
  - 工作区根目录
  - 日志路径
  - 启动方式 / launchd 状态

### Non-Goals

- 不实现“默认场景工作区”偏好写入
- 不实现多语言
- 不实现账号连接
- 不新建 `general.json` / `settings.json`
- 不在这里顺手做 doctor/log tail

## Plan

- [x] 新增 workspace-general 读面
- [x] 增加 appliance general CLI
- [x] 补黑盒测试
- [x] 做一轮圆桌复核

## Acceptance Criteria

1. `appliance general` 能稳定输出 JSON。
2. 只复用现有事实：`WORKSPACE_ROOT`、log 路径、launchd 状态。
3. 缺少 `WORKSPACE_ROOT` 时返回 warning 空态，而不是系统自造默认值。
4. 没有新建第二份 settings 真相源。

## Notes

- 这条切片只负责“通用”页里已经有硬事实的那一部分。
- 语言 / 账号连接 继续留在原型层，不提前实现。
