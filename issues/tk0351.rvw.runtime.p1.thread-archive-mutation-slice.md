# 任务：thread archive / restore 切片

## Context

workspace archive/restore 已有正式 mutation。

接下来 archive 主线还差半边：

- 当前工作区 thread markdown 的 archive / restore

这条如果没有正式 CLI，UI 就只能自己改文件或继续假装“恢复按钮存在”。

## Goal / Non-Goals

### Goal

- 增加 `msgcode appliance archive-thread --workspace ... --thread-id ... --json`
- 增加 `msgcode appliance restore-thread --workspace ... --thread-id ... --json`
- 只移动 thread markdown 文件
- 不碰 session jsonl

### Non-Goals

- 不做 session archive
- 不做 thread 批量归档
- 不改 `/clear`
- 不做 archive 索引表

## Plan

- [x] 增加 thread archive mutation
- [x] 增加 thread restore mutation
- [x] 用 `threadId` 找到真实文件
- [x] 补不存在 threadId 的失败路径

## Acceptance Criteria

1. 活跃线程能移入 `archived-threads/`。
2. 归档线程能恢复回 `threads/`。
3. 不存在的 `threadId` 直接失败。
4. `/clear` 语义不被污染。

- 这条线服务 archive，不替代 `/clear`
