## Context

桌面能力（截图/视觉/点击）在 macOS 上依赖 TCC 权限。若等到首次工具调用才发现未授权，用户体感是“程序卡住/有 bug”。需要把权限入口触发前置到启动链路，让用户在还没开始跑任务前就能看到系统设置页并完成授权。

## Goal / Non-Goals

- Goal: msgcode daemon 启动时 best-effort 打开 Accessibility/Screen Recording 系统设置页，并尝试触发一次 screen capture 权限提示；写 marker 避免重复打扰。
- Non-Goals: 不做拦截/代决/gate；不自动授予权限；不与特定 provider（ghost）耦合为唯一入口。

## Plan

- [x] 新增 `src/runtime/desktop-permissions-preauth.ts`（独立模块）
- [x] `startBot()` 启动链路调用（best-effort）
- [x] 增补最小测试（非 launchd/非 force 直接跳过）
- [x] 更新 `docs/CHANGELOG.md`
- [x] 跑回归并提交

## Acceptance Criteria

1. launchd daemon 启动时会打开系统设置的 Accessibility/Screen Recording 页面，并写 marker（避免每次启动都弹）。
2. 功能失败不影响 daemon 启动（best-effort）。
3. 实现不引入新的控制面/审批层。

## Notes

- marker: `~/.config/msgcode/flags/desktop-permissions-preauth.v1.json`
- disable: `MSGCODE_DESKTOP_PREAUTH=0`
- force: `MSGCODE_DESKTOP_PREAUTH_FORCE=1`
