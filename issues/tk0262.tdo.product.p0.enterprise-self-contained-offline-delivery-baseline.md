# 任务：企业版自包含离线交付基线

## Context

当前仓库默认交付仍依赖：

- `brew`
- `npm install`
- 源码态 `tsx`
- 若干本机预装工具

这不满足企业内网“安装时不下载”的正式要求。

## Goal / Non-Goals

### Goal

- 明确企业版“自包含、不下载”的正式定义
- 区分：
  - 离线安装
  - 运行期外网依赖
  - 完全内网运行
- 输出能力分层与依赖清单

### Non-Goals

- 不直接实现打包工具
- 不直接修改 runtime
- 不先决定 UI 框架

## Plan

- [ ] 盘点当前安装/运行外部依赖
- [ ] 定义企业版最小核心包与可选扩展包
- [ ] 明确“离线安装版”和“完全内网版”的差异

## Acceptance Criteria

1. 能清楚回答“当前哪些依赖阻止自包含交付”。
2. 有明确的企业版核心能力边界。
3. 有正式验收口径，而不是抽象愿景。

## Notes

- 这张单只定义企业版交付基线，不直接决定 UI 与模型后端方案。
- 后续安装链和 App 方案都应以这张单的基线为前提。
