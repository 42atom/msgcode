# 任务：0363 最小实现切片

## Parent Task

- `tk0363`

## Context

现有 runtime 里已经存在三种相邻但不同的轻量观测事实：

- `src/tools/telemetry.ts`
  - 已有 `ToolEvent` 内存 ring buffer
  - 已有 `recordToolEvent()` 单一写点
  - 已有 `getToolStats()` 读侧聚合
- `src/probe/index.ts`
  - 已有 `runAllProbes()` / `runSingleProbe()` 探针汇总
  - 但现在是现算报告，不是 append-only 事件账本
- `src/runtime/status-log.ts`
  - 已有共享近况簿
  - 但表达的是决策/状态，不是 runtime telemetry

问题不在“没有事件”，而在“没有一条文件化、append-only、最小字段的主链”。

按奥卡姆收口，第一刀只接现成最稳定的单一写点：

- `recordToolEvent()`

不同时把 `verify`、`probe`、`runner` 一起拉进来。

## 已有轻量事件

### 已有稳定写点

- `src/tools/bus.ts`
  - 工具成功
  - 工具失败
  - 策略拒绝
  - 参数校验失败
  - 最终都统一进入 `recordToolEvent()`

### 已有事件结构，但还不是 ledger

- `ToolEvent`
  - `workspacePath`
  - `tool`
  - `source`
  - `durationMs`
  - `ok`
  - `errorCode`
- `StatusReport` / `ProbeResult`
  - 有 probe 状态和 message
  - 但没有单一 append-only 写入口

### 明确不属于 0363 第一刀

- `status.log`
  - 决策/状态真相，不是 telemetry
- `.msgcode/evidence/*.json`
  - 失败证据真相，不是 telemetry
- provider / inbound 快照
  - 属于 0362 或后续子线，不属于 ledger 第一刀

## Goal / Non-Goals

### Goal

- 增加单一 telemetry ledger helper
- 把 tool bus 事件按天追加写入 `telemetry-YYYY-MM-DD.ndjson`
- 保留现有内存 ring buffer 与 `/toolstats` 读法，不打断现有工具统计主链

### Non-Goals

- 不写 verify ledger
- 不写 probe ledger
- 不写 runner ledger
- 不改 `status.log` 合同
- 不做 dashboard
- 不新增 server、数据库、控制面

## Why This Slice First

- `recordToolEvent()` 已经是单一写点，最适合做 append-only write-through
- `probe` 现在更适合做读侧，而不是先做第二个写侧
- `verify` 已在 `0365` 落了证据面，再同时做 telemetry 只会和 0362/0365 撞边
- `runner` 事件分散在多处，先碰会逼出新的中间层

## Ledger Shape For This Slice

```json
{
  "ts": "2026-03-21T00:00:00.000Z",
  "kind": "tool",
  "source": "tool-bus",
  "name": "bash",
  "ok": false,
  "durationMs": 321,
  "workspace": "/abs/workspace",
  "errorCode": "TOOL_EXEC_FAILED",
  "count": 1
}
```

约束：

- 第一刀只写 `kind=tool`
- 一行一个 JSON 对象
- 只保留短字符串和标量
- 不写 `errorMessage`
- 不写 `artifactPaths`
- 不写 payload / preview / prompt 文本

## Writers / Readers / Full-Scan Ban

### Writers

- 第一刀只允许 `src/tools/telemetry.ts` 里的 `recordToolEvent()` 写

### Readers

- 后续读侧正式入口：`probe`
- 可接受的消费者：运维排障脚本、开发调试脚本
- 第一刀不要求改现有读侧代码

### 不能全量扫

- UI 主界面
- 普通线程 prompt
- `/toolstats`
- 任何把 ledger 当实时监控流的消费者

读取原则应保持：

- tail 最近窗口
- 按天聚合
- 按 `kind` / `ok` / `errorCode` 做最小统计

3. `probe` 被明确为后续读侧入口，但第一刀不新增全量读取主链。
4. 不新增 server、数据库、控制面。
