## Context

- `docs/tasks/README.md` 已明确声明本目录归档，不再接收新任务。
- 但大量老任务单仍物理留在 `docs/tasks/`，会继续制造“现役目录里还有老命名任务单”的错觉。
- 这不是命名协议问题，而是目录职责还没彻底收口。

## Goal / Non-Goals

- Goal:
  - 把 `docs/tasks/*.md` 的历史任务单整批迁入 `docs/archive/`
  - `docs/tasks/` 最终只保留 `README.md`
  - 更新关键入口与必要回链，避免误导
- Non-Goals:
  - 不重写历史正文
  - 不重命名单个历史任务单
  - 不重开文档协议迁移

## Plan

- [ ] 盘点 `docs/tasks/` 历史文件与目标归档目录
- [ ] 批量 `git mv` 到 `docs/archive/tasks/` 或等价归档目录
- [ ] 更新 `README` / 入口文档 / 必要回链
- [ ] 跑 `npm run docs:check`

## Acceptance Criteria

1. `docs/tasks/` 只保留 `README.md`。
2. 历史任务单已进入明确归档目录。
3. 入口文档不再把 `docs/tasks/` 表述成现役任务目录。
4. `npm run docs:check` 通过。

## Notes
