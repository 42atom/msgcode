## Context

`verify` 的状态机和 run-event 裁判权已经在 `0113` 退出，但 `tool-loop` 收尾阶段仍然会执行 `runVerifyPhase()`，再额外做一轮文件存在性 / exitCode 检查，并把 `verify` journal / `verifyResult` 塞进返回值。这仍然让主链多了一层系统检查。

## Goal / Non-Goals

### Goal

- 删除 `tool-loop` 中的 verify hot path
- 删除 tool-loop 返回中的 verify phase 结果与 verify action journal
- 让工具执行链只保留真实 `act` journal 与模型原话

### Non-Goals

- 本轮不删除 `VerifyResult` / `verifyEvidence` 类型
- 本轮不改 task-supervisor 外部注入执行器可返回的 `verifyResult`
- 本轮不改其他预算 / 安全边界

## Plan

- [x] 新建 `0114` issue 与对应 plan，冻结边界
- [x] 删除 `tool-loop` 的 `runVerifyPhase()` 及两处调用点
- [x] 更新直接锁 verify hot path 的 tool-loop 回归测试
- [x] 更新 `docs/CHANGELOG.md`
- [x] 跑定向测试、类型检查和 docs:check

## Acceptance Criteria

- `runAgentToolLoop` / `runLmStudioToolLoop` 收尾阶段不再执行 verify phase
- tool-loop 的 `actionJournal` 不再出现 `phase=verify`
- tool-loop 返回的 `verifyResult` 默认为 `undefined`
