# 任务：企业版 App Shell 与 UI 方向

## Context

用户已经明确提出：

- 企业版可能需要 App
- 业内常见选择是 Electron
- 但当前系统核心仍是 CLI / daemon / Feishu-first 主链

这意味着需要先回答“App 是不是正式主交付形态”，而不是直接开做 Electron。

## Goal / Non-Goals

### Goal

- 明确企业版是否需要 App shell
- 明确 UI 最小范围：
  - 配置
  - 运行状态
  - 日志/诊断
  - 工作区与任务视图
- 明确 Electron 与替代路线的取舍口径

### Non-Goals

- 不直接搭 Electron 工程
- 不直接做视觉设计稿
- 不替跨平台模型问题做结论

## Plan

- [ ] 盘点企业场景下 CLI 不足的具体点
- [ ] 列出最小 App/UI 面，不先膨胀成全平台工作台
- [ ] 对比 Electron 与替代路线的成本、风险、复用度

## Acceptance Criteria

1. 能说明为什么需要 App，或为什么不需要。
2. 能给出最小 UI 面，而不是抽象“做个客户端”。
3. Electron 不会被当成默认答案，而是有对比结论。

## Notes

- 这张单先做产品与架构方向判断，不直接进入前端实现。
- App shell 必须服务企业交付，不为“看起来完整”硬做桌面端。
