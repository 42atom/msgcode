## Context

真实 Feishu 验收已经证明，只看“能不能完成任务”还不够。  
在执行前，主脑可能根本：

- 看不到 skill
- 没去读 skill
- 读了但没读懂正式合同

如果直接上完整执行 case，就会把“skill 没暴露”“skill 没同步”“主脑没理解合同”和“执行链有 bug”混在一起，导致定位成本很高。

## Goal / Non-Goals

### Goal

- 冻结一套 skill 真实验收的标准前置流程
- 把“认知关 -> 执行关”写入现有 live corpus / Feishu BDD 验收文档
- 让以后所有 skill 测试先验证主脑是否看见、读到、读懂

### Non-Goals

- 不修改运行时代码
- 不修改 tool-loop / prompt / tool surface
- 不新增新的测试框架

## Acceptance Criteria

- `skill-live-prompt-corpus-v1.md` 明确要求所有 skill 测试先过认知关
- `feishu-live-bdd-acceptance-suite-v1.md` 明确加入前置认知关
- 文档中明确认知关至少包含：
  - 能不能看到 skill
  - 能不能读取 skill
  - 能不能复述 skill 的正式合同和适用边界
  - 如果用户要求某件事，主脑将如何做
- 文档中明确：只有认知关通过后，才进入完整执行关

## Notes

- 本次规则来源于真实 Feishu `subagent` 验收：
  - 第一次失败是因为运行时 optional skill 没同步
  - 同步后，主脑才在真实群里正确读到 `subagent` skill
- 这说明“认知关”不是礼节，而是必要的诊断前置步骤
