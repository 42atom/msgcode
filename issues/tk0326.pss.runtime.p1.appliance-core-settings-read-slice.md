# 任务：appliance core settings 聚合读实现切片

## Context

`0323` 已经把边界钉死：

- 只聚合 `profile/general/capabilities`
- `doctor` 继续独立
- 不起 RPC
- 不做 settings 中心

现在缺的只是把这条入口做出来。

## Goal / Non-Goals

### Goal

- 增加最小 `msgcode appliance settings --workspace <...> --json`
- 保留三段各自的：
  - `status`
  - `warnings`
  - `errors`
  - `data`
- 顶层只做最小状态归约

### Non-Goals

- 不引入 mutation
- 不把 `doctor` 并进来
- 不 shell-out 回调现有子命令
- 不加缓存、默认值填充器、后台守护进程

## Plan

- [x] 在 `src/cli/appliance.ts` 增加 `settings` 子命令
- [x] 直接进程内复用现有三段 read surface
- [x] 保留分段 status/warnings/errors/data
- [x] 补黑盒测试
- [x] 做一轮圆桌复核

## Acceptance Criteria

1. `appliance settings` 只聚合三段 core settings。
2. `doctor` 不出现在聚合结果里。
3. 顶层 warning 不会把分段 warning 吃掉。
4. 工作区不存在时，顶层和分段都诚实报错。

## Notes

- 这刀是 UI 初始化入口，不是新的 settings 平台。
- 仍然保持“子面是真相，聚合只是 fan-in”。
