# 任务：appliance workspace-tree 读面切片

## Context

现有正式读面已经有：

- `appliance threads`：单工作区
- `appliance archive`：单工作区 archive

但主界面左侧需要的是：

- 工作区根目录
- 活跃工作区
- 每个工作区的线程摘要

这层如果继续塞进 `workspace-thread-surface`，就会把单工作区读面和根扫描读面混在一起。

## Goal / Non-Goals

### Goal

- 增加 `msgcode appliance workspace-tree --json`
- 只输出活跃工作区与线程摘要
- `.archive` 和隐藏目录不进入活跃面
- 坏线程文件只给 warning，不中断整棵树

### Non-Goals

- 不输出当前线程正文
- 不输出 people/workStatus/schedules
- 不实现归档页
- 不实现排序偏好

## Plan

- [x] 增加 `msgcode appliance workspace-tree --json`
- [x] 根目录只扫描活跃工作区
- [x] `.archive` 和隐藏目录不进入活跃面
- [x] 每个工作区只输出线程摘要
- [x] 坏线程文件只给 warning，不中断整棵树

## Acceptance Criteria

1. 根目录存在时，能输出活跃工作区及其线程摘要。
2. 缺少 `.msgcode/threads` 的工作区仍会出现在树里，只是 `threads=[]`。
3. `.archive` 不进入活跃工作区列表。
4. 坏线程文件只产生 warning，不让整条命令失败。

## Verify

`PATH="$HOME/.bun/bin:$PATH" bun test test/p7-tk0355-appliance-workspace-tree-contract.test.ts && ./node_modules/.bin/tsc --noEmit && npm run docs:check`

## Notes

- 左树只读现有目录和 thread front matter，不新造索引文件。
- 用户偏好排序仍属于 UI 层，真相源以后单独定，不在这刀偷塞。
